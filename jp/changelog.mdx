### コネクタ: Twitch、Twilio、Linear、Telegram、Contentful

![2026 年 3 月 16 日の共有コネクタ](/images/zenopus/changelog/2026-mar-16-shared-connectors.png)

アプリは、通信、コンテンツ、開発者のワークフローのためのより多くのサービスと統合できるようになりました。これらの統合により、アプリが外部プラットフォームと直接対話できるようになり、ZenOpus で構築できる内容が拡張されます。

[Twitch](/jp/integrations/twitch) を使用すると、アプリが Twitch API にアクセスできるため、チャンネル上のアクティビティに応答するオーバーレイ、ダッシュボード、モデレーション ツール、コミュニティ アプリを構築できます。

[Twilio](/jp/integrations/twilio) を使用すると、アプリはメッセージングや音声を通じてユーザーにアクセスできます。 Twilio を使用すると、アプリは SMS 通知を送信し、電話ベースのワークフロー、Support WhatsApp メッセージングを管理し、アプリ インターフェイスの外部でユーザーと通信する機能を構築できます。

[Linear](/jp/integrations/linear) を使用すると、アプリが Linear Workspace 内の課題、プロジェクト、チーム、サイクル、コメントと対話できるため、ワークフローを自動化し、進捗状況を追跡するダッシュボードを構築し、エンジニアリング タスクをアプリケーションに直接接続できます。

[Telegram](/jp/integrations/telegram) を使用すると、アプリは Telegram Bot API を使用して、チャット、グループ、チャネルを通じてユーザーと通信できるようになります。 Telegram ボットはアプリの会話インターフェイスとして機能するため、ユーザーは更新情報を受信したり、情報をリクエストしたり、チャットから直接ワークフローをトリガーしたりできます。

[Contentful](/jp/integrations/contentful) を使用すると、ZenOpus アプリは Content Delivery API を介して Contentful スペースから公開コンテンツを取得できます。これにより、CMS コンテンツを直接活用して動的なページ、製品カタログ、ブログ、ドキュメントを構築できます。

Workspace 管理者と所有者は、*[設定] → [コネクタ] → [共有コネクタ]* でこれらのサービスに接続でき、Workspace 内のすべてのプロジェクトのすべてのメンバーがこれらのサービスを利用できるようになります。

### パーソナル コネクタ: Polar および Sanity

![2026 3 16 Mcp](/images/zenopus/changelog/2026-mar-16-mcp.png)

2 つの新しい MCP サーバーが [パーソナル コネクタ](/jp/integrations/mcp-servers) として利用可能です。これにより、ZenOpus は外部サービスと直接対話し、アプリの構築中にそのデータをコンテキストとして使用できるようになります。

**Polar** は ZenOpus を Polar の請求設定に接続するため、アプリに実際のサブスクリプションと請求のコンテキストを組み込むことができます。 ZenOpus は製品、顧客、サブスクリプション、および価格設定にアクセスできるため、SaaS アプリ、チェックアウト フロー、サブスクリプション管理機能を足場にすることができます。

**Sanity** は ZenOpus を Sanity CMS に接続し、エージェントがアプリの構築中にドキュメント、スキーマ、構造化コンテンツにアクセスできるようにします。 ZenOpus は、コンテンツ モデルを読み取り、CMS 構造に合わせたアプリ、ランディング ページ、インターフェイスを生成できます。

個人コネクタは、[設定] → [コネクタ] → [個人コネクタ]* で構成できます。

### 認証メール

デフォルトの ZenOpus Cloud 送信者ではなく、独自のドメインから [認証メール](/jp/features/custom-emails) を送信できるようになりました。これにより、配信可能性が向上し、ドメインの評判が保護され、ログイン エクスペリエンスがブランドと一貫した状態に保たれます。

ZenOpus はドメインの検証と認証 (DNS、SPF、DKIM、および DMARC) を自動的に管理するため、外部の電子メール プロバイダーのセットアップは必要ありません。送信ドメインを構成し、ブランドとテンプレートをカスタマイズし、サインアップ確認、パスワード リセット、マジック リンク、招待状、電子メール変更確認、再認証メッセージなどの認証電子メールを管理できます。

ZenOpus Cloud プロジェクトの有料プランで利用できます。

### Workspace の知識

共有ルールと規約を一度定義すると、Workspace 内のすべてのプロジェクトに自動的に適用できるようになりました。

[Workspace ナレッジ](/jp/features/knowledge) は、コーディング標準、アーキテクチャ パターン、優先ライブラリ、ブランド ボイス ガイドラインなど、プロジェクト間で一貫性を保つ必要がある指示用に設計されています。これにより、チームはすべてのプロジェクトで同じ指示を繰り返すことがなくなり、新しいプロジェクトが自動的に同じ規則に従うようになります。

Workspace 管理者と所有者は、*設定 → ナレッジ* または *プロジェクト設定 → ナレッジ* から Workspace ナレッジを管理できます。

### 監査ログ

Workspace [監査ログ](/jp/features/audit-logs) が Enterprise プランで利用できるようになりました。監査ログは、Workspace 全体にわたる検索可能なアクティビティの履歴を提供するため、変更を確認し、アクセスを監視し、予期しない動作を調査できます。

Workspace 管理者と所有者は、アクション、メンバー、または時間範囲ごとにログをフィルター処理して、Workspace メンバーシップの変更、プロジェクト アクティビティ、認証イベント、統合の変更、Workspace 構成の更新などのイベントを確認できます。

各エントリには、アクションを実行した人、アクションがいつ発生したか、何が変更されたか、どのリソースが影響を受けたかが表示されます。エントリを展開して、イベントに関連付けられた詳細なメタデータを表示できます。

監査ログは、*設定 → Workspace → 監査ログ* で利用できます。

### Workspace メンバーの一括管理

Workspace 管理者と所有者は、一度に [複数のメンバーを管理](/jp/features/workspace#Workspace-メンバーと招待状を管理する) できるようになりました。 [ユーザー] ページで *選択モード* を有効にして、ロールの変更、クレジット制限の設定、ユーザーの削除、または複数のメンバーの招待の取り消しを同時に行います。

これにより、大規模なワークスペースの管理オーバーヘッドが大幅に削減されます。

### Workspace 招待リンク

編集者以上は、電子メール アドレスを個別に入力する代わりに、[共有可能なリンク](/jp/features/workspace#リンク経由で招待する) を使用して他のユーザーを Workspace に招待できるようになりました。招待リンクは、Free、Pro、Business プランで利用できます。

招待リンクはロールベースであり、5 日後に期限切れになります。ロールごとに一度に存在できるアクティブな招待リンクは 1 つだけであり、新しいリンクを作成すると、以前のリンクが自動的に置き換えられます。

Workspace 管理者と所有者は、[設定] → [ユーザー]* からいつでもリンクを再生成または削除できます。また、[設定] → [プライバシーとセキュリティ] → [リンクの招待]* からリンクを完全に無効にすることができます。

### Nano Banana 2 (Gemini 3.1 Flash イメージ) Support

[ZenOpus AI](/jp/integrations/ai) は、Google の Gemini 3.1 Flash Image モデルである Nano Banana 2 をサポートするようになりました。この特殊なモデルは、高速で高品質な画像の生成と編集を提供するため、プロジェクト内でより豊富なビジュアル機能を直接構築できます。

### ZenOpus Cloud の更新

[ZenOpus Cloud](/jp/integrations/cloud) により、可視性が向上し、ワークフローがよりスムーズになりました。

- [プロジェクトごとのクラウド使用量の内訳](/jp/integrations/cloud#使用法) が、クラウド セクションの *使用量* タブに表示されるようになりました。各プロジェクトがクラウド リソースをどのように消費するか、またそれがクラウド全体のバランスにどのように寄与するかについての割合の内訳を確認できます。
- [クラウド ツールはデフォルトで自動承認されるようになりました](/jp/integrations/cloud#zenopus-cloud-ツールの構成)。これにより、ビルド中の中断が軽減されます。ツールのアクションを実行前に確認したい場合は、設定で手動承認に戻すことができます。

### ダッシュボードの構成とナビゲーションの改善

これらの更新により、ダッシュボード全体でプロジェクトを整理および移動する方法が改善されます。

- 最近表示したプロジェクトは、ダッシュボード、サイドバー、検索全体で大幅に高速に読み込まれるようになりました。
- 新しい *最後に表示した* 並べ替えオプションを使用すると、最近アクセスしたプロジェクトにすぐに戻ることができます。
- ダッシュボードのサイドバーにある *Created by me* フィルターを使用すると、自分が作成したプロジェクトにすぐにアクセスできます。
- フォルダー検索に、ネストされたサブフォルダー内のプロジェクトが含まれるようになりました。
- フォルダーがまだ存在しない場合、*フォルダーへ移動* ダイアログから直接フォルダーを作成できるようになりました。
- *cmd+K* 検索モーダルでフォルダーの結果がより明確に表示されるようになり、プロジェクトやフォルダー間のキーボード ナビゲーションが向上しました。

### プロジェクト管理の最新情報

これらの更新により、ワークスペース間でプロジェクトを管理する方法が改善されます。

- プロジェクトをリミックスするときに、宛先 Workspace を選択できるようになりました。
- 後で名前を変更するのではなく、リミックス フロー中にプロジェクトに名前を付けることができるようになりました。
- ダッシュボードのプロジェクト カード ドロップダウン メニューからプロジェクトを直接転送できるようになりました。
- プロジェクト カードに共有ドロップダウンが含まれるようになり、プロジェクト リンクや公開アプリの URL をダッシュボードから直接コピーできるようになりました。
- エンタープライズ プランでは、Workspace 管理者と所有者は、Workspace 編集者が所有するプロジェクトを個人ワークスペースに転送することを許可できます。これにより、企業コホートの学生とチーム メンバーは、コースまたはプログラムが終了した後もプライベート プロジェクトにアクセスし続けることができます。 *[設定] → [プライバシーとセキュリティ] → [編集者にプロジェクトの転送を許可する] で見つけます。*

#### 画像生成の更新

- ZenOpus は、複雑な詳細、正確な画像内テキスト、リアルなアプリ UI モックアップを備えた高品質の画像を生成するようになりました。エージェントは、リクエストごとに最適なイメージ生成モデルを自動的に選択します。
- 背景が透明な画像をチャットから直接生成できるようになったので、生成されたアセットがデザインにシームレスに溶け込みます。

### 改善とバグ修正

- 新しく作成されたプロジェクトはすべて、自動的に全画面モードで開きます。これにより、最初からより焦点を絞った Workspace が提供され、一貫したプロジェクト作成エクスペリエンスが保証されます。
- ZenOpus はコードをより効率的にロードするようになりました。 ZenOpus は、最初にファイル ツリーをロードし、オンデマンドでファイルの内容をフェッチします。これにより、大規模なプロジェクトのパフォーマンスが大幅に向上します。
- コード エディターのサイドバーでは、ファイル名検索とコンテンツ検索が 1 つのエクスペリエンスに組み合わされるようになり、1 つのクエリで一致するファイル名と一致するコードを含むファイルの両方が表示されます。
- Shopify のオンボーディングに必要な手順が減り、セットアップ中に自動的に国が検出されるようになりました。
- Enterprise プランでは SSO のみのログインが強制されるようになり、他の認証方法によるサインインが防止されます。
- ZenOpus によってプッシュされたコミットには、共同作成者として GitHub ID が含まれるようになり、リポジトリ内の追跡可能性と監査履歴が向上します。
- *Plan mode* トグルがプロンプト ボックスにアイコン ボタンとして表示されるようになりました。これにより、他のプロジェクト コントロールと揃えられます。

### プロジェクト間の参照

![ZenOpus プロンプト ボックス内でプロジェクトを参照する例](/images/zenopus/changelog/2026-feb-23-cross-project.png)

ZenOpus は、[プロジェクト間参照](/jp/features/cross-project-referencing) を使用して、Workspace 内の他のプロジェクトの実装にアクセスして再利用できるようになりました。

チャットで `@` メンションを使用してプロジェクトを明示的に参照することも、必要に応じて ZenOpus に既存のプロジェクトを使用するように依頼することもできます。 ZenOpus は、ファイル構造、ソース コード、および関連するチャット履歴を読み取り、何かがどのように構築されたかを理解し、それを現在のプロジェクトで再作成または適応させます。参照されたプロジェクトは変更されません。

プロジェクトは *プロジェクト設定 → プロジェクト間共有* でオプトアウトでき、Workspace 管理者は *設定 → プライバシーとセキュリティ* でこの機能を無効にすることができます。

### Slack を共有コネクタとして利用可能

![ZenOpus 背景に Slack ロゴ](/images/zenopus/changelog/2026-feb-23-slack-connector.png)

[Slack](/jp/integrations/slack) は、アプリにメッセージングと通知を提供します。チャンネル更新を送信し、構造化されたアラートを投稿し、プロジェクトから直接メッセージを読み取ります。導入アラート、内部更新、Support 通知、またはリアルタイムでチームに情報を提供するのに最適です。

Workspace 管理者と所有者は、[設定] → [コネクタ] → [共有コネクタ]* で Slack に接続でき、Workspace 内のすべてのプロジェクトで利用できるようになります。きめ細かい権限制御を使用して、チーム全体で使用する共有ボット、または個人で使用する個人トークンを選択できます。

### パーソナルコネクタとしてグラノーラMCPが利用可能

[ZenOpus を Granola アカウントに接続](/jp/integrations/mcp-servers) できるようになり、会議メモや会話からエージェントにコンテキストを提供できるようになり、Granola で取り込まれた意思決定やディスカッションを反映するアプリや機能を生成できるようになります。

### Amplitude MCP をパーソナルコネクタとして利用可能

[ZenOpus を Amplitude アカウントに接続](/jp/integrations/mcp-servers) できるようになり、エージェントが製品分析データにアクセスできるようになり、実際の使用状況の分析情報や指標に基づいたアプリや機能の構築に役立ちます。

### ZenOpus Cloud の更新

[ZenOpus Cloud](/jp/features/cloud) 全体でいくつかの改善が行われました。

- [クラウド領域の選択](/jp/integrations/cloud#地域の選択)  

 クラウド プロジェクトをホストする場所をアメリカ、ヨーロッパ、アジア太平洋から選択できるようになりました。クラウド コネクタの設定で優先するデフォルト リージョンを設定すると、ZenOpus はそのリージョンを自動的に使用します。それ以外の場合、ZenOpus は現在の場所に最も近い地域を選択します。近くのリージョンを選択すると、レイテンシーと全体的なバックエンドのパフォーマンスが向上します。クラウドが有効になった後は、選択したリージョンを変更したり、リージョン間で既存のプロジェクトを移動したりすることはできません。
- **コード以外のデプロイ**シークレットの追加、ストレージ バケットの構成、環境設定の更新などのコード以外の変更は、コード変更を必要とせずにライブ環境に直接デプロイされるようになりました。これにより、プレースホルダー コミットを必要とせずに、環境の更新が [テスト環境とライブ環境](/jp/features/environments) 全体で確実に適用されます。
- **クラウド エッジ機能のエラーの減少**バックエンド エッジ機能のリクエストに関連するエージェント エラーが大幅に減少することで、クラウドの信頼性が向上しました。これにより、予期しない障害が軽減され、クラウド プロジェクトの全体的な安定性が向上します。
- **電子メールのレート制限**プロジェクトから 1 時間あたりに送信できる電子メールの数を構成できるようになりました。この制限を調整すると、配信ペースを制御し、クラウドの使用状況を管理するのに役立ちます。

### SCIMプロビジョニング

Workspace エンタープライズ プランの管理者と所有者は、[SCIM プロビジョニング](/jp/features/business/scim) を使用して、アイデンティティ プロバイダーからユーザーを自動的にプロビジョニングおよびプロビジョニング解除できるようになりました。

ID プロバイダー グループは、ビューア、編集者、管理者などの ZenOpus ロールにマップできるため、新しいユーザーは適切な権限を自動的に受け取ります。

### ガイド付き公開フロー

[出版](/jp/features/publish) では、ガイド付きの複数ステップのフローが使用されるようになりました。 Web サイトのアドレスを設定し、アクセス許可を構成し、タイトルと説明を追加し、ファビコンとソーシャル イメージをアップロードし、セキュリティを確認してから公開します。ファビコンが自動的にトリミングされ、正しいサイズと `.ico` 形式に変換されるようになりました。

### フォルダーの可視性コントロール

フォルダーの可視性を次のように設定できるようになりました。

- Workspace の可視性: すべての Workspace メンバーはプロジェクトを表示し、フォルダーにプロジェクトを追加できます。
- プライベート公開: 自分だけがプロジェクトを表示し、フォルダーにプロジェクトを追加できます。

可視性を設定できるのは、最上位のフォルダーのみです。ネストされたフォルダーは、親フォルダーの表示設定を継承します。

フォルダー内のプロジェクトはそのフォルダーの表示設定を継承し、それらの表示設定を個別に設定することはできなくなります。プロジェクトを移動すると、アクセス プレビューに誰がアクセス権を取得するのか、誰がアクセス権を失うのかが表示され、移動中に余分な共同作業者を削除できます。

すでにフォルダー内にあったプロジェクトは、新しい動作に移行されていません。フォルダーと一致するように手動で更新するまで、現在の表示設定が維持されます。

### 共有コネクタの管理コントロール

Business および Enterprise プランの Workspace 管理者と所有者は、特定の [共有コネクタ](/jp/integrations/introduction#共有コネクタ-公開アプリに機能を追加します。) を Workspace レベルで有効または無効にできるようになりました。 ZenOpus Cloud を無効にすることはできません。

これにより、チームは Workspace 全体でどのサードパーティ サービスを利用できるかを制御できるようになります。

### 外部コラボレーターの可視性

ZenOpus は、誰かが組織外にいるとき、その人を招待するときと、「共有」ダイアログやその他の関連領域でアクセス権を表示するときの両方で明確に示すようになりました。

これにより、チームは外部ユーザーと作業する際のガバナンスと認識を維持することができます。

### プロジェクト所有権の譲渡

「プロジェクト設定」からプロジェクトの所有権を別の Workspace メンバーに直接譲渡できるようになりました。これにより、元の所有者が Workspace を離れた場合、または組織内で所有権を変更する必要がある場合でも、プロジェクト管理がシームレスに維持されます。確認ダイアログには、転送の結果としてアクセス権が変更されるかどうかが明確に示されます。

### 改善点

- [Plan mode](/jp/features/plan-mode) エディターは、読み取りモードと編集モードを 1 つのエクスペリエンスに結合する、レンダリングされたリッチ テキスト ビューを使用するようになりました。

- ナレッジ ファイルの処理が強化されてスライドの生成が改善され、単一のプロンプトからより良いスライドショーの結果が生成されます。
- プロジェクト検索は、全文検索の高速化、部分文字列の一致と関連性のスコアリングの向上、予期しないリロードや結果の消失のない安定性の向上によって改善されました。
- Workspace メンバーシップを持たないユーザーには、新しい無料の Workspace を作成するか、参加する既存の__TERM__ を見つけるためのダイアログが表示されるようになり、Workspace へのアクセスを失った後でも ZenOpus を引き続き使用できるようになりました。
- ワークスペースやプロジェクトへの招待は、デフォルトで自動的に受け入れられるようになりました。この動作は、*設定 → アカウント* で変更できます。

### 削除されました

- カスタム ドメイン プロキシ オプションはカスタム ドメイン設定から削除されました。 Cloudflare、CloudFront、Fastly などの CDN またはリバース プロキシの背後でアプリを実行する場合は、[DNS またはプロキシ プロバイダーを使用して直接構成] (/jp/features/custom-domain#how-do-i-use-a-cdn-or-reverse-proxy-with-my-custom-domain) する必要があります。
