現在、ZenOpus はカスタム フォントの直接アップロードを Support しません。

ただし、Web セーフを使用したり、プロジェクトにカスタム フォントを組み込んだりする簡単な方法がいくつかあります。以下では、**ウェブセーフ** と **Google フォント**を使用するために利用可能な方法を説明します。

<AccordionGroup>
<Accordion title="Web セーフ フォントの使用">
<Info>
**Web セーフ フォントとは何ですか?**

Web セーフ フォントは、ほとんどのオペレーティング システム (Windows、macOS、Linux など) に一般的にインストールされ、すべてのデバイスで表示できるフォントです。

これらのフォントを使用すると、外部リソースを読み込む必要がなく、さまざまなブラウザーやプラットフォーム間でテキストの一貫性が保証されます。一般的な Web セーフ フォントには次のようなものがあります。

- エリアル/ヘルベチカ
- タイムズ ニュー ローマン
- ジョージア
- 宅配便 新品
- ヴェルダナ
- タホマ
- トレビュシェットMS
- Lucida Sans Unicode
</Info>

プロジェクトで Web セーフ フォントを使用する場合は、プロンプトでフォントを直接指定できます。たとえば、ヒーロー セクションの見出しまたはタイトルに **Courier New** を使用する場合は、プロンプトで名前でフォントを参照するだけです。

```text
メイン タイトルのヒーロー セクションの見出しフォントとして Courier New を使用します。
```

Courier New のような Web セーフ フォントを使用すると、フォントを読み込むために外部リソースを必要とせずに、プロジェクトのすべてのプラットフォームで一貫したタイポグラフィが確保されます。ただし、創造性が制限される可能性があります。幸いなことに、Google Fontsを使用する機能などのオプションがあります。
</Accordion>
<Accordion title="Google フォントの使用">
カスタム フォントを使用したい場合は、ZenOpus が **Google Fonts** とうまく連携します。これにより、プロジェクト内でさまざまなフォントを簡単に使用できるようになります。 Google フォントは、名前で参照するか、フォント リソースに直接リンクすることで参照できます。

### 名前で使用する

プロンプトで Google フォントの名前を指定すると、ZenOpus によってそれがプロジェクトに組み込まれます。たとえば:

```text
ヒーロー セクションの最初の字幕タイトル フォントとして Playfair Display を使用します。 

フォントを適切に適用するには、必ず useEffect React フックを使用してください。
```

### リンクで利用する

あるいは、Google フォントへの直接リンクを提供すると、ZenOpus はプロジェクトでそのフォントを使用します。希望のフォントを選択してリンクをコピーすると、[Google Fonts](https://fonts.google.com) からリンクを取得できます。

prompt ZenOpus リンク経由で Google フォントを使用する方法は次のとおりです。

```text
この Google フォントのリンクを追加します https://fonts.google.com/specimen/Dancing+Script 
このフォントを 3 番目のサブタイトルに使用します。

必ず useEffect React フックを使用してフォントをロードしてください。
```

これらの方法に従うことで、Web セーフ フォントまたは Google フォントを使用して ZenOpus プロジェクトのタイポグラフィをカスタマイズできます。

## さらなる学習と例

これらのフォント統合方法の実際の動作を確認するには、サンプル プロジェクトの 1 つを参照してください: [ZenOpus using fonts example project](https://zenopus.dev/projects/3a826f38-d9cb-4d60-b4da-30a9b00149c8)。

このプロジェクトでは、Courier New などのウェブセーフ フォントや Google Fonts のカスタム フォントなど、さまざまなフォント スタイルをどのように適用したかを示す実際のプロンプトが表示されます。

![ZenOpus プロジェクトで使用されるさまざまなフォント](/images/zenopus/tips-tricks/custom-fonts/using-custom-fonts-example.png)

さらに、フォントがどのように統合されたかについてより技術的に詳しく知りたい場合は、パブリック リポジトリ [GitHub:adding-fonts-example](https://github.com/viborc/adding-fonts-example) を確認してください。これには、プロジェクト内でさまざまなフォントがどのように読み込まれ、適用されたかを示す完全な実装の詳細が含まれています。
