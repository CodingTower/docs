## 再送信とは何ですか?

[再送信](https://resend.com/emails) は開発者向けに構築されたメール API です。これは、トランザクションおよびマーケティング電子メールを送信するための、高い配信可能性、統合の容易さ、クリーンな開発者エクスペリエンスに重点を置いています。

- **ZenOpus**: フロントエンドとバックエンドのロジックを生成します。

## 統合に関する FAQ を再送信する

<AccordionGroup>
<Accordion title="再送信とは何ですか?">
Resend は、開発者向けに構築された電子メール API です。これは、トランザクションおよびマーケティング電子メールを送信するための、高い配信可能性、統合の容易さ、クリーンな開発者エクスペリエンスに重点を置いています。
</Accordion>
<Accordion title="このチュートリアルを実行するには、有料の再送信プランが必要ですか?">
いいえ。**Resend の無料枠**を使用して CRM セットアップ全体を完了できます。この無料枠では、検証済みのドメインからのメールの送信と、マーケティング メール用に最大 3,000 件の連絡先へのメールの送信がサポートされます。
</Accordion>
<Accordion title="再送信でドメインを確認するにはどうすればよいですか?">
[再送信] → [**ドメイン**] → [ドメインを追加] に移動します。 DNS プロバイダー (Cloudflare、Namecheap など) 経由で追加する DNS レコード (TXT、MX) が提供されます。追加したら、[**確認**] をクリックします。

<Tip>
`updates.yourdomain.com` のようなサブドメインを使用して、電子メールの評判を分離します。
</Tip>
</Accordion>
<Accordion title="私の再送信メールがスパムになるのはなぜですか?">
よくある理由がいくつかあります。

- ドメインを認証していません
- あなたのメールにはプレーンテキストバージョンがありません
- `onboarding@resend.dev` のような一般的な送信者から送信している
- ベストプラクティスに従っていない（マーケティングメールに登録解除リンクがないなど）

Resend の **Deliverability Insights** パネルを使用して、実用的なヒントを入手してください。
</Accordion>
<Accordion title="送信するメールのデザインをカスタマイズできますか?">
はい。 Resend チームによるオープンソース ライブラリである [React Email](https://react.email/) を使用して、React コンポーネントを使用してスタイル付きの応答性の高いメールをデザインします。
</Accordion>
<Accordion title="Support マーケティング電子メールを再送信しますか、それともトランザクションのみですか?">
再送信は**両方**をサポートします:

- API または SDK を使用した **トランザクション電子メール** (確認、リセット、通知)。
- **ブロードキャスト**機能と**オーディエンス**を使用した**マーケティングメール**。

ダッシュボードからキャンペーンを作成、プレビュー、追跡できます。
</Accordion>
<Accordion title="再送信でニュースレター購読者を管理するにはどうすればよいですか?">
**対象者**を使用します:

- CSV 経由、または API を使用してプログラムで連絡先を手動で追加します。
- 各オーディエンスには、サブスクリプションを関連付けるために使用できる一意の ID があります。
- ユーザーは、Resend に組み込まれた購読解除リンクを介して自動的に購読を中止できます。
</Accordion>
<Accordion title="ニュースレターにダブル オプトインを設定できますか?">
デフォルトではありませんが、手動で実装できます。

1. フォーム送信時に、最初の確認メールを送信します。
1. ユーザーにリンクをクリックするよう依頼します。
1. クリックすると、サブスクリプションを確認し、再送信対象ユーザーに追加します。
</Accordion>
<Accordion title="再送信で電子メール フローをスケジュールまたは自動化できますか?">
まだです。現在、再送信では Support ドリップ キャンペーンや自動シーケンスは行われません。

<Tip>
回避策:

**Supabase Cron ジョブ**を使用して、設定された間隔で電子メールの再送信をトリガーするタスクをスケジュールします。
</Tip>
</Accordion>
<Accordion title="Resend を Supabase Auth に接続するにはどうすればよいですか?">
Resend による公式の Supabase 統合を使用します。

1. Supabase プロジェクトで認証します。
1. ドメインと送信者名を入力します。
1. Resend は、Resend アカウントを使用するように Supabase 認証メールを自動的に構成します。
</Accordion>
<Accordion title="再送信は購読解除をどのように処理しますか?">
再送信により購読解除が自動的に処理されます。

- ブロードキャストメールには購読解除リンクを含める必要があります。
- ユーザーが購読を解除すると、選択した対象ユーザーから削除されます。
- これは電子メールのコンプライアンス (CAN-SPAM、GDPR など) に必要です。
</Accordion>
</AccordionGroup>
