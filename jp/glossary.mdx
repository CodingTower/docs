## 一般的な概念

- **AI (人工知能):** 機械、特にコンピューター システムによる人間の知能プロセスのシミュレーション。学習、推論、問題解決などのタスクを可能にします。
- [**プロンプト**](/jp/prompting/prompting-one)**:** 出力の生成または特定のタスクの実行において AI モデルをガイドするテキストまたは入力。これは ZenOpus のコア機能の一部であり、アプリケーション内の特定のコンポーネントまたは要素を作成、変更するプロンプトとともに使用されます。

## ZenOpus 固有の用語

- [**Plan mode**](/jp/features/plan-mode)**:** ZenOpus はコードを生成するだけではありません。開発のあらゆる段階をガイドするインタラクティブなアシスタントになります。これは、創業者が批判的に考え、効果的に計画し、インテリジェントにデバッグし、自信を持って出荷するのに役立ちます。
- **編集モード**: コンテンツまたはコードに変更を加えるアクション。
- [**Edit**](/jp/features/design)**:** Tailwind ネイティブのビジュアル コントロールを使用して AI 主導の開発を可能にし、簡単に調整できるツールです。
- **履歴:** アプリとコードへの変更を時間の経過とともに追跡および管理するシステム。
- [**知識**](/jp/features/knowledge)**:** プロジェクトの重要な詳細を、アプリとともに進化する生きたドキュメントに記録します。
- [**Labs**](/jp/features/labs)**:** 新しい機能、革新的な機能、または開発中の機能のテストと紹介に特化した ZenOpus 内の実験機能。これらはいつでも変更または削除される可能性があります。
- **リミックス:** 既存の公開プロジェクトまたは独自のプロジェクトをリミックスできます。リミックスにより、プロジェクトの現在の状態を出発点として再利用し、それを基にして構築することができます。これは、元のバージョンを維持しながら、新しいアイデアを検討したり、調整を行ったり、さまざまな変更を繰り返したりするのに最適な方法です。 Supabase が接続されている場合、プロジェクトをリミックスすることはできないことに注意してください。
- **プレビュー:** ユーザーは、完成または公開される前に、ライブ コンテンツや機能をインタラクティブな方法で表示または体験できます。
- **Diff:** ファイルまたはコードの 2 つのバージョン間の違いを示す比較。
- **/index:** Web サイトまたはアプリのメイン ページまたはデフォルト ページを指し、多くの場合「index.html」または「index.js」という名前が付けられます。
- [**ZenOpus API - Build with URL**](/jp/integrations/build-with-url)**:** ZenOpus API Build with URL 機能を使用すると、URL パラメーターを介してプロンプトと画像を渡すことにより、プログラムでアプリケーションを作成できます。これにより、Web サイト、内部ツール、自動化ワークフローとのシームレスな統合が可能になり、ユーザーはワンクリックで ZenOpus アプリを生成できるようになります。

## 製品と開発のコンセプト

### 製品管理と戦略

- **MVP (Minimum Viable Product)** – 早期採用者を満足させ、本格的な開発前にアイデアを検証するのに十分な機能を備えた製品のバージョン。
- **ロードマップ** – 製品の長期的なビジョン、方向性、計画された機能の概要を示す高レベルの戦略計画。
- **機能リクエスト** – 製品の新機能に対するユーザーまたは関係者からの公式または非公式の提案。
- **ユーザー ストーリー** – エンド ユーザーの観点から書かれた機能または要件の短くて簡単な説明。
- **ユーザー ジャーニー:** 製品またはサービス内の特定の目標を達成するためにユーザーが実行する一連のステップ。
- **ペルソナ:** デザインと開発の決定を導くための、調査に基づいたターゲット ユーザーの架空の表現。

### エンジニアリングの概念

- **PRD (製品要件ドキュメント):** 開発の指針となる製品の目的、機能、仕様の概要を説明する包括的なドキュメント。
- [**API**](https://zenopus.dev/blog/2025-01-28-the-power-of-apis)**(アプリケーション プログラミング インターフェイス):** さまざまなサービスが通信する方法。これは、システム間でどのような種類の情報を取得または送信できるかを示すプロトコルと考えてください。最も一般的なタイプは、多くのサイトで使用されている REST API と、データをクエリするためのより柔軟なアプローチを提供する GraphQL API です。
- **リファクタリング:** 可読性、保守性、またはパフォーマンスを向上させるために、外部の動作を変更せずに既存のコードを再構築するプロセス。
- **GitHub:** Git を使用したバージョン管理と共同ソフトウェア開発のための Web ベースのプラットフォーム。

### データと分析

- **A/B テスト** – Web ページまたは機能の 2 つのバージョンを比較し、ユーザーの行動に基づいてどちらのパフォーマンスが優れているかを判断する方法。
- **コンバージョン率** – サインアップや購入など、目的のアクションを完了したユーザーの割合。
- **チャーンレート** – 一定期間内に製品の使用を中止したユーザーの割合。
- **継続率** – 長期間にわたって製品を使用し続けるユーザーの割合。
- **イベント追跡** – 特定のユーザーのアクション (ボタンのクリック、フォームの送信など) を監視して、行動とエンゲージメントに関する洞察を収集します。

## UI/UX およびフロントエンド開発プレイブック

### フロントエンド開発

- **フロントエンド:** ユーザーが直接操作するソフトウェア アプリケーションの部分。ユーザー インターフェイスとユーザー エクスペリエンスの要素が含まれます。
- **React:** ユーザー インターフェイス、特にシングルページ アプリケーションを構築するために Facebook によって開発された JavaScript ライブラリ。
- **グラデーション:** デザイン内の 2 つ以上の色または色合いの間の段階的な移行。
- **Tailwind CSS**: 事前定義されたクラスの包括的なセットを提供するオープンソースのユーティリティ ファースト CSS フレームワーク。開発者は、これらのユーティリティ クラスを要素に適用することで、HTML 内で直接カスタム デザインを構築できます。
- **デザイン システム:** 製品またはブランド全体でのデザインの一貫性と一貫性を確保する、再利用可能なコンポーネント、ガイドライン、標準のコレクション。

### UI/UXデザインのコンセプト

- **アクセントカラー:** デザイン内の重要な要素を強調するために使用される独特の色合い。多くの場合、ブランドの原色と調和したり補完したりして、視覚的な興味を高めます。
- **テーマ:** アプリケーションまたは Web サイトの全体的な美しさとユーザー エクスペリエンスを定義する、色、フォント、レイアウトを含む、まとまったデザインの選択肢のセット。
- **レスポンシブ デザイン:** コンテンツがさまざまなデバイス サイズや向きにシームレスに適応し、最適なユーザー エクスペリエンスを提供する Web デザインのアプローチ。
- **スクロールせずに見える部分:** スクロールせずにユーザーに表示される Web ページの部分。注目を集め、重要な情報をすぐに伝えるために重要です。
- **CTA (Call to Action):** 多くの場合、ボタンまたはリンクの形式で、「サインアップ」や「詳細を確認」などの特定のアクションを実行するようユーザーに促すプロンプト。

### ページ構造とナビゲーション

- **見出し (タイトル):** コンテンツ セクションを紹介および整理するテキスト要素。通常は、明確なコンテンツ階層を確立するためにさまざまなレベル (H1、H2 など) で書式設定されます。
- **フッター:** Web ページの下部セクション。通常、連絡先の詳細、ナビゲーション リンク、法的免責事項などの補足情報が含まれます。
- **パンくずリスト:** サイト階層内でのユーザーの現在位置を表示するナビゲーション補助であり、多くの場合、水平方向のリンク リストとして表されます。
- **ファビコン** – ブラウザーのタブ、ブックマーク、検索結果に表示される小さなアイコンで、通常はブランドまたは Web サイトを表します。
- **メタ タイトル (タイトル タグ)** – 検索エンジンの結果やブラウザのタブに表示される Web ページのタイトル。SEO とユーザーのクリックスルー率に影響を与えます。
- **メタ ディスクリプション** – 発見しやすさとユーザー エンゲージメントの向上を目的とした、検索エンジンの結果に表示される Web ページのコンテンツの短い概要。
- **正規 URL** – Web ページの優先バージョンを指定するために使用されるタグ。これは、検索エンジンのインデックス作成における重複コンテンツの問題を回避するのに役立ちます。
- **URL スラグ** – 読み取り可能な形式で特定のページを識別する URL の部分 (`example.com/product-name` など)。
- **サイトマップ** – Web サイトのページのリストを提供する構造化ファイル (XML または HTML)。これにより、検索エンジンが効率的にインデックスを作成できるようになります。
- **ナビゲーション バー (ナビゲーション バー)** – 簡単にナビゲーションできるように、Web サイトの重要なセクションへのリンクを提供する水平または垂直のメニュー。
- **リンクをスキップ** – ユーザーが繰り返しのコンテンツを回避してメイン コンテンツに直接移動できるようにする非表示のリンクにより、アクセシビリティが向上します。
- **ページネーション** – コンテンツを複数のページに分割する方法。ユーザー エクスペリエンスを向上させるために、ブログ、検索結果、製品リストで一般的に使用されます。
- **アンカー リンク** – 新しいページを読み込む代わりに、同じページ内の特定のセクションにユーザーを誘導するハイパーリンク。
- **404 ページ** – ユーザーが存在しない URL にアクセスしようとすると表示されるカスタム エラー ページで、関連するコンテンツに戻されるように誘導します。

### 通知とフィードバックの要素

- **トースト:** ユーザーにアクションの結果を通知したり、簡単なフィードバックを提供したりするために一時的に表示される、簡潔で非侵入的な通知。
- **スナックバー** – トーストに似ていますが、通常は画面の下部にあり、オプションのアクション (削除されたアイテムの「元に戻す」など) を備えています。
- **ツールヒント** – 要素の上にマウスを移動するか要素にフォーカスすると、追加情報が表示される小さなポップアップ ボックス。
- **バッジ** – 通知、カウント、またはステータスの更新を表示するために、アイコンやボタンでよく使用される小さな視覚的なインジケーター。
- **ローディング スピナー (ローダー)** – ページの読み込みやフォームの送信など、進行中のプロセスを表す視覚的なインジケーター。
- **進行状況バー** – タスクまたはプロセスの完了率を示す水平バー。
- **スケルトン ローダー** – ロードを待機している間に最終コンテンツのレイアウトを模倣するプレースホルダー UI で、体感的なパフォーマンスが向上します。

### オーバーレイとポップアップ

- **ポップオーバー:** 特定の UI 要素に関連する追加情報またはオプションを表示する一時的なオーバーレイ。通常はユーザー操作時に表示されます。
- **ダイアログ (モーダル):** ユーザーの注意を引くためにメイン コンテンツの前に表示されるウィンドウ。多くの場合、プライマリ インターフェイスに戻る前にアクションが必要です。
- **ドロワー (サイドバー パネル)** – 画面の横からスライドして、メイン コンテンツを中断することなくナビゲーション オプションや設定を表示するパネル。
- **ライトボックス** – 画像またはメディアを拡大表示し、焦点を合わせるために背景を暗くするモーダル ウィンドウ。
- **アラート ボックス** – エラーや警告などの重要な情報をユーザーに通知する、システム生成またはアプリケーションによってトリガーされるメッセージ ボックス。

### ナビゲーションと選択要素

- **ボタン:** フォームの送信やダイアログの開始など、アクションまたはイベントを開始するクリック可能な要素。
- **スイッチ (トグル):** ユーザーが 2 つの状態 (オンまたはオフなど) を切り替えることができるトグル コントロールで、多くの場合物理スイッチに似ています。
- **タブ** – コンテンツを 1 つのインターフェイス内の個別のビューに編成するコンポーネント。これにより、ユーザーは別の場所に移動することなくセクション間を切り替えることができます。
- **ステッパー (ウィザード)** – 複数ステップのプロセスに使用されるコンポーネント。ユーザーに現在のステップを表示し、完了までガイドします。
- **ページネーション** – コンテンツをページに分割する UI パターン。検索結果やコンテンツの多いアプリケーションでよく使用されます。
- **ブレッドクラム** – サイト階層内のユーザーの現在位置を表示するナビゲーション補助であり、多くの場合、水平方向のリンク リストとして表されます。
- **アコーディオン** – 拡張または縮小してコンテンツを動的に表示または非表示にする、折りたたみ可能なセクション。
- **ドロップダウン メニュー** – ユーザーがボタンまたはフィールドをクリックするか、ボタンまたはフィールドの上にマウスを移動すると表示されるオプションのリスト。ナビゲーションや選択によく使用されます。

### フォームと入力要素

- **フォーム:** ユーザーが連絡先情報や検索クエリなどのデータを送信できるようにする入力フィールドの構造化されたコレクション。
- **ラジオ グループ:** ユーザーが複数の選択肢から 1 つのオプションだけを選択できるようにする一連の関連ラジオ ボタン。
- **チェックボックス:** ユーザーがチェックを入れるかチェックを外してオプションを選択または選択解除できる対話型ボックス。セット内で複数の選択が可能です。
- **テキスト フィールド (入力フィールド)** – ユーザーが検索バー、ログイン フォーム、コメントなどにテキストを入力できる基本的な UI 要素です。
- **テキスト領域** – 複数行のテキスト入力用に設計された大きな入力フィールドで、メッセージやフィードバック フォームによく使用されます。
- **選択 (ドロップダウン選択ボックス)** – ユーザーが事前定義されたリストからオプションを 1 つ選択できるようにする UI 要素で、多くの場合ドロップダウンとしてスタイル設定されます。
- **日付ピッカー** – ユーザーが手動で日付を入力する代わりにカレンダーから日付を選択できるようにする UI 要素。
- **スライダー (範囲セレクター)** – ユーザーがトラックに沿ってハンドルをスライドさせることにより、範囲内の値を設定できるコントロール。
- **ファイル アップロード フィールド** – ユーザーがデバイスからファイルを選択してアップロードできるコンポーネントです。
- **オートコンプリート (先行入力検索)** – ユーザーの入力時に一致する結果を動的に提案するテキスト入力で、検索の使いやすさが向上します。

## バックエンド開発とデータベース

### バックエンドの基礎

- **バックエンド:** データ処理、ストレージ、ビジネス ロジックを処理するアプリケーションのサーバー側インフラストラクチャ。
- [**Supabase**](/jp/integrations/supabase)**:** Postgres データベース、認証、インスタント API、およびリアルタイム機能を提供する、オープンソースのサービスとしてのバックエンド プラットフォーム。
- **Webhook** – 特定のアクションが発生したときに外部アプリケーションに通知する、イベント駆動型の HTTP コールバック。

### データベース管理とクエリ

- **CRUD (作成、読み取り、更新、削除):** データベースまたはアプリケーション内のデータに対して実行される 4 つの基本操作。
- [**SQL (構造化クエリ言語)**](https://supabase.com/docs/guides/database/overview)**:** リレーショナル データベースを管理および操作するために設計されたドメイン固有のプログラミング言語で、データのクエリ、更新、整理などのタスクを実行できます。
- **PostgreSQL** – スケーラビリティ、拡張性、ACID 準拠で知られる強力なオープンソース リレーショナル データベース。

### Superbase 固有の概念

- [**エッジ関数**](https://supabase.com/docs/guides/functions)**:** サーバー側の TypeScript 関数。ユーザーに近いエッジでグローバルに分散されます。これらは、Webhook をリッスンしたり、Supabase プロジェクトを Stripe、Anthropic、Resend などのサードパーティと統合したりするために使用できます。
- [**RLS (行レベル セキュリティ)**](https://supabase.com/docs/guides/database/postgres/row-level-security)**:** ユーザーのロールまたは属性に基づいてテーブル内の特定の行へのアクセスを制御できるデータベースの機能。基本的にユーザーは詳細な認可ルールを設定できるため、独自のビジネス ニーズに合わせた複雑な SQL ルールを作成できます。
- [**ストレージ**](https://supabase.com/docs/guides/storage)**:** 画像、ビデオ、ドキュメント、その他のファイル タイプなどのデータを保存および管理するためのサービスまたはシステム。
- [**バケット**](https://supabase.com/docs/guides/storage/buckets/fundamentals)**:** Supabase Storage では、バケットはファイルとフォルダーを整理する個別のコンテナーであり、アクセス モデル (パブリックまたはプライベート) を決定し、最大ファイル サイズや許可されるコンテンツ タイプなどのアップロード制限を設定します。
- [**エンドポイント**](https://supabase.com/docs/guides/api)**:** Supabase のコンテキストでは、エンドポイントは、クライアントが RESTful API 呼び出しを通じてデータベースと対話できるようにする特定の URL を指し、データの作成、読み取り、更新、削除などの操作を可能にします。
- [**認証プロバイダ:**](https://supabase.com/docs/guides/auth) ユーザーの資格情報を検証し、アプリケーションまたはシステムへの安全なアクセスを可能にするエンティティまたはサービス。 Supabase のコンテキストでは、認証プロバイダーには、パスワード ベースのログイン、マジック リンク、ワンタイム パスワード (OTP)、ソーシャル ログイン、シングル サインオン (SSO) 統合などのさまざまな方法が含まれており、柔軟で安全なユーザー認証を促進します。

### セキュリティと認証

- **OAuth** – ユーザーが Google、Facebook、GitHub などのサードパーティ サービスを使用してアプリケーションにログインできるようにする、広く使用されている認証プロトコル。
- **2 要素認証 (2FA)** – ユーザーがアカウントにアクセスする前に 2 つの形式の ID を提供することを要求するセキュリティ対策。
- **CORS (Cross-Origin Resource Sharing)** – どの Web サイトまたはアプリケーションがサーバー上のリソースにアクセスできるかを制御するセキュリティ ポリシー。
