## ZenOpus プラットフォームの概要

<AccordionGroup>
<Accordion title="最初のプロジェクトを作成する">
<Steps>
<Step title="ステップ1">
まず、[zenopus.dev](https://zenopus.dev) にアクセスしてアカウントを作成してください。 登録すると、最初のプロジェクトを作成できるようになります。
</Step>
<Step title="ステップ2">
最初のプロンプトを入力するだけで作業が開始され、残りは ZenOpus が実行します。これは ZenOpus のあらゆるプロジェクトの**開始ブロック**であり、ここでアイデアを即座に実現できます。
</Step>
</Steps>
</Accordion>
<Accordion title="ZenOpus ダッシュボード">
ZenOpus では、シンプルさが重要です。

- [**こちら**](https://zenopus.dev/)**単一のプロンプトでプロジェクトを開始できます**。
- **最新のプロジェクトや注目のプロジェクトを含むすべてのプロジェクトを探索します**。
- **私たちが厳選したテンプレートからすぐに始めましょう**。
- **プロフィールを表示および編集して、エクスペリエンスをカスタマイズ**します。
- [**アカウントを管理**](https://zenopus.dev/settings) して、毎月のクレジットを確認し、設定を調整します。
- [**アップグレードまたはダウングレード**](https://zenopus.dev/settings/plans)**プラン**をシームレスに実行します。
</Accordion>
<Accordion title="プロジェクト概要">
ZenOpus では、すべてのプロジェクトが構造化されたワークフローに従います。

- **チャットベースのインターフェイスを使用して簡単に編集**。
- **画像を添付**すると、正確な編集とインスピレーションが得られます。
- **コンポーネントを選択して視覚的に編集**し、シームレスな設計体験を実現します。
- **簡単に** Agent mode と Plan mode を切り替えます。
- **バージョン履歴で変更を追跡**し、以前のバージョンに戻します。
- プロジェクトの **ページ間を簡単に移動**できます。
- **リミックスを有効にする** か、必要に応じてプロジェクトの名前を変更します。
- **バックエンド機能には ZenOpus Cloud** を使用します (または Supabase と統合します)。
- **GitHub 経由で共同作業**して、コードを編集および管理します。
- プレビュー リンクを使用してプロジェクトを **公開、デプロイ、共有**します (変更後は必ず再公開してください)。
- **レスポンシブ デザインの場合は、Web ビューとモバイル ビューを切り替えます**。
- **プレビュー モーダルを更新**すると、更新内容がすぐに表示されます。
- **プレビュー リンクを使用してプロジェクトを直接開きます**。
</Accordion>
<Accordion title="プロジェクト設定">
すべてのプロジェクトの設定により、次のことが可能になります。

- **合計編集内容や作成日など、主要なプロジェクトの詳細を表示**します。
- **カスタム ナレッジを設定**して、プロジェクトのコンテキストを調整します。
- **GitHub リポジトリに接続**してシームレスなコラボレーションを実現します。
- **必要に応じてプロジェクトの公開設定を調整します**。
- **バッジ表示を管理** (有料プランでのみ利用可能)。
- **プロジェクトの名前をいつでも変更**できます。
- **プロジェクトを削除** (⚠️ 危険ゾーン)。
</Accordion>
<Accordion title="プロジェクトへのクイックアクセス">
いつでもすぐにアクセスできます。

- **ダッシュボード**: プロジェクトの概要を表示します。
- **アカウント設定**: プロフィールと設定を管理します。
- **ドキュメントと Support** のガイダンスと支援。
- ZenOpus アカウントから安全に**ログアウト**します。
</Accordion>
</AccordionGroup>

## プロジェクトを開始する

ZenOpus の使用を開始するには、好みやリソースに応じていくつかの方法があります。

<AccordionGroup>
<Accordion title="プロンプト">
ZenOpus のプロンプトベースのシステムにより、アプリの作成が簡単になります。

- プロンプト ボックスに何を構築するかを説明するだけです。
- より具体的であればあるほど、より良い結果が得られます。
- 明確かつ詳細なプロンプトから始めます。
- プロジェクトを進めながら、改良や調整を行うことができます。

<Info>
例:

*「ユーザーのログイン情報、折れ線グラフの月次売上高、円グラフの顧客層を含むダッシュボードを作成します。」*
</Info>
</Accordion>
<Accordion title="既存のプロジェクトをリミックスする">
既存の公開プロジェクトまたは独自のプロジェクトをリミックスできます。リミックスにより、プロジェクトの現在の状態を出発点として再利用し、それを基にして構築することができます。これは、元のバージョンを維持しながら、新しいアイデアを検討したり、調整を行ったり、さまざまな変更を繰り返したりするのに最適な方法です。

<Note>
プロジェクトが Supabase に接続されていない場合、他のユーザーのパブリック プロジェクトのみをリミックスできます。
</Note>
</Accordion>
<Accordion title="Figma の使用">
Figma にデザインがある場合は、その一部のスクリーンショットを撮ります。スクリーンショットを ZenOpus に直接貼り付けることも、画像ファイルをドラッグ アンド ドロップすることもできます。 Enter キーを押すと、ZenOpus が設計を機能コードに変換します。
</Accordion>
<Accordion title="スケッチの使用">
<Steps>
<Step title="ステップ1">
[**Excalidraw**](https://excalidraw.com/) または同様のツールを使用して、UI をスケッチします。
</Step>
<Step title="ステップ2">
図面のスクリーンショットを撮り、ZenOpus に貼り付けるかドラッグ アンド ドロップします。 プラットフォームはスケッチを実用的なコードに変換します。
</Step>
</Steps>

[詳細はこちら](https://www.youtube.com/watch?v=c0zhLzcVJRI)
</Accordion>
<Accordion title="Web サイトまたはアプリケーションの複製">
既存の Web ページを複製する場合:

<Steps>
<Step title="ステップ1">
Mac の *Cmd+Shift+4* などのショートカット、または [GoFullPage Chrome Extension](https://chromewebstore.google.com/detail/gofullpage-full-page-scre/fdpohaocaechififmbbbbbknoalclacl) などのツールを使用してスクリーンショットを撮ります。
</Step>
<Step title="ステップ2">
スクリーンショットを ZenOpus に貼り付けるかドラッグします。 ZenOpus は、プロジェクト内の Web ページの構造を再作成します。
</Step>
</Steps>

[詳細はこちら](https://www.youtube.com/watch?v=tYDqBMilHkM)
</Accordion>
</AccordionGroup>

## プロジェクトを編集する

<AccordionGroup>
<Accordion title="カスタムナレッジをプロジェクトに追加する">
ZenOpus の [カスタム ナレッジ](/jp/features/knowledge) はプロジェクトの **青写真** として機能し、機能、デザイン、目標を 1 か所にまとめます。
</Accordion>
<Accordion title="ビジュアル編集を使用する">
ビジュアル編集を使用すると、ページ上の要素を選択して、テキストや色の更新などを即座に編集したり、プロンプトを使用して機能やレイアウトを調整したりできます。

詳細については、[ビジュアル編集](/jp/features/visual-edit) をご覧ください。
</Accordion>
<Accordion title="変更を元に戻すか、メッセージを編集して元に戻して別のアプローチを使用する">
AI 駆動のアプリを構築する場合は速度が重要ですが、安定性と制御も重要です。そのため、バージョン管理をアップグレードして、変更を簡単に追跡できるようにしました。

<Steps>
<Step title="過去のバージョンに即座に戻すか、別のアプローチを使用する">
古い作業バージョンをプレビューしてから、必要に応じてそのバージョンに戻すことも、過去のメッセージを編集して元に戻して別のアプローチを検討することもできます。何も失われることはありません。それ以降に行われたすべての変更はチャットに残り、いつでも再適用できます。
</Step>
<Step title="ブックマークキーの編集">
重要な編集をブックマークして作業を整理し、後で簡単に見つけて再確認できるようにします。
</Step>
<Step title="直感的なバージョン履歴">
履歴パネルが Google ドキュメントのように機能するようになり、変更の追跡が簡単になりました。
</Step>
</Steps>
</Accordion>

## フルスタック機能をプロジェクトに追加する

<AccordionGroup>
<Accordion title="バックエンドを接続する">
アプリケーションにバックエンド機能を追加する必要がある場合は、次のことができます。

- ZenOpus の組み込みバックエンドに接続します - [ZenOpus Cloud](/jp/features/cloud)
- ZenOpus のネイティブ [Supabase 統合](/jp/integrations/supabase) を使用する
</Accordion>
<Accordion title="Stripe を使用してアプリケーションに支払い機能を追加する">
Stripe を ZenOpus に接続する方法はたくさんありますが、最も簡単な方法は、Stripe に組み込まれている支払いリンクを利用することです。詳細については、「ストライプ統合」(/jp/integrations/stripe) を参照してください。
</Accordion>
