## はじめに

<AccordionGroup>
<Accordion title="ZenOpus とは何ですか?">
ZenOpus は、あらゆるスキル レベルのユーザーが自然言語を通じてフルスタックの Web サイトを作成できる AI 搭載プラットフォームです。必要なものを説明するだけで、ZenOpus がそれを構築します。
</Accordion>
<Accordion title="ZenOpus を使用して何を構築できますか?">
想像できれば、ZenOpus を使用して構築できます。シンプルなランディング ページからニッチなユースケースに合わせた強力なツールまで、ZenOpus を使用すると、必要なものを無制限に作成できる柔軟性が得られます。
</Accordion>
<Accordion title="ZenOpus を使用して構築するにはコーディングの経験が必要ですか?">
いいえ。ZenOpus はあらゆるレベルの経験を対象に設計されています。開発者でも技術者以外のユーザーでも、ZenOpus を使用して Web サイトを作成できます。
</Accordion>
<Accordion title="プロジェクトを作成するにはどうすればよいですか?">
プロジェクトを作成するには、ダッシュボードのメッセージ ボックスにメッセージを入力するだけです。何を構築したいかを記述するだけで、ZenOpus がそこからそれを取得して、プロジェクトの開始点を生成します。小さなことから始めて、少しずつ増やしていくことを忘れないでください。
</Accordion>
<Accordion title="既存のプロジェクトを再利用できますか?">
はい、既存の公開プロジェクトまたは独自のプロジェクトをリミックスできます。リミックスにより、プロジェクトの現在の状態を出発点として再利用し、それを基にして構築することができます。これは、元のバージョンを維持しながら、新しいアイデアを検討したり、調整を行ったり、さまざまな変更を繰り返したりするのに最適な方法です。
</Accordion>
</AccordionGroup>

## ZenOpus を使用して構築する

<AccordionGroup>
<Accordion title="ZenOpus を使用して効率的にビルドするにはどうすればよいですか?">
ZenOpus を使用して効率的に構築するには、適切なテクニックを使用することが重要です。少し練習すれば誰でもできるようになります。開始するには、[ガイド](https://mastering-zenopus.zenopus.app/) をご覧ください。
</Accordion>
<Accordion title="最良の結果を得るには、プロンプトをどのように構成すればよいですか?">
- 明確かつ具体的にしてください。
- 一度に 1 つの機能に焦点を当てます。
- 段階的に構築します。
- デザインを正しくするために流行語を使用します。
</Accordion>
<Accordion title="エラーが発生した場合はどうすればよいですか?">
<Steps>
<Step title="「修正を試みる」ボタン">
**修正を試す** オプションを使用します。 ZenOpus は問題を自動的に解決しようとします。ボタンを使用してもクレジットは減りません。
</Step>
<Step title="調査する">
**修正を試みる**でも問題が解決しない場合は、ZenOpus エージェントに問題の調査を依頼してください。何が表示されているのか、どの特定の部分が機能していないのかを説明してください。
</Step>
<Step title="古いバージョンに戻す">
それが機能しない場合は、古い作業バージョンをプレビューして、必要に応じてそのバージョンに戻すか、過去のメッセージを戻して編集して別のアプローチを検討できます。何も失われることはありません。それ以降に行われたすべての変更はチャットに残り、いつでも再適用できます。
</Step>
</Steps>
</Accordion>
<Accordion title="リファクタリングとは何ですか?なぜ重要なのでしょうか?">
リファクタリングにより、機能を変更することなくコードの構造と効率が向上します。役に立ちます:

- 冗長性を削除してパフォーマンスを最適化します。
- 将来のアップデートに備えて保守性を向上させます。
- ロジックをより明確に構造化することで競合を防ぎます。
</Accordion>
<Accordion title="プロジェクトをバックエンドに接続してデータを保存できますか?">
アプリケーションにバックエンド機能を追加する必要がある場合は、次のことができます。

- ZenOpus の組み込みバックエンドに接続します - [ZenOpus Cloud](/jp/features/cloud)
- ZenOpus のネイティブ [Supabase 統合](/jp/integrations/supabase) を使用する
</Accordion>
<Accordion title="自分の Web サイトにログインを追加できますか?">
はい、[ZenOpus Cloud](/jp/features/cloud) を使用するか、[Supabase](/jp/integrations/supabase) と統合することで認証を追加できます。 AI に追加するよう促し、指示に従うだけです。
</Accordion>
<Accordion title="自分のプロジェクトにサードパーティ API を追加できますか?">
はい、プロンプトで指定することで、OpenAI、Stripe などの API を統合できます。詳細は[こちら](/jp/integrations/introduction)をお読みください。
</Accordion>
<Accordion title="機密の API キーを ZenOpus に保存できますか?">
いや！ API キーを ZenOpus に直接入力しないでください。 API と統合する場合は、[ZenOpus Cloud](/jp/features/cloud) または [Supabase](/jp/integrations/supabase) のシークレットを使用して、Edge 関数と組み合わせて API キーを保存することをおすすめします。
</Accordion>
<Accordion title="自分のウェブサイトに支払いを追加するにはどうすればよいですか?">
Stripe をプロジェクトに追加することで、支払いを追加できます。設定方法については、[ストライプ統合](/jp/integrations/stripe) を参照してください。
</Accordion>
<Accordion title="自分のプロジェクトの SEO を改善するにはどうすればよいですか?">
[SEO と GEO のベスト プラクティスを実装する](/jp/tips-tricks/seo-geo) ガイドに従って、最良の結果を確保してください。
</Accordion>
<Accordion title="プロジェクトが完了するまでにどれくらい時間がかかりますか?">
それは複雑さによって異なります。タスクを小さなステップに分割し、事前に計画を立て、フィードバックに基づいて調整することで、品質を確保しながら迅速に行動できます。
</Accordion>
</AccordionGroup>

## 特長

<AccordionGroup>
<Accordion title="プロジェクト設定を見つけるにはどうすればよいですか?">
**プロジェクト設定**を見つけるには、エディターの左上にあるプロジェクト名をクリックし、**設定**を選択します。
</Accordion>
<Accordion title="プロジェクトの可視性を変更できますか?">
はい、[**プロジェクト設定**] に移動して、可視性を変更できます。詳細については、「プロジェクトの可視性の制御」(/jp/features/private-projects) を参照してください。
</Accordion>
<Accordion title="プライベート プロジェクトへのアクセスをリクエストできますか?">
はい。プライベート プロジェクトに移動するときは、プロジェクト所有者に閲覧者または編集者のアクセス権をリクエストできます。これにより、共同作業が容易になりながら、プライベートな作業が安全に保たれます。
</Accordion>
<Accordion title="ZenOpus バッジを非表示にすることはできますか?">
はい。**プロジェクト設定**に移動すると、バッジを非表示にするオプションがあります。有料ユーザーのみがバッジをオフに切り替えることができます。
</Accordion>
<Accordion title="プロジェクトの名前を変更できますか?">
はい。**プロジェクト設定**に移動すると、プロジェクトの名前を変更するオプションがあります。
</Accordion>
<Accordion title="プロジェクトをコピー（リミックス）するにはどうすればよいですか?">
プロジェクトのコピーを作成するには、**リミックス**します。 **プロジェクト設定** に移動し、**リミックス** オプションを選択します。

リミックスすると、現在選択されている Workspace にプロジェクトの正確なコピーが作成されます。その後、独立して編集できます。

<Note>
プロジェクトが Supabase に接続されていない場合、他のユーザーのパブリック プロジェクトのみをリミックスできます。
</Note>
</Accordion>
<Accordion title="プロジェクトを削除できますか?">
はい。**プロジェクト設定**に移動すると、プロジェクトを**削除**するオプションがあります。

<Note>
これは最終的なものであり、削除されたプロジェクトを復元する方法はありません。
</Note>
</Accordion>
<Accordion title="Plan mode とは何ですか?">
Plan mode (以前のチャット モード) は、コードを作成または変更する前に機能を計画し、問題をデバッグし、変更を理解するのに最適です。詳細については、[Plan mode を使用したブレインストーミング](/jp/features/plan-mode) を参照してください。
</Accordion>
<Accordion title="ZenOpus が生成するコードを確認して手動で編集できますか?">
はい。コードを表示し、ZenOpus の [Code mode](/jp/features/code-mode) を使用してコードを手動で編集できます。
</Accordion>
<Accordion title="テキストや色を手動で編集できますか?">
はい、[ビジュアル編集](/jp/features/visual-edit) を使用してテキストや色を編集できます。
</Accordion>
<Accordion title="自分のプロジェクトで共同作業できますか?">
はい、ZenOpus 内のプロジェクトで共同作業できます。詳細については、[コラボレーション](/jp/features/collaboration) を参照してください。
</Accordion>
<Accordion title="プロジェクトを公開するにはどうすればよいですか?">
[公開](/jp/features/publish) を参照し、そこに記載されている手順に従ってプロジェクトを公開します。

<Accordion title="ZenOpus プロンプトに画像を追加できますか?">

<Accordion title="プロジェクトを以前のバージョンに戻すことはできますか?">

- チャットの各応答の下、または元に戻したいバージョンを選択するときに履歴タブで、元に戻すオプションを見つけます。
- チャットで送信した各過去のメッセージの下で、過去のメッセージを編集し、元に戻すオプションを見つけます。
</Accordion>
</AccordionGroup>

## ZenOpus API

<Accordion title="ZenOpus API では何ができますか?">
**ZenOpus API** を使用すると、ZenOpus アプリをプログラムで作成および共有できるようになり、アイデア、プロンプト、画像さえも 1 つのリンクで完全に生成されたアプリケーションに変換できます。

最初のリリースである **Build with URL** では、共有可能な URL から直接アプリを生成できます。 これを使用すると、次のことが可能になります。

- Web サイトまたはアプリケーションに **Build with ZenOpus** ボタンを埋め込みます。
- 事前定義されたプロンプトを URL 経由で友人、同僚、またはソーシャル メディアと共有します。
- 内部ワークフローの一部としてアプリの生成を自動化します。
- プロンプトと参照画像を含む URL を使用して、アプリのプロトタイプを迅速に作成します。

この機能と実際の使用例については、お知らせ投稿: [ZenOpus Build with URL](https://zenopus.dev/blog/introducing-zenopus-api-build-with-url) を使用したリンクからアプリを構築するをご覧ください。

ここで完全な技術ガイドをお読みください: [ZenOpus API: URL ドキュメントを使用して構築する](/jp/integrations/build-with-url)。
</Accordion>

## アカウントの管理

<AccordionGroup>
<Accordion title="ログインに使用するメールアドレスを変更できますか?">
残念ながら、現時点ではメール アドレスを直接変更することはできません。回避策として、次の手順に従って、希望の電子メール アドレスで新しいアカウントを作成し、プロジェクトを転送できます。

1. 既存のプロジェクトを一時的に公開し、URL をコピーします。
1. 新しいアカウントにログインした状態で URL にアクセスします。
1. プロジェクトを新しいアカウントにリミックスします。 Supabase に接続されているプロジェクトは、リミックスする前に Supabase から切断する必要があることに注意してください。
1. プロジェクトを転送したら、必要に応じて再度非公開にすることができます。

注: ターゲット Workspace が有料プランの場合、アクセスするにはパブリック プロジェクトのリミックスが必要になる場合があります。新しい Workspace にもサブスクリプション レベルの機能が備わっていない限り、リミックスされたプロジェクトはサブスクリプション レベルの機能を引き継ぎません。
</Accordion>
<Accordion title="アカウントを削除できますか?">
アカウントを削除するには、**設定 → アカウント** に移動し、下にスクロールして **アカウントを削除** を押します。これにより、アカウントと関連するすべてのデータが完全に削除されます。
</Accordion>
<Accordion title="削除したアカウントを復活させることはできますか?">
いいえ、削除されたアカウントを再度アクティブにすることはできません。ただし、アカウントを削除するボタンを押した後、48 時間以内にこのアクションをキャンセルして削除を停止することができます。
</Accordion>
<Accordion title="Workspace の所有者を変更できますか?">
はい、所有者は複数の Workspace 所有者を追加できます。
</Accordion>
<Accordion title="プロジェクトの所有権を他の人に譲渡するにはどうすればよいですか?">
これを回避する 1 つの方法は、プロジェクトをリミックスすることです。プロジェクトをリミックスした人がそのバージョンの所有者になります。
</Accordion>
<Accordion title="2 要素認証 (2FA) を有効にするにはどうすればよいですか?">
**2 要素認証 (2FA)** は、通常のサインイン方法に加えて、サインイン時にワンタイム コードを要求することで、ZenOpus アカウントに追加のセキュリティ層を追加します。

### 2FA のセットアップ

1. **[設定] → [アカウント] → [アカウント]** に移動し、**2 要素認証**を有効にします。
1. 画面上の指示に従って、検証方法を選択して設定します。
    - **認証アプリ (推奨)**  
    Google Authenticator、Microsoft Authenticator、Authy、1Password などのアプリを使用して、時間ベースのワンタイム コードを生成します。
    - **電話 (SMS)**  
    6 桁の確認コードをテキスト メッセージで受け取ります。
1. [**確認して有効にする**] をクリックします。

#### 2FA でサインインする

サインインするとき:

1. 通常のサインイン方法を完了します。
1. 確認方法を選択します。
1. 認証アプリまたは SMS から 6 桁のコードを入力します。

#### 2FA の管理

- 2FA メソッドはいつでもアカウント設定に追加または削除できます。
- ロックアウトを避けるために、**少なくとも 2 つのメソッド**を構成しておくことをお勧めします。
</Accordion>
<Accordion title="バイブ コーディング レベルとは何ですか? LinkedIn アカウントにレベルを追加するにはどうすればよいですか?">
Vibe コーディング レベルは、ビルド中に送信するメッセージの数に基づいて、ZenOpus をどの程度積極的に使用しているかを反映します。 ZenOpus をさらに使用すると、レベルが自動的に上がります。

レベルは次のとおりです。

- **ブロンズ**: ZenOpus の使用を開始する
- **シルバー**: 通常の ZenOpus ユーザー
- **ゴールド**: アクティブな ZenOpus ビルダー
- **プラチナ**: 熱心な ZenOpus ビルダー
- **ダイヤモンド**: 最上位の ZenOpus エンゲージメント

**[設定] → [アカウント]** でレベルを確認できます。必要に応じて、LinkedIn アカウントに接続して、**ライセンスと認定** に現在のレベルを表示できます。これは完全にオプションであり、いつでも切断できます。

詳細については、[Vibe コーディング LinkedIn 認定資格](/jp/tips-tricks/linkedin-certification) (ベータ版) を参照してください。
</Accordion>
</AccordionGroup>

## ZenOpus の仕組み

<AccordionGroup>
<Accordion title="ZenOpus はどのような技術スタックを知っていますか?">
ZenOpus は、React、Tailwind、Vite を使用してフロントエンドを構築し、OpenAPI バックエンドに接続できます。データの永続性と認証のための Support は、ZenOpus Cloud または Supabase を使用して利用できます。
</Accordion>
<Accordion title="ZenOpus Support はモバイル アプリの開発を行いますか?">
いいえ、ZenOpus は **Web アプリケーション**に重点を置いていますが、モバイル フレンドリーな Web アプリケーションを設計することもできます。
</Accordion>
<Accordion title="ZenOpus はどのようにしてコンテキストを記憶するのでしょうか?">
ZenOpus は、コンテキストを維持するために最近のメッセージを処理します。最良の結果を得るには、各プロンプトに関連する詳細を含めて、明確さと正確さを確保してください。
</Accordion>

</Accordion>
