## ZenOpus チュートリアル

<AccordionGroup>
<Accordion title="AI を活用したカロリー追跡 Web アプリを構築する">
<iframe src="https://www.youtube.com/embed/c0zhLzcVJRI?si=7vcZf5eoxIobOxdE" title="YouTubeビデオプレーヤー" width="100%" height="315" frameBorder="0" allow="accelerometer; autoplay; clipboard-write; encrypted-media; gyroscope; picture-in-picture; web-share" referrerPolicy="strict-origin-when-cross-origin" allowFullScreen />

このチュートリアルでは、ZenOpus を使用して AI を活用したカロリー追跡アプリを作成し、認証、美しいレスポンシブ デザイン、OpenAI の統合、運用環境への展開を完了する方法を説明します。

<Steps>
<Step title="財団の設立">
ZenOpus プロジェクトの基礎は適切なプロンプトです。このアプリの例は次のとおりです。

```text
次の機能を備えたカロリー追跡アプリを作成します。
ランディング ページ、ダッシュボード、食事ページ。
GPT-4 による分析による食事の説明または画像の入力フィールド。
毎日の進捗状況の追跡と、認証とデータベース管理のための Supabase との統合。
美しくモバイル対応の UI。
```

ZenOpus が UI スケルトンを生成したら、次のようにします。

- ナビゲーションの設定とランディング ページのデザインを確認します。
- プロンプトを繰り返してレイアウトを調整し、モバイルの応答性を確保します。

<Note>
UI から始めることを明確にしてください。 ZenOpus は、タスクが明確なステップに分割されている場合に最適に機能します。たとえば、プロンプトの早い段階で、ナビゲーション、ページ構造、デザイン テーマの概要を説明します。
</Note>
</Step>
<Step title="UIのデザイン">
ZenOpus は、建物の建設に似た段階的なアプローチを強調しています。

- **足場**から始める: アプリの構造 (ユーザー エクスペリエンス フロー、ページ レイアウトなど) を計画します。
- **最初のレイヤー** を追加します。基本的なデザイン要素を含む空のページを作成します。
- **詳細**を調整する: 色、アニメーション、モバイルの応答性を追加します。

<Note>
Figma に事前に作成されたデザインがある場合は、スクリーンショットを撮って ZenOpus プロンプトに含めることができます。高解像度の画像を提供すると、より良い結果が得られます。
</Note>
</Step>
<Step title="バックエンドと認証のために Supabase を統合する">
UI の準備ができたら、バックエンド機能を追加します。

1. **Supabase****アカウント**を設定し、プロジェクトを作成します。
1. 「Connect Supabase」ボタンを押して、ZenOpus にリンクします。
1. データ ストレージの場合、ZenOpus は SQL テーブルを生成します。たとえば:

- ユーザー情報を保存する `profiles` テーブル。
- 毎日のエントリを追跡するためにユーザーにリンクされた `meals` テーブル。

<Note>
Supabase で行レベルのセキュリティ (RLS) ルールを使用して、ユーザーのプライバシーを確保します。 ZenOpus はこれを自動的に処理します。
</Note>
</Step>
<Step title="食事分析のための OpenAI の統合">
このアプリの魅力はその AI 機能にあります。食事を分析するには:

1. 開発者ダッシュボードから OpenAI API キーを取得します。
1. 食事の説明や画像を分析するには GPT-4 を使用します。
1. 不正アクセスを防ぐために、API キーが Edge 機能に安全に保存されていることを確認します。

**含める機能:**

- **食事の説明を分析**: 栄養価 (カロリー、タンパク質、炭水化物、脂肪) を推定します。
- **画像認識**: 食事の写真をアップロードすると、GPT-4 が説明と栄養データを抽出します。

<Note>
*Pro OpenAI 統合のヒント:*

- 構造化された応答には **関数呼び出し** を使用し、データがデータベースに挿入できるように正しくフォーマットされていることを確認します。
- 「卵 2 個」などの単純な食事エントリを分析するようにアプリに依頼して、機能をテストします。
</Note>
</Step>
<Step title="Stripe を使用した決済の実装">
アプリを収益化するには、サブスクリプションベースの支払いを追加します。

1. Stripe アカウントを作成し、製品を設定します (例: 月額 12 ドルの「スタンダード プラン」)。
1. API キーと製品価格 ID を追加して、Stripe を ZenOpus と統合します。
1. 料金を支払っていないユーザーをランディング ページにリダイレクトします。メインアプリには有料ユーザーのみがアクセスできます。

<Note>
*追加のヒント:*

- Stripe の **カスタマー ポータル** を有効にして、ユーザーがサブスクリプションを管理できるようにします (支払い方法のキャンセルや更新など)。
- 提供されたテスト用クレジット カードの詳細を使用して、Stripe のテスト モードで支払いをテストします。
</Note>
</Step>
<Step title="テストとデバッグ">
エラーは開発の自然な一部分であり、ZenOpus によりデバッグが簡素化されます。

- ビルド エラーが発生するたびに、**「修復を試みる」** ボタンを使用してください。
- バックエンドの問題については、Supabase ログで詳細なエラー レポートを確認してください。
- Netlify にデプロイする場合は、デプロイメント ログでビルドの問題を確認してください。 ZenOpus とログを共有して、調整された修正を取得します。

<Note>
行き詰まった場合は、ZenOpus の「元に戻す」機能を使用して以前のバージョンに戻してください。これにより、クレジットと時間を節約できます。
</Note>
</Step>
<Step title="アプリのデプロイ">
アプリの準備ができたら、ZenOpus または Netlify を使用してデプロイします。

1. ZenOpus のデプロイ機能を使用して、デフォルトのドメインでアプリをホストします。
1. カスタム ドメインの場合:

- GitHub リポジトリを Netlify に接続します。
- ドメインを購入し、Netlify にリンクするように DNS 設定を構成します。

<Note>
ブランディングとSEOを向上させるために、アプリのメタデータ（ファビコン、メタタグ、OG画像）を更新します。
</Note>
</Step>
</Steps>

<Accordion title="Web 開発でインタラクティブな要素を構築する">
<iframe src="https://www.youtube.com/embed/b-O7Ew0Q2ig?si=UPyQQdnualImIP8j" title="YouTubeビデオプレーヤー" width="100%" height="315" frameBorder="0" allow="accelerometer; autoplay; clipboard-write; encrypted-media; gyroscope; picture-in-picture; web-share" referrerPolicy="strict-origin-when-cross-origin" allowFullScreen />

ZenOpus の最新アップデートにより、魅力的でインタラクティブな Web エクスペリエンスの構築がさらに簡単になりました。

<Steps>
<Step title="ビジュアル編集による簡単な UI カスタマイズ">
ZenOpus への最大のアップグレードの 1 つは、AI クレジットを消費せずに UI 要素を即座に変更できる機能です。次のことができるようになりました。

- プロジェクト上でテキストを直接編集します。
- ボタンの色とスタイルをリアルタイムで調整します。
- 画像をシームレスにアップロードして交換します。
- 要素の上にマウスを置くと、簡単に調整できます。

<Note>
ユーザーは、AI の介入を必要とせずに、テキストの選択、色の変更、アイコンの交換を簡単に行うことができるため、よりスムーズなデザインの反復プロセスが可能になります。
</Note>
</Step>
<Step title="高度なデバッグと API エラー処理">
以前は、ユーザーはサードパーティ API 統合エラーに対処する際に課題に直面していました。現在 ZenOpus:

- デバッグを容易にするために、より豊富なエラー メッセージが表示されます。
- 無関係なポップアップを排除し、重要な問題に集中できるようにします。
- AI によるエラー検出を有効にして自動修正します。

<Note>
一般的なポップアップでユーザーを圧倒するのではなく、ZenOpus はエラー コンテキストを AI に直接フィードし、問題にいつ対処する必要があるか、およびその修正方法を判断できるようになりました。
</Note>
</Step>
<Step title="動的な背景とアニメーションの作成">
ZenOpus は、**p5.js** およびその他のライブラリを使用したアニメーション要素とインタラクティブ要素のシームレスな統合をサポートするようになりました。

#### プロジェクト例: インタラクティブなフラグを備えた旅行代理店の Web サイト

最近のライブ デモでは、次のような旅行代理店のランディング ページを作成する方法が紹介されました。

1. **アニメーション化されたヒーロー セクション**。マウスの操作に応じてフラグが動的に表示され、動きます。
1. **ユーザーのスクロール位置に基づいて変化するスティッキー スクロール セクション**。
1. **インタラクティブなホバー効果**。画像がユーザーの動きに反応します。

#### アニメーション効果を実装する方法

これらの効果を作成するには、ユーザーは次の簡単なプロセスに従いました。

- **目的のアニメーションを定義します:**

<Note>
背景にマウスを置くと遠ざかるフローティング要素をインタラクティブにします。
</Note>

- **p5.js を使用してエフェクトを生成します:**
    - 有彩色の煙のパターンを実装します。
    - 静的な画像を動的な旗のアニメーションに置き換えます。
- **物理ベースのインタラクションでユーザー エクスペリエンスを向上:**
    - ユーザーのアクションに基づいて要素を動的に移動させます。

<Note>
アニメーションが機能しない場合は、ZenOpus の Plan mode を使用してデバッグし、段階的な AI ガイダンスで効果を調整します。
</Note>
</Step>
<Step title="外部ライブラリからのカスタム要素のインポート">
**21st.dev** を使用すると、ユーザーは事前に構築された UI コンポーネントを ZenOpus プロジェクトにシームレスに統合できるようになりました。

1. ボタンのデザインについては **21st.dev** を参照してください。
1. AI が生成したプロンプトをコピーします。
1. これを ZenOpus に貼り付けて、既存のボタンを置き換えます。

このアプローチにより、設計ワークフローが加速され、より洗練された、視覚的に一貫したインターフェイスが可能になります。

<Note>
これと同じ方法が **motion.dev** からアニメーション プリセットをインポートする場合にも機能し、カスタム コーディングなしで高品質のモーション エフェクトを簡単に追加できるようになります。
</Note>
</Step>
</Steps>

## 本物のマイクロ SaaS を構築、ブランド化、立ち上げましょう。

スペイン語をこれほどインタラクティブに学習できることはかつてありませんでした。会話を通じてスペイン語を教えるだけでなく、完全に AI を活用してリアルタイムの音声インタラクション、ダイナミックなコース、視覚的に魅力的なコンテンツを作成するアプリを想像してみてください。このチュートリアルは、クリスチャンとハリーが構築したものを段階的に理解するのに役立ちます。

<iframe src="https://www.youtube.com/embed/8QDL9cCIXqU?si=JPHSSBoWO4dXl7vB" title="YouTubeビデオプレーヤー" width="100%" height="315" frameBorder="0" allow="accelerometer; autoplay; clipboard-write; encrypted-media; gyroscope; picture-in-picture; web-share" referrerPolicy="strict-origin-when-cross-origin" allowFullScreen />

<Steps>
<Step title="基礎を築く">
AI 機能を実装する前に、**ZenOpus** を使用してアプリの **強力な基盤を構築**することに重点を置きました。これには、認証の設定、ダッシュボードの構築、ユーザー管理の実装が含まれます。
</Step>
<Step title="認証システム">
- バックエンド認証用に **Supabase** を統合しました。

- レスポンシブ UI を確保し、デバイス間でシームレスなエクスペリエンスを実現します。
</Step>
<Step title="ユーザープロファイルとデータストレージ">
- ユーザーの進行状況を保存するデータベース構造を追加しました。

<Accordion title="ステップ 2: Replicate を使用した AI を活用した学習機能">
<iframe src="https://www.youtube.com/embed/eSDhk4VkVLE?si=-JhSjxsKIrPACNOK" title="YouTubeビデオプレーヤー" width="100%" height="315" frameBorder="0" allow="accelerometer; autoplay; clipboard-write; encrypted-media; gyroscope; picture-in-picture; web-share" referrerPolicy="strict-origin-when-cross-origin" allowFullScreen />

<Steps>
<Step title="AI が生成したコース">
学習をインタラクティブにするために、OpenAI の GPT モデルを使用して AI 主導のコース生成システムを実装しました。仕組みは次のとおりです。

1. ユーザーはトピックを選択します (例: 「バーベキューで尋ねる質問」)。
1. ZenOpus は OpenAI Edge 関数を呼び出して、**10 の魅力的なスペイン語の質問**を生成します。
1. ユーザーはクイズに答え、進捗状況を追跡し、回答に対する AI フィードバックを受け取ります。
</Step>
<Step title="ユーザーエンゲージメントの強化">
- **動的に生成される質問**: 各クイズは AI によって生成され、毎回新鮮でユニークなコンテンツが保証されます。
- **インタラクティブなフィードバック**: AI は、学習を改善するために正解と不正解の説明を提供します。
- **進捗状況の追跡**: ユーザーはコースを保存して再訪できるため、定着率と実践力が高まります。
</Step>
<Step title="Replicate によるビジュアルエンゲージメントの強化">
視覚的な学習要素を追加するために、**Replicate の Flux Schnell** 画像生成モデルを統合しました。

- **自動化されたコース バナー**: すべてのコースは、トピックに一致する画像を動的に生成します。
- **視覚的に没入できるエクスペリエンス**: AI 主導のビジュアルにより、ユーザーのエンゲージメントと維持率が向上します。
- **高速画像処理**: Replicate の API を使用すると、エクスペリエンスを低下させることなく画像がリアルタイムで生成されます。
</Step>
<Step title="リアルタイムの音声会話">
アプリ開発における大きなマイルストーンは、OpenAI のリアルタイム WebRTC API を使用した **リアルタイムのスペイン語会話練習** の追加でした。

- **インタラクティブ AI 講師**: ユーザーは、スペイン語で動的に応答する AI と直接話すことができます。
- **発音支援**: AI がリアルタイムで修正と提案を提供します。
- **シームレスな会話**: 自然な会話の流れにより、人間の家庭教師と対話しているように感じられます。
</Step>
</Steps>

<iframe src="https://www.youtube.com/embed/fILdgFNqvmU?si=_Xg-XTMTQuBN7bE1" title="YouTubeビデオプレーヤー" width="100%" height="315" frameBorder="0" allow="accelerometer; autoplay; clipboard-write; encrypted-media; gyroscope; picture-in-picture; web-share" referrerPolicy="strict-origin-when-cross-origin" allowFullScreen />

<Steps>
<Step title="コンセプトと対象者の設定">
ブランド化する前に、私たちは中核となる目標を次のように定義しました。

- **AI を活用したチャット** でリアルタイムのスペイン語会話が可能。
- **自動化されたクイズ**で簡単な練習セッションが可能。
- **構造化された学習** レッスンの保存と進捗状況の追跡。

これはブランディングの選択を形作るのに役立ち、アプリの目的と確実に一致するようになりました。
</Step>
<Step title="命名とドメインの選択">
私たちは、**学習の旅**を反映した短くて記憶に残る名前を目指しました。 AI を使用してブレインストーミングを行い、**Ruta** (スペイン語で「ルート」または「道」を意味) にたどり着きました。

#### プロセス:

- **AI を使用してキャッチーなスペイン語の単語について名前のブレインストーミング**を行いました。
- **「Camino」や「Verbo」などのオプションをテストしました** が、「Ruta」が際立っていました。
- **ドメインの空き状況を確認**し、すぐに**rutaap​​p.com**を確保しました。
</Step>
<Step title="ビジュアルアイデンティティの作成">
私たちはすぐにブランドをデザインしました。

- **ロゴ**: デザイン ライブラリから取得され、Figma で編集された **コンパスをテーマにした**アイコン。
- **フォント ペアリング**: ロゴには **太字でモダンな書体**、アプリ インターフェイスには読みやすい UI フォントを使用しました。
- **カラー パレット**: 旅行と発見を象徴する **Tailwind CSS ブルー** スキーム。
- **スタイリングのインスピレーション**: **ネオブルータリズム**を実験し、コントラスト、影、境界線を微調整して、視覚的に魅力的な UI を作成しました。
</Step>
<Step title="一貫したユーザーエクスペリエンスの構築">
ZenOpus を使用して、アプリ全体で **視覚的な一貫性** を確保しました。

- **ランディング ページ**: AI によって生成された、ダイナミックで魅力的なヒーロー セクション。
- **ダッシュボードとコース ページ**: ブランドのアイデンティティに合わせたスタイル。
- **反復的なスタイリング**: **ネオ ブルータリスト** 要素を適用して、さまざまなデザイン スタイルをテストします。
- **ユーザー フロー テスト**: サインアップからリアルタイム チャットまでのシームレスなナビゲーションを保証します。
</Step>
