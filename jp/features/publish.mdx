---
title: "ZenOpus プロジェクトを公開する"
description: "プロジェクトをライブ URL にデプロイし、公開されたアプリにアクセスできるユーザーを制御し、サイトのメタデータを構成し、いつでも更新または非公開にします。"
---
公開すると、共有可能な URL にスナップショットが展開され、ZenOpus プロジェクトがライブ Web アプリに変わります。現在のバージョンのみがデプロイされ、プランに応じて、公開されたアプリにアクセスできるユーザーを制御できます。

<Note>
**プロジェクトの可視性と Web サイトへのアクセス (重要な違い)**

ZenOpus は 2 つの独立した表示設定を使用します。

- **プロジェクトの可視性 (エディター アクセス)** は、ソース コード、進行中の作業、およびまだ公開されていない変更を含む、エディターでプロジェクトを表示および編集できるユーザーを制御します。
- **Web サイト アクセス (公開 URL アクセス)** は、ライブ URL で公開アプリにアクセスできるユーザーを制御します。

これらの設定は独立しています。公開しても、エディターでプロジェクトにアクセスできるユーザーは変わりません。また、プロジェクトの表示設定を変更しても、公開されたアプリにアクセスできるユーザーには影響しません。
</Note>

## プロジェクトを公開できるのは誰ですか?

デフォルトでは、**編集者以上**はすべてのプランでプロジェクトを公開できます。

**エンタープライズ プラン** では、管理者と所有者は、Web への外部公開を許可するユーザーを制限できます。 **Workspace 設定 → プライバシーとセキュリティ → 外部公開できるユーザー** に移動し、次を選択します。

- 管理者と所有者
- 所有者のみ

## 社内外の出版と Web サイトのアクセス制御

公開オプションと Web サイトのアクセス制御は、プランによって異なります。

### Free および Pro プラン

**リンクを知っている人なら誰でも**、公開されたアプリにアクセスできます。これは、外部の Web に公開することを意味します。 **ウェブサイトへのアクセス**はこれらのプランでは制限できないため、公開する前に共有の準備ができていることを確認してください

### Business およびエンタープライズ プラン

公開されたアプリにアクセスできるユーザーを選択できます。

- **Workspace**: 認証された Workspace メンバーのみが公開されたアプリにアクセスできます。つまり、内部的に Workspace に公開します。選択すると、公開ボタンのラベルが **Workspace に公開** に変わります。
- **誰でも**: リンクを知っている人は誰でもあなたの公開アプリにアクセスできます。つまり、**外部の Web** に公開することになります。

これにより、次のことが可能になります。

- Workspace に秘密を保つ内部アプリを構築して共有します
- 誤って外部に公開することを防止する
- Support 機密データのガバナンスとコンプライアンス

Workspace 管理者と所有者は、**Workspace 設定 → プライバシーとセキュリティ → デフォルトの Web サイト アクセス**で、公開されているすべてのプロジェクトに対して **デフォルトの Web サイト アクセス** ポリシーを設定することもできます。

**公開モーダル**で **URL にアクセスできるユーザー**を選択することで、個々のプロジェクトの Workspace デフォルトをオーバーライドできます。変更しない場合、プロジェクトは Workspace デフォルトを継承します。

## プロジェクトを公開する方法

公開するには、次の手順に従います。

<Steps>
<Step title="プロジェクトを開き、「発行」モーダルに移動します。">
プロジェクトで、右上隅にある [**公開**] アイコンをクリックします。以下で説明するように、公開設定を確認して構成します。
</Step>
<Step title="Webサイトのアドレス（公開URL）を設定します。">
- 希望の URL を入力するか、ZenOpus の入力フィールドを空のままにして URL を自動生成します。デフォルトでは、アプリは `[published-url].zenopus.app` に公開されます
- 公開したら、[カスタム ドメイン](/jp/features/custom-domain) を追加できます (有料プランで利用可能)。
</Step>
<Step title="Web サイトのアクセス許可を構成する">
プランに応じて、**URL にアクセスできるユーザー**を制御できます。

- **誰でも**: リンクを知っている人は誰でも、公開されたアプリの URL にアクセスできます (公開 Web サイトへのアクセス)
- **Workspace** (Business および Enterprise プラン): 認証された Workspace メンバーのみが、公開されたアプリの URL にアクセスできます (プライベート Web サイト アクセス)。
</Step>
<Step title="Web サイト情報の構成 (SEO およびソーシャル プレビュー用のメタデータ)">
ZenOpus は **サイト メタデータ**を提供しますが、ブラウザーのタブ、検索結果、リンク プレビューでのサイトの表示方法をカスタマイズできます。

- **アイコンとタイトル**: ブラウザーのタブ、検索結果、リンク プレビューに表示されるサイト アイコン (**ファビコン**) と **サイト タイトル**。
    - アイコンをクリックして新しいアイコンをアップロードします。
    - タイトルのテキストをクリックして更新します。
- **説明**: 検索結果とリンクのプレビューで使用される**メタ説明**。
    - テキストをクリックして更新します。
- **ソーシャル共有画像**: Web サイトのリンクがソーシャル メディアやメッセージング アプリなどで共有されるときに表示されるソーシャル共有画像 (**OG 画像**)。
    - クリックして画像をアップロードするか、画像の生成を選択します (生成するにはタイトルと説明を入力する必要があります)。デフォルトでは、アプリのスクリーンショットが使用されます。

検索エンジン、ソーシャル メディア プレビュー、AI システム向けに ZenOpus アプリを最適化する方法の詳細については、[SEO と GEO のベスト プラクティスを実装する](/jp/tips-tricks/seo-geo) をご覧ください。
</Step>
<Step title="公開前にセキュリティを確認する">
[**セキュリティの確認**] をクリックして脆弱性を確認します。

次のことを防ぐために、公開する前に必ずセキュリティ レビューを実行してください。

- データ漏洩
- 不正アクセス
- 機密情報の漏洩

詳細については、[セキュリティの概要](/jp/features/security) および [プロジェクト セキュリティ ビュー](/jp/features/security-view) を参照してください。
</Step>
<Step title="プロジェクトを公開する">
準備ができたら、[**公開**] をクリックします。デプロイが完了すると、公開されたリンクを含むポップアップが表示されます。

プロジェクトを公開した後、引き続きそれを反復処理できます。更新を後でプッシュするには、**公開 → 更新** をクリックします。
</Step>
</Steps>

公開後は、いつでもライブ アプリを更新したり、Web サイト アドレス (公開 URL) を変更したり、[カスタム ドメイン](/jp/features/custom-domain) に接続したりして、洗練されたブランドに合わせたエクスペリエンスを実現できます。

## ZenOpus での公開の仕組み

公開するたびに、ZenOpus はプロジェクトのスナップショットをライブ URL にデプロイします。現在のバージョンのみがデプロイされ、将来の変更は自動的に公開されません。

- 変更は自動的に公開されず、ライブにプッシュされません。
- 新しい変更を展開するには、**公開 → 更新** をクリックします。

ライブサイトに最近の変更が表示されない場合は、再公開する必要があります。

## プロジェクトを非公開にする方法

ライブアプリを削除するには、**プロジェクト設定 → プロジェクトの非公開 → 非公開** に移動します。

非公開にすると:

- ライブ URL にアクセスできなくなります
- プロジェクトはエディターに残ります

## よくある質問

<AccordionGroup>
<Accordion title="公開するとプロジェクトとコードが公開されますか?">
いいえ。公開すると、公開された URL でアプリが利用可能になるだけです。エディター内のプロジェクトまたはプロジェクト コードへのアクセスを誰にも許可するものではなく、プロジェクトが自動的にリミックス可能になることもありません。

エディター、コード、および未公開の変更へのアクセスは、[プロジェクトの可視性](/jp/features/project-visibility) によって常に制御されます。公開プロジェクトの公開設定を持つプロジェクトのみが、誰でも表示またはリミックスできます。
</Accordion>
<Accordion title="プロジェクトの可視性と Web サイトへのアクセスはどのように連携しますか?">
プロジェクトの公開設定と Web サイトへのアクセスは **独立した設定**です。ニーズに応じて、さまざまな方法でそれらを組み合わせることができます。

- **プロジェクトの可視性**は、コードや進行中の作業を含む**エディタ内のプロジェクト**にアクセスできるユーザーを制御します。
- **ウェブサイト アクセス**は、ライブ URL で**公開アプリ**にアクセスできるユーザーを制御します。

一般的な構成は次のとおりです。

1. **内部チーム アプリ****結果:** Workspace メンバーのみがエディターでプロジェクトを表示および編集し、公開されたアプリにアクセスできます。
    - プロジェクトの可視性: Workspace
    - ウェブサイトへのアクセス: Workspace
1. **プライベート作業中、公開アプリ****結果:** エディターでプロジェクトを表示および編集できるのは自分だけですが、リンクを知っている人は誰でも公開アプリにアクセスできます。
    - プロジェクトの可視性: 制限付き
    - ウェブサイトへのアクセス：誰でも
1. **チームで構築された公開共有アプリ****結果:** Workspace メンバーのみがエディターでプロジェクトを表示および編集できますが、リンクを知っている人は誰でも公開アプリにアクセスできます。
    - プロジェクトの可視性: Workspace
    - ウェブサイトへのアクセス：誰でも
1. **オープンなコラボレーション****結果:** 誰でもエディターでプロジェクトを表示およびリミックスでき、リンクを知っている人は誰でも公開されたアプリにアクセスできます。
    - プロジェクトの公開範囲: パブリック
    - ウェブサイトへのアクセス：誰でも
1. **プライベート プロトタイプが内部で共有されました****結果:** エディターでプロジェクトを表示および編集できるのは自分だけであり、公開されたアプリにアクセスできるのは Workspace メンバーだけです。
    - プロジェクトの可視性: 制限付き
    - ウェブサイトへのアクセス: Workspace

<Note>
**重要な注意事項:** 公開しても、エディターでプロジェクトにアクセスできるユーザーは変わりません。また、プロジェクトの公開設定は、公開されたアプリにアクセスできるユーザーに影響しません。
</Note>
</Accordion>

**Workspace** アクセスを選択して内部公開すると、**認証された Workspace メンバーのみ**が公開されたアプリにアクセスできるようになります。

更新を公開するには、**公開**をクリックしてから**更新**をクリックします。

<Accordion title="ファビコン、サイトのタイトル、メタディスクリプション、OG 画像などのサイトのメタデータを変更するにはどうすればよいですか?">
はい。いつでも、ブラウザのタブ、検索結果、リンク プレビューでのサイトの表示方法をカスタマイズできます。

デフォルトでは、ZenOpus はサイト メタデータを生成し、ZenOpus ロゴをファビコンとして追加し、アプリのスクリーンショットを Open Graph（OG）画像として追加します。

これを変更するには、**公開** をクリックし、**Web サイト情報**を確認して構成します。

- **アイコンとタイトル**: ブラウザーのタブ、検索結果、リンク プレビューに表示されるサイト アイコン (ファビコン) とページ タイトル。アイコンをクリックして新しいアイコンをアップロードします。ページ タイトルのテキストをクリックして更新します。
- **説明**: 検索結果とリンクのプレビューで使用されるメタ説明。テキストをクリックして更新します。
- **共有画像**: Web サイトのリンクがソーシャル メディアやメッセージング アプリなどで共有されるときに表示されるソーシャル共有画像 (OG 画像)。クリックして画像をアップロードします。

検索エンジン、ソーシャル メディア プレビュー、AI システム向けに ZenOpus アプリを最適化する方法の詳細については、[SEO と GEO のベスト プラクティスを実装する](/jp/tips-tricks/seo-geo) をご覧ください。
</Accordion>
<Accordion title="公開した URL (Web サイトのアドレス) を変更するにはどうすればよいですか?">
はい。 [zenopus.app](http://zenopus.app) のウェブサイト アドレスを形成するプロジェクト **URL サブドメイン**は、次の 2 つの方法で変更できます。

1. [**公開**] をクリックし、サブドメインを編集し、[**更新**] をクリックします。
1. **プロジェクト設定 → URL サブドメインの編集** に移動し、サブドメインを変更して、**URL サブドメインの更新** をクリックします。

プロジェクトの名前を変更してもプロジェクトの URL は変更されず、プロジェクトの表示名が変更されるだけであることに注意してください。

有料プランでは、[カスタム ドメイン](/jp/features/custom-domain) を追加することもできます。
</Accordion>
<Accordion title="ZenOpus プロジェクトを公開できないのはなぜですか?">
公開エラーは通常、現在のバージョンのビルドの問題が原因で発生します。 ZenOpus エージェントに調査して問題の解決を支援するよう依頼してください。
</Accordion>
