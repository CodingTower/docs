---
title: "プロジェクト間の参照"
description: "コード、アセット、ファイル、チャット履歴など、Workspace 内のプロジェクト全体で実装を再利用します。一貫性を維持し、同じ機能の再構築を避けます。"
---
プロジェクト間参照により、ZenOpus は Workspace 内の他のプロジェクトの実装にアクセスして再利用できるようになります。

他のプロジェクトを参照する場合、ZenOpus はファイル構造の探索、ソース コードの読み取り、関連パターンの検索、関連チャット履歴へのアクセス、画像やフォントなどの資産の再利用またはコピーを行うことができます。これにより、エージェントは何かがどのように構築されたかを理解し、それを現在のプロジェクトで再作成または適応させることができます。

`@` メンションを使用してプロジェクトを明示的に参照することも、必要に応じて既存のプロジェクトを使用するように ZenOpus に依頼することもできます。 **参照されたプロジェクトは変更されません**。

## 他のプロジェクトを参照する方法

チャットに「`@`」と入力し、プロジェクトを選択します。例:

```text
@BrandSite のロゴとカラー パレットを使用します。
```

ZenOpus に既存のプロジェクトを使用するように依頼します。例:

```text
既存のメイン アプリと同じ認証設定を使用してこれを構築します。
```

同じプロンプトで複数のプロジェクトを参照します。たとえば:

```text
@MainApp の認証設定を使用してこのオンボーディング フローを構築します
@ContactApp からのフォーム検証パターン。
```

プロジェクト間の参照は、[Agent mode](/jp/features/agent-mode) および [Plan mode](/jp/features/plan-mode) で ZenOpus を求める場合、または **質問ツールで「その他」を選択して回答する場合に機能します**。

`+` をクリックして **@ 参照の追加** を選択することで、プロンプト ボックスから参照を直接追加することもできます。

<Note>
大きなファイルや結果は切り捨てられる場合があります。
</Note>

## 参考にできること

以下を再利用できます。

- コンポーネントとレイアウト システム
- アニメーションとスタイリング システム
- 認証フローと API 統合
- 機能の実装
- 画像やフォントなどのアセット
- 実装コンテキストに関連するチャット履歴

すべてのアクセスは読み取り専用で、Workspace 内のプロジェクトに限定されます。

## 権限とアクセス

プロジェクト間の参照は、Workspace 境界と共有設定によって制限されます。

- 同じ Workspace 内のプロジェクトのみを参照できます。 Workspace 間の参照はサポートされていません。
- 協力者を含むすべての Workspace メンバーは、アクセス権限を持つ任意のプロジェクトを参照できます。
- プロジェクト間のアクセスは読み取り専用です。参照されたプロジェクトは変更できません。

## プロジェクト間の共有を無効にする

プロジェクト間の共有は、さまざまなレベルで無効にすることができます。

- **Workspace レベル:**   

 Workspace 管理者と所有者は、Workspace 全体のプロジェクト間共有を無効にできます。  

**Workspace 設定 → プライバシーとセキュリティ → プロジェクト間共有。**
- **プロジェクトレベル:**   

 プロジェクト編集者以上は、特定のプロジェクトのプロジェクト間共有を無効にできます。  

**プロジェクト設定 → プロジェクト間共有。**プロジェクト レベルで無効にすると、そのプロジェクトは他のプロジェクトを参照できなくなり、他のプロジェクトも参照できなくなります。

これらのコントロールのいずれかがアクセスをブロックすると、参照は失敗します。

## よくある質問

<AccordionGroup>
<Accordion title="別の Workspace からプロジェクトを参照できますか?">
いいえ。プロジェクト間の参照は、同じ Workspace 内でのみ機能します。 Workspace 間の参照はサポートされていません。
</Accordion>
<Accordion title="プロジェクトを参照すると変更されますか?">
いいえ。プロジェクト間のアクセスはすべて読み取り専用です。参照されたプロジェクトは変更できません。
</Accordion>
<Accordion title="チームメンバーは私の制限されたプロジェクトを参照できますか?">
いいえ。**制限付き**の公開設定を持つプロジェクトは、その所有者のみが参照できます。チーム メンバーは、同じ Workspace 内であっても、制限されたプロジェクトを参照できません。
</Accordion>
<Accordion title="ZenOpus が別のプロジェクトにアクセスできないのはなぜですか?">
ZenOpus がプロジェクトにアクセスできない場合は、次の点を確認してください。

- プロジェクト間の共有が Workspace またはプロジェクト レベルで無効になっていないことを確認します。
- 両方のプロジェクトが同じ Workspace にあることを確認してください。 Workspace 間のアクセスはサポートされていません。
- プロジェクトの可視性を確認します。 **制限付き** に設定されたプロジェクトには、その所有者のみがアクセスできます。
- プロジェクトにアクセスする権限があることを確認してください。参照できるのは、アクセス権のあるプロジェクトのみです。

これらのコントロールのいずれかがアクセスをブロックすると、参照は失敗します。
</Accordion>
</AccordionGroup>
