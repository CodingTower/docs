<Tip>
**モードを選択してください**

ZenOpus には 2 つのモードがあります。

- **Plan mode** → 問題を熟考し、選択肢を検討し、アプローチを決定する
- [**Agent mode**](/jp/features/agent-mode) → 変更を実装し、結果を確認する

Plan mode は意思決定用です。 Agent mode は実行用です。

2 つのモードは連携して動作するように設計されており、いつでも切り替えることができます。
</Tip>

## 概要

Plan mode (以前のチャット モード) は ZenOpus の計画および推論モードであり、即時の実行よりも思考、探索、または明確さが重要な瞬間のために設計されています。複数のステップにわたって推論し、必要に応じてファイル、ログ、またはその他の関連するプロジェクト コンテキストを検査できます。

Plan mode では、コードを作成する前にアイデアを検討し、問題を調査し、変更について推論することができます。 ZenOpus では、目標と制約をより深く理解するために、明確な質問をすることがよくあります。提案すべき明確な実装がある場合、ZenOpus は検査、編集、調整できる正式な計画を作成します。

Plan mode がコードを変更することはありません。 Plan mode 内のすべてのメッセージから **1 クレジット**が差し引かれます。

## Plan mode の目的

Plan mode を使用すると、次のことができます。

- 方向性を決める前に製品や機能のアイデアを検討する
- コードベースまたは既存の動作について質問する
- コードを変更する前に、バグ、エラー、または混乱を招く動作を安全に調査します。
- 複数のアプローチを比較し、トレードオフを理解する
- アーキテクチャまたはデータベース スキーマの設計
- 潜在的な変更を行う前にその影響を理解する

Plan mode の会話の多くは探索的なものであり、明確さや指示で終わります。会話が具体的な変更につながった場合、Plan mode は、何かを実装する前に、提案されたアプローチをレビューして改良するための安全な方法を提供します。

## Plan mode の使用方法

Plan mode は次の 2 つの方法で使用できます。

- **新しいプロジェクトを開始する場合、またはアイデアを検討する場合は、Plan mode** から開始してください
- **開発中の任意の時点で Plan mode** に切り替えて、実行を一時停止して考えます

Plan mode は意図的に対話型になっています。 ZenOpus では、計画を提案する前に要件、制約、またはトレードオフを理解するために明確な質問をすることがあります。

### Plan mode をアクティブ化する

メッセージ入力の横にある [**計画**] をクリックし、検討または決定したい内容を説明します。

**あなたが望むものを具体的にしてください**

```text
パスワードのリセットによる電子メール/パスワード認証を追加します。 
ユーザーは 30 日間ログインしたままにする必要があります。 ZenOpus Cloud を使用します。
```

**探索に使用してください**

```text
このプロジェクトでリアルタイム通知を実装する最善の方法は何ですか? 
オプションとそのトレードオフについて説明します。
```

**デバッグに使用します**

```text
ユーザーは、ログインフォームがモバイルでは機能しないと報告しています。 
理由を理解するのを手伝ってください。
```

**詳細な内訳をリクエスト**

```text
ショッピング カート機能の実装を詳しく説明します。 
どのようなコンポーネント、データベース テーブル、API エンドポイントが必要ですか?
```

**レビューと改善を求める**

```text
現在の認証設定を確認します。 
どのようなセキュリティ改善を行う必要がありますか?
```

## Plan mode が計画を作成するとき

Plan mode は、常に構造化された実装計画を作成するとは限りません。計画は、提案すべき明確な実装がある場合にのみ作成されます。

計画が生成されると、専用の **計画ビュー** に読みやすい構造化ドキュメントとして表示されます。何かを実装する前に、全画面で開いてアプローチ全体を確認できます。

通常、計画には次のものが含まれます。

- アプローチの概要
- 重要な決定、仮定、制約
- コンポーネント、データモデル、API
- 段階的な実装順序
- スキーマ、フロー、アーキテクチャなどのオプションの図

「プラン」ビューからは、次のことができます。

- **プランをマークダウンとして直接編集**して、制約を追加したり、ステップを削除したり、セクションを書き換えたりできます。
- アプローチに満足したら、計画を**承認**してください

チャットで直接フォローアップの質問をしたり、変更をリクエストしたりできます。それに応じて ZenOpus が計画を更新します。

計画を承認する場合:

- ZenOpus は **Agent mode** に切り替わります
- 承認された計画に厳密に基づいて実装が開始されます

いつでも Plan mode に戻って、コードベースの現在の状態に基づいて新しい計画を生成できます。

## 計画と粘り強さ

プランを承認すると、**承認された最新バージョン** が `.zenopus/plan.md` に保存されます。

このファイルは、Agent mode が実装する現在の計画を表しており、他のプロジェクト ファイルと同様に検査できます。

**以前の計画は失われません**。 `plan.md` に保存されているのは最新の承認済みプランだけであっても、**チャット履歴**では利用可能なままであり、参照または比較のためにプラン ビューで再度開くことができます。

これにより、次のことが可能になります。

- 時間をかけてアイデアを繰り返す
- さまざまなアプローチを比較する
- 要件の進化に応じて安全に再計画
- 新しい共同作業者が実装の背後にある理由を理解できるように支援する

## よくある質問

<AccordionGroup>
<Accordion title="Plan mode によってコードが変更されることはありますか?">
いいえ。コードの変更は、計画を承認し、ZenOpus が Agent mode に切り替わった後にのみ行われます。
</Accordion>
<Accordion title="Plan mode にはクレジットがかかりますか?">
はい。 Plan mode 内のすべてのメッセージは **1 クレジット**を差し引きます。
</Accordion>
<Accordion title="計画はどこに保存されますか?">
最新の承認された計画は `.zenopus/plan.md` に保存されます。以前のプランには引き続きチャット履歴からアクセスできます。
</Accordion>
<Accordion title="古いプランにアクセスできますか?">
はい。以前のプランには引き続きチャット履歴からアクセスできます。
</Accordion>
<Accordion title="プランを作成した後に編集できますか?">
はい。計画は承認前に完全に編集可能です。
</Accordion>
<Accordion title="デバッグに Plan mode を使用できますか?">
はい。 Plan mode は、問題を調査し、実装前に修正を決定するのに適しています。
</Accordion>
<Accordion title="チャットモードはどこにありますか?">
**Plan mode** は、以前は **チャット モード** と呼ばれていました。名前は、実装前にモードがどのように使用されることを意図しているかをより適切に反映するために変更されました。
</Accordion>
<Accordion title="ZenOpus とチャットするときに特定のファイルを参照できますか?">
はい。現在のプロジェクトのコード ファイルをチャットで直接参照して、編集やディスカッションをより迅速かつ正確に行うことができます。

`@` と入力してファイルを選択するか、[コード エディター](/jp/features/code-mode) の **チャットでファイルを参照** ボタンを使用します。たとえば:

```text
@src/components/UserProfile.tsx 読み込み状態をここに追加します
```

これは、現在のプロジェクト内のファイルのみを参照します。別のプロジェクトを参照するには、[プロジェクト間参照](/jp/features/cross-project-referencing) を参照してください。
</Accordion>
<Accordion title="ZenOpus は別のプロジェクトのコードや機能を再利用できますか?">
はい。 ZenOpus は、同じ Workspace 内で [プロジェクト間参照](/jp/features/cross-project-referencing) をサポートします。

`@` メンションを使用して別のプロジェクトを直接参照することも、単に ZenOpus に既存の実装を再利用するよう依頼することもできます。エージェントは、他のプロジェクトから関連するコード、ファイル、アセット、チャット履歴にアクセスし、それらを現在のプロジェクトに適応させることができます。

プロジェクト間のアクセスは読み取り専用であり、Workspace 権限が尊重されます。
</Accordion>
</AccordionGroup>
