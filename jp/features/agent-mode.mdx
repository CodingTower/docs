<Tip>
**モードを選択してください**

ZenOpus には 2 つのモードがあります。

- [**Plan mode**](/jp/features/plan-mode) → 問題を徹底的に考え、選択肢を検討し、アプローチを決定する
- **Agent mode** → 変更を実装し、結果を確認する

Plan mode は意思決定用です。 Agent mode は実行用です。

2 つのモードは連携して動作するように設計されており、いつでも切り替えることができます。
</Tip>

## 概要

Agent mode は ZenOpus の自律実行モードで、プロジェクトに変更を直接実装するように設計されています。

ZenOpus にタスクを与えると、そのタスクがエンドツーエンドで実行の所有権を取得します。ユーザーの意図を理解し、コンテキストのコードベースを探索し、ファイル全体に変更を適用し、開発中に発生する問題を解決します。動作中は、目に見えるタスクの進行状況を確認し、次に進む前に結果を確認できます。

## Agent mode の目的

Agent mode を使用すると、次のことができます。

- アプローチが決まったら機能や変更を実装する
- バグやエラーをエンドツーエンドで修正する
- 複数のファイルにわたるコードをリファクタリングする
- フロントエンド、バックエンド、構成全体にわたって調整された変更を適用します。
- 実装中に発生する問題をデバッグする
- ログとネットワークアクティビティを検査して問題を特定する
- 必要に応じて外部ドキュメントや資産を取得します
- プロジェクトで使用する画像やビデオを生成または編集する
- 終了前に結果を確認する

すべての変更はプロジェクトに直接適用され、ファイルの差分と概要を通じて明らかになります。 **詳細ビュー**でステップごとに確認できます。

## Agent mode の使用方法

[Plan mode](/jp/features/plan-mode) に切り替えない限り、Agent mode はアクティブです。

使用するには、必要なものを記述してメッセージを送信してください。明確な要件と制約は、ZenOpus がより良い結果を生み出すのに役立ちます。

**あなたが望むものを具体的にしてください**

```text
名前、電子メール、メッセージフィールドを含むお問い合わせフォームを追加します。電子メールの形式を検証し、エラー メッセージをインラインで表示します。
```

**予想される動作について説明します**

```text
ユーザーが「カートに追加」をクリックすると、成功メッセージが表示され、ヘッダーのカート数が更新されます。カートのアイテムをローカル ストレージに保存します。
```

**複雑な変更のコンテキストを提供します**

```text
過去 30 日間の売上データを表示するようにダッシュボードを更新します。既存のグラフ コンポーネントを使用し、分析ページで使用されているスタイルと一致させます。
```

**敏感なエリアにはガードレールを設置してください**

```text
新しい機能を @src/pages/dashboard に追加します。 @src/shared/Layout.tsx または既存の認証ロジックを変更しないでください。
```

## タスクと実行の可視性

ZenOpus が動作している間、チャット インターフェイスに次のようなタスクが表示されます。

- 実行中の現在のステップ
- 変更中のファイル
- 使用されているツール (検索、Web フェッチ、画像生成)
- 複数のステップの実装を通じて進歩する

この可視性は次のことに役立ちます。

- 複雑なビルドの進行状況を追跡する
- 実行されている手順を理解する
- 複雑な変化の際にも方向性を保つ
- 何かがおかしいと思われる場合は、問題を早期に発見します

## プロンプトキュー

ZenOpus **一度に 1 つのタスクを処理します**。 ZenOpus が動作している間、プロンプトの送信を続けることができ、プロンプトはチャット入力の上に表示されるキューに追加されます。

- 必要に応じてキュー全体を一時停止および再開します
- キューに入れられた個々のプロンプトの並べ替え、編集、コピー、または削除
- キューに入れられたプロンプトを指定した回数 (最大 50 回) 繰り返す

これにより、バッチ作業、待ち時間なしの共同作業、反復的なワークフローの自動化が容易になります。

## デバッグと検証

Agent mode を使用する場合、ZenOpus はコードのコンパイルを保証するだけではありません。

調査と問題解決の一環として、エージェントは次のことを行うことができます。

- テスト実行によるビルド エラーと失敗を観察する
- 検証ツール使用時のコンソール出力とネットワーク要求を検査する
- ブラウザーのテストまたは対象を絞ったバックエンド呼び出しを使用して、報告された問題を再現します。
- 適切なチェックを実行して、修正が期待どおりに機能することを確認します。

ZenOpus は、ブラウザー テスト、フロントエンド テスト、エッジ機能検証などの検証ツールにアクセスして、システムの動作を理解し、ユーザー ワークフローを検証し、バックエンド ロジックが期待どおりに動作することを確認します。これらのツールのほとんどは、ユーザーが要求した場合にのみ実行されます。

テストおよび検証機能の詳細については、「アプリのテストと検証」(/jp/features/testing) を参照してください。

## 価格設定

Agent mode の価格は **使用量ベース**です。コストは次のような要因によって異なります。

- 変更されたファイルの数
- ロジック変更の複雑さ
- 必要なコードベースの探索量
- 検証、ブラウザチェック、Web検索、画像生成などのツールの使用

多くのリクエストの費用は 1 クレジット未満ですが、より複雑なタスクの費用はさらに高くなる場合があります。メッセージのコストは、そのメッセージの 3 点メニューから表示できます。詳細については、[クレジットの使用](/jp/introduction/plans-and-credits#クレジットの使用状況) を参照してください。

## よくある質問

<AccordionGroup>
<Accordion title="Agent mode の場合、ZenOpus はプロジェクトを直接変更しますか?">
はい。 ZenOpus は、Agent mode を使用するときに変更をプロジェクトに直接適用します。すべての変更は、ファイルの差分と概要を通じて確認できます。
</Accordion>
<Accordion title="実装中にエラーが発生した場合はどうなりますか?">
ZenOpus は、ログ、ランタイム出力、およびネットワーク アクティビティを検査し、問題が解決または明確になるまで修正を繰り返すことができます。詳細については、[アプリのテストと検証](/jp/features/testing) を参照してください。
</Accordion>
<Accordion title="ZenOpus が動作中に何をしているのかを見ることはできますか?">
はい。表示されるタスクには、進行状況やどのファイルが変更されているかなど、実行の各ステップが表示されます。
</Accordion>
<Accordion title="Agent mode での作業中に複数のメッセージを送信できますか?">
はい。追加のメッセージはキューに入れられ、順番に処理されます。キューに入れられたメッセージはいつでも並べ替え、一時停止、または削除できます。
</Accordion>
<Accordion title="リクエストを停止するにはどうすればよいですか?また、停止するとどうなりますか?">
ZenOpus が応答している間に **停止ボタン** をクリックすると、現在のタスクが直ちに停止されます。これにより、リクエストを停止して、さらにコンテキストを提供したり、エージェントを修正したりできます。

ZenOpus は、その時点までに行われたすべての変更を保持するため、完了した作業が失われることはありません。停止されたリクエストは、それまでに完了した作業に基づいて課金されます。

変更を削除したい場合は、**元に戻すボタン**を使用して前の状態に戻します。
</Accordion>
<Accordion title="いつ Plan mode に切り替える必要がありますか?">
次の場合は、[Plan mode](/jp/features/plan-mode) に切り替えます。

- 実装前に複雑な変更を検討する
- 複数のアーキテクチャアプローチを比較する
- コードを変更する前に計画を確認または編集してください
- 問題を調査し、最初に解決策を決定する
</Accordion>
<Accordion title="ZenOpus とチャットするときに特定のファイルを参照できますか?">
はい。現在のプロジェクトのコード ファイルをチャットで直接参照して、編集やディスカッションをより迅速かつ正確に行うことができます。

`@` と入力してファイルを選択するか、[コード エディター](/jp/features/code-mode) の **チャットでファイルを参照** ボタンを使用します。たとえば:

```text
@src/components/UserProfile.tsx 読み込み状態をここに追加します
```

これは、現在のプロジェクト内のファイルのみを参照します。別のプロジェクトを参照するには、[プロジェクト間参照](/jp/features/cross-project-referencing) を参照してください。
</Accordion>
<Accordion title="ZenOpus は別のプロジェクトのコードや機能を再利用できますか?">
はい。 ZenOpus は、同じ Workspace 内で [プロジェクト間参照](/jp/features/cross-project-referencing) をサポートします。

`@` メンションを使用して別のプロジェクトを直接参照することも、単に ZenOpus に既存の実装を再利用するよう依頼することもできます。エージェントは、他のプロジェクトから関連するコード、ファイル、アセット、チャット履歴にアクセスし、それらを現在のプロジェクトに適応させることができます。

プロジェクト間のアクセスは読み取り専用であり、Workspace 権限が尊重されます。
</Accordion>
</AccordionGroup>
