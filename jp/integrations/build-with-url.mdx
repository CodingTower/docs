ZenOpus API **URL を使用して構築** 機能を使用すると、URL パラメーターを介してプロンプトと画像を渡すことにより、プログラムでアプリケーションを作成できます。これにより、Web サイト、内部ツール、自動化ワークフローとのシームレスな統合が可能になり、ユーザーはワンクリックで ZenOpus アプリを生成できるようになります。

各リンクは ZenOpus に *何を構築するか*、オプションで *どのようなイメージを含めるか* を指示し、ZenOpus が残りを処理します。

例:

```text
https://zenopus.dev/?autosubmit=true#prompt=Create%20a%20todo%20app
```

このリンクをクリックすると、ZenOpus が自動的に開き、ToDo アプリの構築が開始されます。

## ユースケース

URL API を使用してビルドすると、次のことができます。

- Web サイトまたはアプリケーションに **Build with ZenOpus** ボタンを埋め込みます。ユーザーがボタンをクリックすると、ZenOpus に移動し、アプリが自動的に構築されます。
- 事前定義されたプロンプトを URL 経由で友人、同僚、またはソーシャル メディアと共有します。
- 内部ワークフローの一部としてアプリの生成を自動化します。
- プロンプトと参照画像を含む URL を使用して、アプリのプロトタイプを迅速に作成します。

URL を自分で構築することも、[ZenOpus リンク ジェネレーター](https://zenopus.dev/links) を使用することもできます。詳細については、以下を参照してください。

## ベースURL

```text
https://zenopus.dev/?autosubmit=true#
```

`autosubmit` (ブール値): リクエストを自動的に処理するには、`true` に設定する必要があります

## パラメータ付きの URL

```text
https://zenopus.dev/?autosubmit=true#prompt=YOUR_PROMPT_HERE&images=IMAGE_URL_1&images=IMAGE_URL_2
```

### パラメータ

|パラメータ |タイプ |説明 |必須/オプション |
| --- | --- | --- | --- |
|プロンプト |文字列 |構築するアプリケーションを説明するテキスト。最大 50,000 文字。 URL エンコードを使用します (たとえば、スペースの場合は %20)。非常に長い URL は、ブラウザーまたは解析の制約により失敗する可能性があります。 |必須 |
|画像 |文字列 |参照として最大 10 個の URL エンコードされた画像 URL。画像の URL はパブリックにアクセスできる必要があります。サポートされている形式: JPEG、PNG、WebP。 |オプション |

## ZenOpus リンク ジェネレーター

[ZenOpus リンク ジェネレーター](https://zenopus.dev/links) を使用すると、選択したプロンプトと画像を含むアプリを生成する共有可能な URL を簡単に構築できます。

1. 構築するアプリケーションを説明するプロンプトを入力します (最大 50,000 文字)。
1. 最大 10 個のパブリック画像 URL (JPEG、PNG、または WebP) をアップロードします。
1. 生成された ZenOpus URL をコピーして共有します。

このツールは、URL エンコード エラーを防止し、リンクが正しくフォーマットされていることを確認するのに役立ちます。

## URL の例を使用して構築する

**基本的なアプリの生成**

```text
https://zenopus.dev/?autosubmit=true#prompt=Create%20a%20todo%20app
```

**単一の参照画像**

```text
https://zenopus.dev/?autosubmit=true#prompt=Display%20these%20images&images=https://img.com/1.jpg
```

**複数の参照画像**

```text
https://zenopus.dev/?autosubmit=true#prompt=Display%20these%20images&images=https://img.com/1.jpg&images=https://img.com/2.jpg
```

## ベストプラクティス

- プロンプトは簡潔にして、主要な機能に焦点を当ててください。
- 最良の生成結果を得るには、明確で説明的な言語を使用してください。
- 生成プロセスのガイドに役立つよう、関連する場合は参照画像を含めます。
- 解析の問題を避けるために、プロンプトおよび画像 URL 内の特殊文字が URL エンコードされていることを確認してください。

## 共有 URL のアプリの作成と認証のフロー

ユーザーが [**URL を使用して構築**] リンクをクリックすると、次のようになります。

- **ログイン ユーザー**: プロジェクトを作成するために Workspace を選択すると、アプリの作成が自動的に開始されます。
- **ログインしていないユーザー**: サインアップ/ログイン ページにリダイレクトされます。認証が成功した後、プロジェクトを作成するために Workspace を選択すると、元のプロンプトと画像を保持したまま、アプリの作成が自動的に開始されます。

## エラー処理

URL が長すぎるか形式が不正な場合、次のような問題が発生する可能性があります。

- ブラウザの「URL が長すぎます」エラー
- 不完全なパラメータ解析
- 自動送信の失敗
