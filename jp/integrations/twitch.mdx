[Twitch](https://www.twitch.tv/) は、ゲーム、クリエイティブ コンテンツ、リアルタイムのコミュニティ インタラクションに重点を置いたライブ ストリーミング プラットフォームです。 Twitch コネクタを使用すると、ZenOpus アプリが Twitch の Helix API にアクセスできるようになり、チャンネルで起こっていることに反応するオーバーレイ、ダッシュボード、コミュニティ ツールをパワー ストリームできるようになります。

Twitch を使用すると、アプリで次のことが可能になります。

- Twitchチャンネルからライブチャットを表示する
- フォロワー数とチャンネル登録者数を表示する
- フォロワーやサブスクリプションのマイルストーンなどのカスタム目標を追跡します
- ストリームのステータス、視聴者数、チャンネルのメタデータを調べる
- クリップ、スケジュール、チャンネル情報の取得
- Twitch チャンネルにチャット メッセージを送信する
- チャンネル、ゲーム、カテゴリを検索する

Twitch は、アプリがストリームと並行して動作する場合に適しています。例には、オーバーレイ、モデレーション ツール、イベント ダッシュボード、クリエイター分析アプリなどが含まれます。

## Twitch接続の仕組み

Twitch は認証に OAuth 2.0 を使用します。 Twitch に接続すると、認証されたアカウントの代わりに ZenOpus が Twitch にアクセスすることを承認したことになります。

Twitch は、ZenOpus のゲートウェイ アーキテクチャを使用して、安全な OAuth 処理と自動トークン更新を実現します。認証と使用制限の詳細については、[ゲートウェイ ベースのコネクタ](/jp/integrations/introduction#ゲートウェイベースのコネクタ) を参照してください。

各 Twitch 接続は 1 つの Twitch アカウントに関連付けられます。必要に応じて、複数の接続を作成できます。たとえば、開発用、運用用、またはさまざまな作成者用に別個のアカウントを接続する場合があります。

## Twitchの接続方法

Workspace **管理者**と**所有者**はTwitchに接続できます。

接続が作成されると、Workspace 内のすべてのプロジェクトで Twitch が利用できるようになります。

### 前提条件

Twitch を接続する前に、以下があることを確認してください。

- Twitchアカウント
- ZenOpus Workspace 管理者または所有者の役割

### ステップ 1: 接続するアカウントを選択する

接続では Twitch アカウントを直接使用します。開発者アプリケーションや API キーは必要ありません。

接続されたアカウントに、ユースケースに必要な権限があることを確認してください。

たとえば:

- フォロワーデータを読み取るには、アカウントにチャンネルのモデレータアクセス権が必要です。
- チャット メッセージを送信するには、アカウントにチャネルへの投稿権限が必要です。
- サブスクリプション データにアクセスするには、接続されているアカウントがチャンネルの所有者である必要があります。

ほとんどのオーバーレイ設定では、ストリーマー自身のアカウントに接続することで最も幅広いアクセスが提供されます。

### ステップ 2: Twitch を ZenOpus に接続する

1. **[設定] → [コネクタ] → [共有コネクタ]** に移動し、**Twitch** を選択します。
1. [**接続の追加**] をクリックします。
1. **表示名** に、接続の名前を入力します。たとえば、`Twitch Production`。
1. **スコープの構成 (オプション)****詳細設定**を展開して、アプリに必要な Twitch API 権限を構成します。デフォルトのスコープには次のものが含まれます。追加のスコープは、クリップ管理、予測、投票、分析、チャネル ポイントなどの機能に使用できます。アプリに必要なスコープのみを有効にします。接続を編集して [**再接続**] を選択すると、後でスコープを更新できます。スコープの完全なリストについては、[Twitch OAuth スコープ リファレンス](https://dev.twitch.tv/docs/authentication/scopes/) を参照してください。
    - `user:read:email` — アカウントの電子メールを表示する
    - `channel:read:subscriptions` — チャンネル登録データの表示
    - `moderator:read:followers` — モデレートされたチャンネルのフォロワー データを表示する
    - `user:write:chat` — Twitch チャットにメッセージを送信する
1. [**接続**] をクリックします。 Twitch 認証ウィンドウが開きます。ポップアップが許可されていることを確認してください。
1. Twitch にログインし、要求された権限を確認し、**承認** をクリックします。

認証が完了すると、ZenOpus に接続が表示され、アプリで Twitch データの使用を開始できるようになります。

## 制限事項

Twitch コネクタは現在次のことを行うことができません。

- **ゲートウェイ経由でリアルタイムで Twitch チャットを読む**  

 Twitch チャットは IRC over WebSocket を使用しますが、ZenOpus ゲートウェイを介してプロキシすることはできません。チャット メッセージを読むには、アプリが Twitch のチャット エンドポイントに直接接続する必要があります: `wss://irc-ws.chat.twitch.tv`パブリック チャットの読み取りには認証は必要ありません。 Twitch API 経由でチャット メッセージを送信するには認証が必要です。
- **EventSub Webhook を受信します**  

 ストリーム開始通知やフォローイベントなどの受信 Twitch イベントは現在サポートされていません。
- **Support ユーザーごとの Twitch ログイン**  

 各接続は単一の Twitch アカウントを認証します。アプリのエンドユーザーは、このコネクタを介して自分の Twitch アカウントに接続できません。

## プロジェクトの接続を解除する方法

編集者以上は、接続を完全に削除しなくても、接続から特定のプロジェクトを削除できます。接続は他のプロジェクトでも引き続き利用できます。

プロジェクトのリンクを解除するには:

<Steps>
<Step title="コネクタに移動する">
**[設定] → [コネクタ] → [共有コネクタ]** に移動し、**Twitch** を選択します。
</Step>
<Step title="接続を開きます">
管理する接続を開きます。
</Step>
<Step title="プロジェクトの選択">
**リンクされたプロジェクト** で、リンクを解除するプロジェクトにチェックを入れます。
</Step>
<Step title="確認する">
[**プロジェクトのリンクを解除**] をクリックして確認します。
</Step>
</Steps>

リンクを解除すると、これらのプロジェクトはこの接続を通じて **Twitch** にアクセスできなくなります。プロジェクトで再び **Twitch** が必要な場合は、利用可能な接続にプロジェクトをリンクできます。

## 接続を削除する方法

Workspace 管理者と所有者は **Twitch** 接続を削除できます。

<Warning>
接続の削除は永続的であり、元に戻すことはできません。リンクされているすべてのプロジェクトから資格情報が削除され、この接続を使用しているアプリは新しい接続が追加されるまで動作を停止します。
</Warning>

削除する前に、**リンクされたプロジェクト** セクションを参照して、現在接続を使用しているプロジェクトを確認してください。

接続を削除するには:

<Steps>
<Step title="コネクタに移動する">
**[設定] → [コネクタ] → [共有コネクタ]** に移動し、**Twitch** を選択します。
</Step>
<Step title="接続を開きます">
削除する接続を開きます。
</Step>
<Step title="リンクされたプロジェクトを確認する">
**リンクされたプロジェクト** セクションを確認してください。
</Step>
<Step title="削除">
[**この接続を削除**] で、[**削除**] をクリックして確認します。
</Step>
</Steps>
