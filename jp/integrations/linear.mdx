[Linear](https://linear.app/) は、ソフトウェア チーム向けに構築されたプロジェクト管理ツールです。これにより、ZenOpus アプリが課題、プロジェクト、チーム、サイクル、コメントを操作できるようになり、チームの Linear Workspace と直接対話する内部ツールや自動化を構築できるようになります。

Linear を使用すると、アプリで次のことが可能になります。

- 問題とコメントを作成および更新する
- チーム、プロジェクト、サイクルデータを読み取る
- 問題のステータスと進捗状況を明らかにするダッシュボードを構築する
- トリアージ、レポート、計画のワークフローを自動化する

Linear は、アプリがエンジニアリング タスク、製品計画、または問題追跡をアプリのインターフェイスから直接操作する必要がある場合に適しています。

## リニア接続の仕組み

各リニア接続は、単一のリニア Workspace に対して許可されます。 Linear に接続すると、認証されたアカウントに代わって ZenOpus が Linear にアクセスすることを承認したことになります。

ZenOpus Workspace 内:

- 複数のリニア接続を作成できます。
- 各接続は個別の OAuth 認証です。
- 各接続では次のことが可能です。
    - 異なるスコープを使用する
    - 別のリニア Workspace に接続します
- 単一の Workspace 内の複数のプロジェクトは同じ接続を使用できます。

これにより、環境 (開発環境と運用環境など) を分離したり、複数の Linear ワークスペースを管理したりすることが簡単になります。

Linear は、ZenOpus のゲートウェイ アーキテクチャを使用して、安全な OAuth 処理と自動トークン更新を実現します。認証と使用制限の詳細については、[ゲートウェイ ベースのコネクタ](/jp/integrations/introduction#ゲートウェイベースのコネクタ) を参照してください。

## リニアの接続方法

Workspace **管理者** と **所有者** は Linear に接続できます。

接続が作成されると、Workspace 内のすべてのプロジェクトで Linear が使用可能になります。

### 前提条件

Linear を接続する前に、次のものがあることを確認してください。

- OAuth 経由で認証できる Linear アカウント
- ZenOpus Workspace 管理者または所有者の役割

### リニア接続をセットアップする

<Warning>
Linear に接続すると、Workspace 内のすべてのユーザーが Linear データにアクセスできるようになります。
</Warning>

<Steps>
<Step title="リニアコネクタに移動します">
**設定 → コネクタ → 共有コネクタ** に移動し、**リニア** を選択します。
</Step>
<Step title="新しい接続を追加する">
[**接続の追加**] をクリックします。
</Step>
<Step title="接続に名前を付けます">
**表示名** に、接続の名前 (`Linear (Prod)` など) を入力します。
</Step>
<Step title="スコープの構成 (オプション)">
[**詳細設定**] を展開して、要求された OAuth スコープ、つまりアプリに必要なリニア権限を確認または調整します。 **読み取り** と **書き込み** はデフォルトで選択されており、最も一般的な使用例をカバーします。

<Tip>
アプリに必要なスコープのみを選択してください。接続を編集して [**再接続**] をクリックすると、後でスコープを更新できます。完全なスコープのドキュメントについては、[線形スコープのリファレンス](https://linear.app/developers#scopes) を参照してください。
</Tip>
</Step>
<Step title="Linear に接続し、接続を承認します">
- [**接続**] をクリックします。リニア認証ウィンドウが開くので、ブラウザがポップアップをブロックしていないことを確認してください。
- 接続する Workspace を選択し、要求された権限を確認して、[**許可**] をクリックします。確認メッセージとともに ZenOpus にリダイレクトされます。
</Step>
</Steps>

接続すると、ZenOpus アプリは線形データの使用を開始できます。

## 制限事項

リニア コネクタは Support を行いません。

- Linear Webhook またはリアルタイム イベント サブスクリプションの受信。
- 各エンドユーザーが自分の Linear アカウントに接続するユーザーごとの Linear ログイン

## プロジェクトの接続を解除する方法

編集者以上は、接続を完全に削除しなくても、接続から特定のプロジェクトを削除できます。接続は他のプロジェクトでも引き続き利用できます。

プロジェクトのリンクを解除するには:

<Steps>
<Step title="コネクタに移動する">
**設定 → コネクタ → 共有コネクタ** に移動し、**リニア** を選択します。
</Step>
<Step title="接続を開きます">
管理する接続を開きます。
</Step>
<Step title="プロジェクトの選択">
**リンクされたプロジェクト** で、リンクを解除するプロジェクトにチェックを入れます。
</Step>
<Step title="確認する">
[**プロジェクトのリンクを解除**] をクリックして確認します。
</Step>
</Steps>

リンクを解除すると、これらのプロジェクトはこの接続を通じて **Linear** にアクセスできなくなります。プロジェクトで再び **Linear** が必要な場合は、利用可能な接続にプロジェクトをリンクできます。

## 接続を削除する方法

Workspace 管理者と所有者は **リニア** 接続を削除できます。

<Warning>
接続の削除は永続的であり、元に戻すことはできません。リンクされているすべてのプロジェクトから資格情報が削除され、この接続を使用しているアプリは新しい接続が追加されるまで動作を停止します。
</Warning>

削除する前に、**リンクされたプロジェクト** セクションを参照して、現在接続を使用しているプロジェクトを確認してください。

接続を削除するには:

<Steps>
<Step title="コネクタに移動する">
**設定 → コネクタ → 共有コネクタ** に移動し、**リニア** を選択します。
</Step>
<Step title="接続を開きます">
削除する接続を開きます。
</Step>
<Step title="リンクされたプロジェクトを確認する">
**リンクされたプロジェクト** セクションを確認してください。
</Step>
<Step title="削除">
[**この接続を削除**] で、[**削除**] をクリックして確認します。
</Step>
</Steps>
