**ZenOpus 統合** を使用すると、アプリを **外部ツール、MCP サーバー、API** に接続して機能を追加し、実際のデータをコンテキストとして使用し、サードパーティ サービスを安全に統合できます。

ZenOpus を使用すると、以下を統合できます。

- **共有コネクタ**: 導入されたアプリに機能を追加します
- **パーソナル コネクタ (MCP サーバー)** : アプリ作成時にコンテキストを提供します
- **カスタム機能用の任意の API**

## ZenOpus 統合の概要

|カテゴリー |目的 |例 |
| --- | --- | --- |
|共有コネクタ |デプロイされたアプリが呼び出すことができる機能と外部サービスを追加します。管理者によって一度構成されれば、Workspace 内の全員が利用できます。 | ZenOpus Cloud、ZenOpus AI、Stripe、Shopify、Firecrawl |
|パーソナルコネクタ（MCPサーバー） |個人のツール アカウントを接続して、構築中にコンテキストを提供します。自分の接続にアクセスできるのは自分だけです。 | Linear、Notion、Miro、任意のカスタム MCP サーバー |
|任意の API |デプロイされたアプリにカスタム機能が必要な場合、ZenOpus に組み込まれていないカスタム API またはサードパーティ API を統合します。任意の API |

<Info>
**API とは何ですか?**  

 API (アプリケーション プログラミング インターフェイス) は、サービスまたはアプリケーションが相互に通信し、データを共有するための構造化された方法です。何を要求できるか、何を返すか、要求の方法を定義します。
</Info>

## 共有コネクタ: 公開アプリに機能を追加します。

共有コネクタは、ZenOpus アプリの能力を拡張する最も簡単な方法です。これらは、アプリがアクションを実行したりデータを取得したりする必要があるときはいつでも呼び出すことができるプラグインのように機能します。

共有コネクタは、**デプロイされたアプリ**が運用機能を得るために外部サービスと対話する必要がある場合に最適です。

### 概要と使用例

- [ZenOpus Cloud](/jp/integrations/cloud): 組み込みのバックエンド サービス、認証、データベース、フルスタック ホスティングを提供します。
- [ZenOpus AI](/jp/integrations/ai): 高度な AI 機能でアプリを強化
- [イレブンラボ](/jp/integrations/eleven-labs): 音声とテキスト読み上げの生成
- [Firecrawl](/jp/integrations/firecrawl): Web サイトのコンテンツのスクレイピング、クロール、抽出
- [困惑](/jp/integrations/perplexity): 質問に答え、Web を利用したリサーチを実行する
- [Shopify](/jp/integrations/shopify): e コマース運営の管理
- [Slack](/jp/integrations/slack): アラートの送信、チャンネルの読み取り、更新の投稿
- [Stripe](/jp/integrations/stripe): 支払いまたはサブスクリプションの処理
- [Supabase](/jp/integrations/supabase): ユーザーの認証とデータの保存
- [Telegram](/jp/integrations/telegram): ボットを介したメッセージの送信とコマンドの受信
- [Twilio](/jp/integrations/twilio): SMS および MMS メッセージの送信、音声通話の発信、電話ベースの通信フローの追加
    <Tip>
    認証、データベース、ストレージなどのバックエンド機能が必要な場合は、組み込みのフルスタック ホスティング プラットフォームである [ZenOpus Cloud](/jp/integrations/cloud) を使用してください。
    </Tip>

### 主な特徴

共有コネクタは、プロジェクト全体で再利用可能なアプリ機能を提供する Workspace レベルの統合です。

- **[設定] → [コネクタ] → [共有コネクタ]** で管理されます
- Workspace レベルで 1 回構成されます
- Workspace 内の全員が利用可能で、すべてのプロジェクトで再利用可能
- アプリの使用中に呼び出すことができる機能を提供する

**Business およびエンタープライズ プラン** では、Workspace 管理者および所有者は、Workspace の共有コネクタの可用性を管理できます。

- **[設定] → [コネクタ] → [共有コネクタ]** のコネクタ設定 (歯車アイコン) を使用して、特定の共有コネクタを有効または無効にします。

<Note>
ZenOpus Cloud を Workspace に対して無効にすることはできません。
</Note>

### ゲートウェイベースのコネクタ

一部の共有コネクタは、ZenOpus アプリとサードパーティ サービス (Slack など) の間で **安全なゲートウェイ** を使用します。

ゲートウェイベースのコネクタを使用する場合:

- **認証は自動的に処理されます**ゲートウェイは OAuth を管理し、バックグラウンドで期限切れのトークンを更新するため、一度接続すると、手動で再認証しなくても統合は機能し続けます。
- **資格情報がプロジェクトに公開されることはありません**トークンはゲートウェイに安全に保存され、プロジェクト設定に表示されず、Workspace 管理者がアクセスすることもできません。アプリはコネクタを呼び出すだけで、認証はバックグラウンドで処理されます。
- **プロジェクトごとの使用制限があります**ゲートウェイ コネクタでは、**プロジェクトごとのコネクタごとに 1 分あたり最大 1,000 リクエスト**が許可されます。各プロジェクトには独自の制限があるため、あるプロジェクトでの使用が他のプロジェクトに影響を与えることはありません。ほとんどのアプリはこのしきい値に達しませんが、大容量の自動化や頻繁なバックグラウンド操作を構築している場合は、この制限を念頭に置いて設計する必要があります。

他の共有コネクタは、サービスに応じて異なる認証モデルを使用する場合があります。 API キーを使用して直接接続するものもありますが、ZenOpus によって完全に管理されるものもあります。ゲートウェイ モデルは、安全なトークン処理と自動更新を必要とする OAuth ベースのサービスに特に使用されます。

## パーソナル コネクタ: アプリ作成時にコンテキストを提供します

[パーソナル コネクタ (MCP サーバー)](/jp/integrations/mcp-servers) を使用すると、**アプリの作成中**に、ZenOpus エージェントに内部ツールとコンテンツへのアクセスが許可されます。これらは、エージェントが実際のワークフローを反映するアプリを構築できるようにコンテキストを提供します。

組織がすでに依存している実際のコンテンツを使用してアプリを作成する必要がある場合は、パーソナル コネクタを使用します。

### 概要と使用例

- **Amplitude:** 製品分析とフィードバックを使用してアプリの決定を通知します
- **Atlassian (Jira および Confluence):** チケット、ロードマップ、ドキュメントとの調整
- **グラノーラ:** 会議のメモ、決定事項、実行項目を仕様とタスクに変換します。
- **線形:** 問題と仕様をインポートして機能を生成する
- **Miro:** Miro ボードと図にアクセスしてインタラクティブなプロトタイプを生成する
- **概念:** アプリの動作をガイドするためのドキュメントとページの参照
- **n8n:** ミラーリングまたは既存の自動化ワークフローへのプラグイン
- **Polar:** 請求、製品、サブスクリプション データを使用して SaaS アプリの足場を構築
- **健全性:** CMS コンテンツとスキーマにアクセスしてコンテンツ主導型アプリを構築する
- **カスタム MCP サーバー:** 内部システムまたはサードパーティ システムに接続します

### 主な特徴

パーソナル コネクタを使用すると、独自のツールに接続して、(アプリの機能ではなく) 構築中にコンテキストを提供できます。

- **[設定] → [コネクタ] → [個人用コネクタ]** で管理します
- ユーザーごとの接続。つまり、自分だけが接続にアクセスして管理できる
- 他の Workspace メンバーとは共有されません
- デプロイされたアプリには含まれていません

**Business およびエンタープライズ プラン** では、Workspace 管理者および所有者は、Workspace のパーソナル コネクタの可用性を管理できます。

- **[設定] → [プライバシーとセキュリティ]** ですべての **MCP サーバー アクセス**を有効または無効にします
- **[設定] → [コネクタ] → [パーソナル コネクタ]** のコネクタ設定 (歯車アイコン) を使用して、特定の MCP サーバーを有効または無効にします。

## 任意の API を ZenOpus と統合

ZenOpus は、パブリックかプライベートか、認証されているかどうかに関係なく、**任意の外部 API** と統合できます。これは最も柔軟な統合タイプであり、ネイティブでは Support にはない機能 ZenOpus を組み込むことができます。

ZenOpus に API の統合を依頼すると、適切なアプローチが自動的に選択されます。

- **API が認証を必要としない場合**、アプリは API を直接呼び出します。
- **API に認証が必要な場合**、ZenOpus はクラウドを有効にしてシークレットを追加するように求め、機密資格情報を保護するためのエッジ機能を自動的に作成します。

### 認証なしの API

API が認証情報を必要としない場合、セットアップは必要ありません。 ZenOpus に API を統合するよう依頼するだけです。たとえば:

```text
チャック・ノリス・ジョークの統合: https://api.chucknorris.io/jokes/random
```

ZenOpus はキーが必要ないことを検出し、機能をアプリに直接追加します。

### 認証済み API (シークレットが必要)

認証資格情報または API キーを必要とする API の場合、ZenOpus は ZenOpus Cloud を使用して統合を安全に処理します。認証情報はリンクされたクラウド プロジェクトにシークレットとして保存され、Edge Functions を介してのみアクセスされるため、確実に保護されます。

**セットアップ**

1. ZenOpus に API を統合し、重要な詳細を含めるよう依頼します。
    - エンドポイント URL と HTTP メソッド
    - 認証方法（APIキーヘッダー、クエリパラメータ、ベアラートークン、OAuth）
    - 必須ヘッダー (Content-Type、カスタム ヘッダー)
    - リクエストとレスポンスのフォーマット例（JSON、params、body）
    - OpenAPI 仕様またはドキュメントのリンク
1. [ZenOpus Cloud](/jp/integrations/cloud) を有効にします。
1. API キーを **Cloud → Secrets** に追加します。

**例:**

```text
OpenWeatherMap API を統合します。 
ベース URL: https://api.openweathermap.org/data/2.5。 
認証: クエリ パラメーター appid として渡される API キー。 
現在の天気を取得するためのエンドポイントが必要です: GET /weather?q={city}&units=metric&appid={API_KEY}。 
ドキュメント: https://openweathermap.org/current
```

ZenOpus は、ユーザーが都市を入力して気象データを取得できるエッジ機能および天気ウィジェットとの統合を生成します。

## よくある質問

<AccordionGroup>
<Accordion title="ZenOpus Support はどのような統合を行いますか?">
ZenOpus は、**共有コネクタ**、**個人コネクタ (MCP サーバー)**、**任意の API** の 3 種類の統合をサポートしています。共有コネクタはデプロイされたアプリに機能を追加し、個人コネクタはアプリの作成中にコンテキストを提供し、API 統合により ZenOpus に組み込まれていない外部サービスに接続できます。
</Accordion>
<Accordion title="ZenOpus の共有コネクタと個人コネクタの違いは何ですか?">
**共有コネクタ**は、展開されたアプリが外部サービスを呼び出すために使用し、Workspace レベルで一度構成されます。 **パーソナル コネクタ (MCP サーバー)** は、ZenOpus エージェントにツールやコンテンツへのアクセスを許可するためにアプリの作成時にのみ使用され、デプロイされたアプリには含まれません。
</Accordion>
<Accordion title="ZenOpus は任意の API と統合できますか?">
はい。 ZenOpus は、パブリックまたはプライベートに関わらず、**任意の外部 API** と統合できます。認証なしの API は直接呼び出すことができますが、認証された API は ZenOpus Cloud、シークレット、およびエッジ関数を使用して安全に統合され、資格情報を保護します。
</Accordion>
</AccordionGroup>
