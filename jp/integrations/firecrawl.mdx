---
title: "アプリを Firecrawl に接続する"
description: "AI を活用したスクレイパーおよびクローラーである Firecrawl にアプリを接続し、Web サイトから構造化データを抽出します。"
---
[Firecrawl](https://www.firecrawl.dev/) は、ZenOpus アプリが **ライブ Web サイト データ**を操作できるようにする Web スクレイピングおよびクロール プラットフォームです。これは、1 回限りのインポートや静的データセットに依存するのではなく、Web サイトから情報を継続的に取得、更新、構造化する必要があるアプリ向けに設計されています。

Firecrawl を使用すると、アプリで次のことが可能になります。

- URLを構造化されたJSONにスクレイピングする
- サイトマップ、サブページ、ナビゲーションを含む Web サイト全体をクロールします。
- クリーンなテキスト、メタデータ、製品の詳細、表などを抽出します
- JavaScript を多用するサイト、遅延読み込み、Cookie モーダルの処理
- スケジュールに従ってライブデータを再取得し、結果を最新の状態に保ちます

Firecrawl は、アプリが **リアルタイムまたは頻繁に変更される Web サイト コンテンツ**に依存している場合に適しています。

## 一般的なユースケースとサンプルアプリ

|サンプルアプリ |プロンプトの例 |説明 |
| --- | --- | --- |

|ジョブアグリゲーター |求人アグリゲーターを構築します。ユーザーは、事前に設定されたリストから企業を選択します。 Firecrawl を使用して、各企業の採用ページからリストを取得し、結果を構造化し、最新の状態に保ち、新しい役割を強調表示します。 |複数のソース (Greenhouse、Lever、Workday など) から求人情報を取得し、1 つの統合インターフェイスに表示します。ユーザーが事前設定されたリストから企業を選択すると、アプリはリストを構造化された結果に収集し、新しい役割が現れるたびにフィードを更新し続けます。 |
|リード強化ツール |見込み顧客強化ツールを構築します。ユーザーは会社のドメインを入力します。 Firecrawl で主要なページ (会社概要、チーム、キャリア、連絡先) をクロールし、公開されている会社の詳細を構造化されたプロファイルに抽出します。 |企業の Web サイトから公開されている情報を使用して、企業プロフィールを充実させます。ユーザーが会社のドメインを貼り付ける (またはリストをアップロードする) と、アプリが主要なページをスキャンして人物と会社の情報、技術スタックのシグナル、場所、経歴、連絡先の詳細を抽出し、抽出されたフィールドを各会社に保存します。 |
| Eコマース価格モニター |価格モニターを構築します。ユーザーは製品の URL を保存します。 Firecrawl を使用して、価格と在庫状況を抽出し、スケジュールを確認し、履歴を保存し、変更を警告します。 |電子商取引サイト全体で製品の在庫状況と価格を追跡します。ユーザーが製品の URL を保存すると、アプリがスケジュールに基づいてそれらをチェックし、スナップショットを記録し、変更にフラグを立てて、価格が下がったり製品が在庫に戻ったときにユーザーに警告します。 |
|ブランド資産抽出ツール |ブランド キット抽出ツールを構築します。ユーザーは Web サイトの URL を入力します。 Firecrawl を使用して色、フォント、ロゴを抽出し、シンプルなブランド キットを出力します。 |任意の Web サイトから軽量のブランド キットを生成します。ユーザーが会社の URL を入力すると、アプリがブランド関連のページをスクレイピングして、フォント、色、ロゴ、スタイル トークンを抽出します。 |
|知識抽出ツール |知識抽出ツールを構築します。ユーザーは URL を貼り付けます。 Firecrawl を使用して、クリーンなマークダウンとメタデータを返し、保存し、再同期を許可します。 |あらゆる Web ページをクリーンで再利用可能なコンテンツに変えます。ユーザーが URL を貼り付けると、アプリがページをクリーンなマークダウン、構造化メタデータ、読み取り可能なテキスト、および表などの有用な構造化要素に変換して、検索、要約、再利用できるようにします。 |

## Firecrawlの接続方法

Workspace **管理者** と **所有者** は Firecrawl に接続できます。接続を追加するときは、次のいずれかを選択できます。

- **ZenOpus** によって管理 (Firecrawl アカウントや API キーは必要ありません)
    <Tip>
    Firecrawl は、ZenOpus による管理を選択した場合、ZenOpus を通じて 2026 年 4 月 30 日まで無料で使用できます。
    </Tip>
- **独自の資格情報** (独自の Firecrawl API キーを持参してください)

接続が作成されると、Workspace 内のすべてのプロジェクトで Firecrawl が使用できるようになります。

Firecrawl の接続方法に基づいて、以下のセットアップ手順に従ってください。

<Tabs>
<Tab title="ZenOpus によって管理されます">
最速のセットアップが必要な場合は、このオプションを選択してください。 Firecrawl アカウントや API キーを作成する必要はありません。 ZenOpus は資格情報を提供および管理します。

Firecrawl に接続するには:

1. **[設定] → [コネクタ] → [共有コネクタ]** に移動し、**Firecrawl** を選択します。
1. [**接続の追加**] をクリックします。
1. **表示名**: 接続に名前を付けます (例: `Firecrawl Prod`)。
1. **認証**で、**ZenOpus** によって管理を選択します。
1. [**作成**] をクリックします。

接続すると、ZenOpus アプリは Firecrawl の使用を開始して、Web サイト データをスクレイピング、クロールし、最新の状態に保つことができます。
</Tab>
<Tab title="あなた自身の資格情報">
独自の Firecrawl アカウントを使用したり、請求を直接管理したり、独自の API キーを使用してレート制限や環境を制御したりする場合は、独自の資格情報を持参してください。

<Note>
このコネクタを通じて実行されるスクレイピングとクロールはすべて Firecrawl の使用量を消費し、請求は ZenOpus ではなく Firecrawl によって直接処理されます。
</Note>

異なる API キーを使用して複数の Firecrawl 接続を作成できます。これは、環境 (開発と運用など) を分離したり、レート制限を管理したりするのに役立ちます。

### 前提条件

Firecrawl を接続する前に、以下があることを確認してください。

- Firecrawl アカウントと API キー
- ZenOpus Workspace 所有者または管理者の役割

### ステップ 1: Firecrawl アカウントと API キーを作成する

Firecrawl では複数の API キーを作成できます。これは、さまざまな環境やレート制限に役立ちます。

Firecrawl API キーを作成するには:

1. [Firecrawl](https://www.firecrawl.dev/) に移動し、アカウントを設定します。
1. Firecrawl ダッシュボードに移動し、[API キー ページ](https://www.firecrawl.dev/app/api-keys) に移動します。
1. [**作成**] をクリックします。
1. キーにわかりやすい名前 (`ZenOpus integration` など) を付けます。
1. 生成された API キーをコピーし、安全に保管します。
    <Info>
    API キーはパスワードのように機能します。安全に保管し、決して公開しないでください。次のステップで必要になります。
    </Info>

Firecrawl はサインアップ時に無料のクレジットを提供します。接続をテストしてこのセットアップを完了するには、これで十分です。

### ステップ 2: Firecrawl を ZenOpus に接続する

異なる API キーを使用して複数の接続を作成できます。

Firecrawl に接続するには:

1. **[設定] → [コネクタ] → [共有コネクタ]** に移動し、**Firecrawl** を選択します。
1. [**接続の追加**] をクリックします。
1. **表示名**: 接続に名前を付けます (例: `Firecrawl Prod`)。
1. [**認証**] で [**独自の資格情報**] を選択し、独自の Firecrawl API キー (`fc-xxxxxxxx…`) を使用します。
1. [**作成**] をクリックします。

接続すると、ZenOpus アプリは Firecrawl の使用を開始して、Web サイト データをスクレイピング、クロールし、最新の状態に保つことができます。
</Tab>
</Tabs>

## プロジェクトの接続を解除する方法

編集者以上は、接続を完全に削除しなくても、接続から特定のプロジェクトを削除できます。接続は他のプロジェクトでも引き続き利用できます。

プロジェクトのリンクを解除するには:

<Steps>
<Step title="コネクタに移動する">
**[設定] → [コネクタ] → [共有コネクタ]** に移動し、**Firecrawl** を選択します。
</Step>
<Step title="接続を開きます">
管理する接続を開きます。
</Step>
<Step title="プロジェクトの選択">
**リンクされたプロジェクト** で、リンクを解除するプロジェクトにチェックを入れます。
</Step>
<Step title="確認する">
[**プロジェクトのリンクを解除**] をクリックして確認します。
</Step>
</Steps>

リンクを解除すると、これらのプロジェクトはこの接続を通じて **Firecrawl** にアクセスできなくなります。プロジェクトで再び **Firecrawl** が必要な場合は、利用可能な接続にプロジェクトをリンクできます。

## 接続を削除する方法

Workspace 管理者と所有者は **Firecrawl** 接続を削除できます。

<Warning>
接続の削除は永続的であり、元に戻すことはできません。リンクされているすべてのプロジェクトから資格情報が削除され、この接続を使用しているアプリは新しい接続が追加されるまで動作を停止します。
</Warning>

削除する前に、**リンクされたプロジェクト** セクションを参照して、現在接続を使用しているプロジェクトを確認してください。

接続を削除するには:

<Steps>
<Step title="コネクタに移動する">
**[設定] → [コネクタ] → [共有コネクタ]** に移動し、**Firecrawl** を選択します。
</Step>
<Step title="接続を開きます">
削除する接続を開きます。
</Step>
<Step title="リンクされたプロジェクトを確認する">
**リンクされたプロジェクト** セクションを確認してください。
</Step>
<Step title="削除">
[**この接続を削除**] で、[**削除**] をクリックして確認します。
</Step>
</Steps>
