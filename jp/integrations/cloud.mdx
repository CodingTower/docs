ZenOpus Cloud は、ユーザーに表示されるフロントエンドから、舞台裏で動作するバックエンドに至るまで、アプリケーションのあらゆる部分を処理するフルスタックのクラウド プラットフォームです。必要なものを ZenOpus に直接記述するだけで、ZenOpus によってフロントエンドとバックエンドの両方が自動的に生成され、クラウドで実行できるようになります。ハードウェアや複雑なインフラストラクチャを管理する代わりに、アプリの成長に合わせて自動的に拡張されるホスティング、コンピューティング、ストレージにオンデマンドでアクセスできます。

ZenOpus Cloud は、Supabase のオープンソース基盤を利用しています。つまり、Supabase を個別にセットアップする必要がなく、初日から本番環境を利用できるようになります。手動によるセットアップ手順を必要とせずに、本番環境に対応したデータベース、リアルタイム更新、ユーザー認証、ストレージを利用できます。

## ZenOpus Cloud を選択する理由

- **成長に合わせて拡張**: 最初のプロトタイプから数百万のユーザーまで、ZenOpus はニーズに合わせてシームレスに拡張します。
- **デフォルトで安全**: エンタープライズ グレードの認証、きめ細かいアクセス制御、コンプライアンスのベスト プラクティスにより、アプリとデータが保護されます。
- **費用対効果の高い**: 使用量ベースの価格設定により、複数のプロジェクトを 1 つの Workspace で管理します。すべての Workspace は **月額 25 ドルの無料使用量**から始まり、複数のプロジェクトを立ち上げて実行するのに十分です (2026 年第 1 四半期末までの一時的な提供であり、変更される可能性があります)。詳細については、[使用量ベースのクラウドと AI の価格設定](/jp/features/cloud#usage-based-cloud-and-ai-pricing) をご覧ください。

## ZenOpus Cloud の有効化

デフォルトでは、ZenOpus Cloud は Workspace に対して有効になっています。 **設定 → コネクタ → 共有コネクタ → ZenOpus Cloud → 権限の管理** で、プロジェクトの ZenOpus Cloud 動作を管理できます。

### 地域の選択

ZenOpus Cloud プロジェクトは、次の 3 つの異なるリージョンでホストできます。

- アメリカ大陸
- ヨーロッパ
- アジア太平洋地域

**[設定] > [コネクタ] > [共有コネクタ] > [ZenOpus Cloud] > [権限の管理]** で、新しいクラウド プロジェクトをホストするための優先デフォルト リージョンを設定できます。

優先地域が設定されていない場合、ZenOpus は現在の場所に最も近い地域を自動的に選択します。詳細については、「ZenOpus Cloud ツールの構成」(#configuring-zenopus-cloud-tools) を参照してください。

ユーザーに近い地域を選択すると、次の結果が得られます。

- 待ち時間の短縮
- データベース応答の高速化
- 地域ベースのデータホスティング
- 全体的なバックエンドのパフォーマンスの向上

<Note>
クラウドを有効にした後は、選択したリージョンを変更したり、リージョン間で既存のプロジェクトを移動したりすることはできません。
</Note>

### 権限の設定

クラウド統合のデフォルト設定は **常に許可** です。これは、ZenOpus がレビューや承認を求めずに自動的にアクションを実行することを意味します。 **設定 → コネクタ → 共有コネクタ → ZenOpus Cloud → 権限の管理** から、いつでも設定を変更できます。

ZenOpus は、機能でバックエンド機能 (認証やファイル ストレージなど) が必要な場合にプロンプトを表示します。次の 3 つのオプションがあります。

- **許可**: 現在のプロジェクトの統合を有効にします。
- **拒否**: このリクエストの統合を拒否します (後でもう一度尋ねられる可能性があります)。
- **環境設定の調整**: 今後のプロジェクトのデフォルトの動作を変更します (現在のプロジェクトには影響しません)。

### ZenOpus Cloud ツールの構成

クラウド ツールには 3 つの権限オプションがあります。

- **常に許可**: ZenOpus は、レビューや承認を求めずに、アクションを自動的に実行します。
- **毎回確認する**: ZenOpus は、アクションが必要な場合は常に承認を求めます。
- **決して許可しない**: ZenOpus はアクションをブロックし、クラウドへのアクセスが必要であることを通知し、クラウドを有効にするように指示します。

**[設定] → [コネクタ] → [共有コネクタ] → ZenOpus Cloud → [権限の管理]** からクラウド ツールの設定をいつでも変更できます。

#### ツールの設定とデフォルト

|設定 |説明 |デフォルトの動作 |
| --- | --- | --- |
|クラウドを有効にする |クラウド プロジェクトを作成して接続し、バックエンド機能 (データベース、認証、ストレージ) を追加します。クラウドツールを使用するために必要です。 |常に許可 |
|クラウド領域 |新しいクラウド プロジェクトのデフォルトのリージョン (最も近い、南北アメリカ、ヨーロッパ、またはアジア太平洋) を選択します。 「常に許可」を使用してクラウドが自動有効化されている場合、このリージョンは自動的に使用されます。 |現在地に最も近い地域 |
|データベースを読む | ZenOpus にデータベースの読み取りを許可します。 |常に許可 |
|分析を読む |ログをクエリして問題と動作を調査します。 |常に許可 |
|セキュリティチェックを実行する |データベースに対して自動セキュリティチェックを実行します。完全な監査ではありません。 |常に許可 |
|データベースを変更する | SQL 移行を通じてスキーマとポリシーを変更します。生成された型を更新します。 |常に許可 |
|データの追加 |データベーステーブルにデータを追加します。 |常に許可 |
|プロジェクト情報を読む |プロジェクトの構成と環境の詳細を確認します。 |常に許可 |
|認証を構成する | ZenOpus が認証設定 (サインアップ ポリシーなど) を構成できるようにします。 |常に許可 |
|バックエンドログを読む |バックエンド サービスからログを読み取ります。 |常に許可 |

## ZenOpus Cloud の機能

**プレビュー** **モード**の横にある `+` ボタンをクリックすると、ZenOpus プロジェクト内に **クラウド タブ**が表示されます。

![ZenOpus Cloud アイコン](/images/zenopus/integrations/cloud/cloud-icon.png)

次の機能と設定は [クラウド] タブ内にあります。

### データベース

- **スキーマを自動的に生成**: 必要なデータを平易な言語で記述すると、ZenOpus がテーブルとスキーマを作成します。
- **SQL を使用せずに管理**: UI で直接レコードを表示、編集、整理します。クエリを作成せずに、値を更新し、テーブルを検査し、アプリのデータを同期させます。

### ユーザーと認証

- **ユーザー管理**: ユーザー アカウントの表示、アクティビティの追跡、アクセス制御の管理を行います。
- **組み込み認証**: ZenOpus は、アプリのサインアップ/ログイン ページを生成できます。電子メール、電話、Google サインインをサポートします。
    - **Google 認証**: ZenOpus Cloud は、OAuth 2.0 を使用した **Google でのサインイン** をサポートしています。 Google 認証は、**マネージド** モード (ZenOpus__ マネージド OAuth) または **独自の Google Cloud 認証情報 (BYOK)** を使用して有効にできます。詳細については、[ZenOpus Cloud アプリに Google 認証を追加する](/jp/features/google-auth) を参照してください。
- **デフォルトで安全**: 認証は ZenOpus Cloud によってサポートされ、ユーザーの資格情報を安全に保ちます。

### ストレージ

- **コードを使用しないファイル処理**: プロフィール写真、ドキュメント、ビデオ、静的アセットなどのファイルをアップロード、管理、提供します。
- **ストレージ バケット**: ファイルは、アプリで使用できる URL または参照とともにプロジェクト バケットに安全に保存されます。
- **制限**: 最大 2 GB のファイルをサポートします。

### エッジ機能

- **サーバーレス ロジック**: サーバーをプロビジョニングせずに、API、ワークフロー、または高度な機能のカスタム コードを実行します。機能はトラフィックに応じて自動的に拡張されます。

**一般的な使用例:**

- AI サマリーや AI チャットボットなどの AI を活用した機能。組み込みの [ZenOpus AI](/jp/features/ai) を使用します。
- メールまたはプッシュ通知の送信
- 支払い処理 ([Stripe](/jp/integrations/stripe#stripe-integration) など)
- スケジュールされたタスク
- 大量の計算または外部 API 呼び出し

**その他のハイライト:**

- **作成は簡単**: ZenOpus チャットに必要な機能を記述するだけです。
- **デバッグ組み込み**: 各関数には、実行、出力、エラーのログが含まれています。何かが失敗した場合は、ZenOpus によってエラーが表示され、迅速なトラブルシューティングが可能になります。

### AI

- **組み込み AI 機能**: ZenOpus Cloud には AI が組み込まれているため、アプリをよりスマートで魅力的なものにする機能をすばやく作成できます。詳細については、[ZenOpus AI](/jp/features/ai) を参照してください。
- AI を活用した機能の例としては、次のようなものがあります。
    - **AI 要約** – 長いテキストを自動的に要約して明確な要点にまとめます
    - **AI チャットボットまたはエージェント** – 会話ヘルパーをアプリに組み込む
    - **感情検出** – ユーザーのフィードバックを大規模に理解します
- AI の概要から、プロジェクトの ZenOpus AI の使用状況とパフォーマンスを監視できます。

### 秘密

- **安全な環境変数**: API キー、トークン、資格情報などの機密データをコード内で公開せずに保存します。
- **自動検出**: 機能にシークレットが必要な場合、ZenOpus は安全な UI を介してシークレットを入力するよう求めます。
- **暗号化され安全**: シークレットは安全に保存され、Edge Functions またはその他の統合に自動的に挿入されます。

**例:** Stripe 秘密キーを一度保存すると、ハードコーディングしなくても ZenOpus によって支払い処理機能に確実に組み込まれます。

<Note>
Supabase キーや ZenOpus API キーなどの一部のシークレットは自動的に追加され、ZenOpus プラットフォームの機能に必要です。これらのキーは削除できません。
</Note>

### ログ

- **リアルタイム監視**: アプリのアクティビティを発生時に監視します。
- **問題のデバッグとトレース**: エラーを特定し、イベントを追跡し、バックエンドが正常であることを確認します。

### 使用法

[**使用状況**] タブには、選択した期間にわたってアプリが ZenOpus Cloud リソースをどのように消費しているかが視覚的に表示されます。

期間のデフォルトは **過去 7 日間**です。ドロップダウンから **最終日** に切り替えて、表示される期間を調整できます。クラウド使用量は毎日午前 10 時 (CET) に更新されます。

使用量は **リソース カテゴリ** に分類され、セグメント化されたバーとして表示され、パーセンテージで個別にリストされます。

- **データベース サーバー**: アプリがデータをクエリまたは更新するときに使用されます。アプリのトラフィックに合わせて [インスタンス サイズ](#詳細設定-インスタンスのアップグレード) を調整します。インスタンスが大きくなると、より多くの容量が提供されますが、クラウドのコストが増加します。
- **データベース ストレージ**: アプリのデータ (ユーザー、注文、設定など) を保存するためにデータベースで使用されるディスク容量。
- **コンピューティング**: アプリのコードが実行されるときに使用されます。
- **ストレージ**: 画像、ビデオ、アップロードなどのファイルを保存するために使用されます。
- **ライブ更新**: ユーザーにリアルタイムの更新を送信するために使用されます (通知やチャットなど)。
- **ネットワーク**: アプリがインターネット経由でデータを送受信するときに使用されます。

カテゴリは最高から最低の順に並べ替えられます。各カテゴリの横にある割合は、選択した期間のクラウドの合計コストに占める割合を示します。たとえば、**コンピューティング** に 29% と表示されている場合、その期間中のクラウド支出の 29% はコンピューティング使用量によるものです。

### 詳細設定 - インスタンスのアップグレード

詳細設定を使用して、現在のインスタンス サイズを表示および管理します。インスタンスが大きいほど、アプリはより多くのユーザーとトラフィックを処理できるようになります。アプリの使用量が増えると、クラウドの使用コストも増加することに注意してください。

- **Free プラン ユーザー**: いつでもプランをアップグレードすることで、**Tiny** から大規模なインスタンスにアップグレードできます
- **有料プラン ユーザー**: 以下のオプションから新しいインスタンス サイズを選択します。
    - **小さい**: 何かを試してみるのに最適
    - **Mini**: 初期のプロジェクトに信頼性の高い
    - **小規模**: アプリに合わせて成長する余地があります
    - **中**: 通常の使用に適した安定した選択
    - **大**: より高い需要に対応する自信のあるオプション

ZenOpus は、ZenOpus Cloud リソース制限に近づくと **警告**します。これにより、インスタンスをタイムリーにアップグレードして、高速で信頼性の高いパフォーマンスを確保できます。

アラートは、次のいずれかが発生したときにトリガーされます。

- ディスク容量が 10% 未満残っています
- IO 予算が 50% 未満です
- CPU負荷が80%を超えています

アップグレード アラートには、使用状況 (ディスク領域、ディスク IO、CPU) の詳細な内訳も含まれます。

![ディスク容量、IO バジェット、CPU 使用率の内訳を含むクラウド インスタンスのリソース制限を示すアラート通知で、ユーザーにインスタンス サイズのアップグレードを促します](/images/zenopus/integrations/cloud/cloud-instance-upgrade-alert.png)

### 詳細設定 - テスト環境とライブ環境 (ベータ版)

<Note>
クラウド プロジェクトで利用でき、ベータ期間中は無料です。価格は今後決定されます。
</Note>

テスト環境とライブ環境を使用すると、ZenOpus Cloud プロジェクトで作業しているときに **開発** を **運用データ** から分離できます。すべての開発は Test で安全に行われ、Live は明示的に公開した場合にのみ更新されます。

詳細については、[テスト環境およびライブ環境で安全に構築する (ベータ)](/jp/features/environments) を参照してください。

## フローの例 - CRM ツールの構築を求める ZenOpus のプロンプト

*クライアントを管理し、取引を追跡し、請求書を処理し、メモを整理するための CRM ツールを構築してください。ナビゲーション用のサイドバーを備え、シンプルかつモダンでビジネス向けのデザインを維持します。*

### ステップ 1: データを整理する (データベース)

ツールが追跡する必要がある情報の種類を ZenOpus に伝えます。 ZenOpus はテーブルを自動的に作成するため、自分でデータベースに触れる必要はありません。

*顧客の名前、メールアドレス、電話番号、メモを記録できる場所が欲しいです。また、どのクライアントを対象としたものなのか、金額、どの段階にあるのか、いつクロージングする予定なのかなど、取引を追跡する方法も必要です。また、所属するクライアントの請求書、金額、ステータス、期日を記載するセクションも必要です。*

### ステップ 2: 安全にする (ユーザーと認証)

ログインを追加すると、データが非公開になり、ユーザーが簡単にサインインできるようになります。

※アプリへのログイン機能を追加します。 CRM ダッシュボードにアクセスする前にログインが必要です。また、「Google でサインイン」ボタンを追加して、ユーザーが Google アカウントでログインできるようにします。*

### ステップ 3: ファイルの処理 (ストレージ)

ユーザーがアプリ内で直接ファイルをアップロードして表示できるようにすることができます。 ZenOpus はストレージとリンクを処理するため、どのファイルをどこに配置するかを記述するだけです。

*各取引に契約アップロードフィールドを追加します。クライアントにプロフィール写真を提供し、クライアントの詳細ページに表示できるようにします。請求書の PDF をアップロードし、請求書の表にダウンロード リンクを表示します。*

### ステップ 4: ワークフローを自動化する (エッジ機能)

時間を節約する自動アクションを設定します。

*新しい見込み客がクライアント テーブルに追加されたら、電子メール通知を送信してください。 ZenOpus AI を使用してクライアントの会議メモを分析し、感情スコアをクライアント テーブルに保存します。新しい連絡先を自動的に同期します。毎日午前 7 時に今日のフォローアップの概要をメールで送信するジョブをスケジュールします。 Stripe で支払いが「支払い済み」とマークされると、関連する請求書のステータスが自動的に「支払い済み」に更新されます。*

### ステップ 5: 他のツールに接続する (秘密)

Stripe や Google カレンダーなどのサービスを使用すると、秘密の「キー」が提供されます。  

 ZenOpus は、必要なときにいつでもそれらを安全に追加するように求めるプロンプトを表示し、アプリだけがそれらを使用できます。

### ステップ 6: AI 超大国 (AI) を追加する

ツールをアシスタントのように機能させます。

*各クライアントの会議メモを要約し、プロファイルに保存します。進行中の取引ごとに次のステップを提案します。潜在顧客をフォローアップするためにパーソナライズされた電子メールの下書きを生成します。クライアント メッセージをカテゴリ (販売、Support、または請求) に分類します。*

## 使用量に応じたクラウドと AI の価格設定

ZenOpus には、Free プランも含め、すべての料金プランにクラウド ホスティングと組み込み AI が含まれています。これらのサービスは **使用量ベースの価格設定モデル** で実行されます。つまり、コストは使用量に応じて変動し、サブスクリプションではカバーされません。

Workspace またはプロジェクトから、**設定 → クラウドと AI バランス** に移動します。ここでは、次のことができます。

- Workspace 残高ステータスの表示
- Workspace 残高に資金を補充して追加します (有料プランでのみ利用可能)
- 請求書履歴にアクセスし、支払い設定を管理します
- Workspace 全体のクラウドと AI の使用コストの合計を追跡します
- プロジェクトごとのクラウドと AI の使用コストの詳細を表示します
    <Note>
    現在、リストには、ZenOpus Cloud または AI を使用していないプロジェクトも含め、すべてのプロジェクトが含まれています。使用されていないプロジェクトには常に $0.00 が表示されます。
    </Note>

「クラウドと AI のバランス」画面が次のように更新されます。

- クラウド使用量は毎日午前 10 時 (CET) に更新されます
- AI の使用状況は継続的に更新されます

<Note>
- **クラウド残高**は、デプロイされたアプリのホスティング料金を支払います。
- **AI 残高**は、デプロイされたアプリ内の AI 機能の支払いに使用されます。
- **Workspace クレジット**は別のもので、アプリの構築中のメッセージングにのみ使用され、**[設定] → [プランとクレジット]** で追跡できます。
</Note>

### 無料使用料込み（全プラン）

すべての Workspace は、プランに関係なく、またプランを月のいつ開始したかに関係なく、毎月無料で使用できます。

- **25 ドルのクラウド残高** (アプリ ホスティング用)
- **AI 機能の AI 残高は 1 ドル**

これらの無料資金は毎月 1 日の 00:00 UTC (タイムゾーンで表示) にリセットされ、サブスクリプション プランの請求サイクルには関連付けられません。未使用の無料資金は翌月に繰り越されません。

<Note>
**一時的なサービス、変更される可能性があります:** 2026 年第 1 四半期の終わりまで、Free プランのユーザーであっても、すべての Workspace は月額 25 ドルのクラウドと 1 ドルの AI を利用できます。
</Note>

### 資金を追加する（有料プランのみ）

ZenOpus Cloud および AI の使用のために Workspace 残高に資金を追加することは、有料プランでのみ利用できます。 Free プランのユーザーは、この機能にアクセスするにはアップグレードする必要があります。

含まれている金額を超える使用量は別途請求され、サブスクリプションの対象にはなりません。

- **クラウドの使用量:** アクティビティ (ユーザー数とインタラクション) およびデータ転送 (アップロード/ダウンロード) によって異なります。
- **AI の使用:** は、使用するモデルとタスクの複雑さによって異なります (高度なモデルほどコストがかかります)。コストを確認するには、[ZenOpus AI](/jp/features/ai) のサポートされる AI モデルのリストにリンクされている公式ソースを参照してください。

有料プランの Workspace 所有者と管理者は、手動で 1 回限りのチャージを行うか、自動チャージを構成することで残高をチャージできます。

未使用の Workspace 資金は月ごとに繰り越されますが、1 年後に期限切れになります。

**ワンタイムチャージ**

Workspace 残高に、選択した金額を手動で補充します。

1. **[設定] → [クラウドと AI バランス] → [チャージ] → [ワンタイムチャージ]** に移動します。
1. カスタム金額を入力するか、事前定義されたオプション (最小 $10、最大 $1000) を選択します。
1. Stripe 経由で安全に支払いを完了してください。

**自動補充**

残高が 10 ドルを下回ると自動的に資金が追加されますが、毎月のチャージ制限を追加することで毎月のチャージを管理できます。

1. **[設定] → [クラウドと AI 残高] → [チャージ] → [自動チャージ]** に移動します。
1. 自動補充を有効にします。
1. 残高が 10 ドル (最低 10 ドル、最大 1000 ドル) を下回ると、自動的に請求されて Workspace 残高に追加される追加金額を入力します。
1. 毎月の料金制限を設定して、毎月合計でいくら請求できるかを制御します (最小 $10、最大 $50,000)。
1. [**保存**] をクリックします。   
月々の請求限度額に達していない限り、残高が 10 ドルを下回ると、デフォルトの支払い方法に自動的に請求されます。

自動チャージを設定すると、**自動チャージ** 画面から毎月の料金を視覚的に監視できます。月額料金期間は、毎月 1 日の 00:00 UTC (タイムゾーンで表示) にリセットされ、サブスクリプション プランの請求サイクルには関連付けられません。

### アラートと通知

ZenOpus は、予期しないサービスの中断を避けるための情報を提供します。

- **メール通知** Workspace の所有者と管理者に送信されます
- **クラウドとAIのバランス画面とクラウドの概要に表示される**使用量アラート**

Workspace が発生した場合にアラートがトリガーされます。

- 含まれる使用量の一定の割合に達する
- 含まれている使用量をすべて使い切る
- ウォレット残高の一定の割合に達する
- ウォレットの資金をすべて使い果たします

Workspace 残高がなくなると、資金が追加されるまで (または翌月の初めに無料使用量がリセットされるまで)、アプリケーションは動作を停止します。

アプリケーションを動作し続けるには:

- **Free プラン:** 有料プランにアップグレードして資金を追加します。
- **有料プラン:** Workspace 残高をいつでも補充するか、自動補充を有効にします。

## よくある質問

<AccordionGroup>
<Accordion title="毎月の出費を完全にコントロールできますか?">
はい。 ZenOpus Cloud は完全に制御できるように設計されています。

- **オプション 1: ハードバジェット (料金ゼロ)**  
厳密に無料許容範囲内に収まるようにしてください。それを超えると、プロジェクトは自動的に一時停止します。請求額は 0 ドルのままです。
- **オプション 2: 手動で補充する (制御されたスケーリング)**  
選択した金額を Workspace 残高に補充します。アプリは拡張可能で、無料枠を超えた使用量に対してのみ料金が発生します。   
Workspace 残高を使い切ると通知が届き、プロジェクトは自動的に一時停止されます。
- **オプション 3: 自動的に補充 (シームレスなスケーリング)**  
残高が 10 ドルを下回ると自動的に資金が追加されますが、毎月のチャージ制限を追加することで毎月のチャージを管理できます。 アプリは中断することなく拡張され、無料枠を超えた使用量に対してのみ料金が発生します。  
月額料金の上限に達したために資金を自動的に追加できない場合、残りの Workspace 残高がなくなるとプロジェクトは一時停止されます。
</Accordion>
<Accordion title="クラウドと AI の使用に対してクレジットは差し引かれますか?">
いいえ、Workspace クレジットは別のものであり、アプリの構築中のメッセージングにのみ使用されます。クレジットの使用状況は、**[設定] → [プランとクレジット]** で追跡できます。

**[設定] → [クラウドと AI のバランス]** でクラウドと AI の使用状況を追跡できます。

- クラウド残高は、デプロイされたアプリのホスティングに使用されます。
- AI 残高は、デプロイされたアプリ内の AI 機能の支払いに使用されます。
</Accordion>
<Accordion title="ZenOpus AI のコストはいくらですか?">
ZenOpus AI を使用するコストは、LLM プロバイダーに直接アクセスする場合とまったく同じです。隠れた手数料はありません。コストを確認するには、[ZenOpus AI](/jp/features/ai) のサポートされる AI モデルのリストにリンクされている公式ソースを参照してください。
</Accordion>
<Accordion title="現在、Supabase 統合を使用しています。この変化は私にとって何を意味するのでしょうか?">
心配しないでください。既存のプロジェクトは何も変わりません。 Supabase の統合は現在とまったく同じように引き続き機能し、Support を継続します。

- 既存のプロジェクトの場合: 変更せずに Supabase を引き続き使用します。
- 新しいプロジェクトの場合: ZenOpus Cloud または Supabase を選択します。
</Accordion>
<Accordion title="クラウドをプロジェクトに追加した後で考えを変えることはできますか?">
いいえ、プロジェクトをクラウドに接続すると、その変更を元に戻したり、プロジェクトをクラウドから切断したりすることはできません。
</Accordion>
<Accordion title="クラウド プロジェクトをリミックスできますか?">
はい、クラウド プロジェクトはクラウド プロジェクトにのみリミックスできます。これにより、現在選択されている Workspace で個別に編集できるプロジェクトのコピーが作成されます。テーブルやエッジ関数などのバックエンド関連データの構造は、情報なしでコピーされます。元のプロジェクトで自動的に生成されたキーについては、ZenOpus によってリミックスされたプロジェクトに新しいキーが生成されます。手動で追加されたキーはコピーされません。

リミックスするときに、新しいプロジェクトのバックエンドをホストする地域 (アメリカ、ヨーロッパ、またはアジア太平洋) を選択できます。 ZenOpus は、現在地に基づいて最も近い地域を事前に選択します。一度選択すると、そのプロジェクトのリージョンを変更することはできません。

プロジェクトが ZenOpus バックエンドを使用している場合、リミックスされたプロジェクト内のアプリを以前のバージョンに戻すことはできないことに注意してください。クラウドでホストされているフロントエンドのみのプロジェクトの場合は、元に戻すことが可能です。
</Accordion>
<Accordion title="自分のクラウド プロジェクトを Supabase にエクスポートできますか?">
GitHub にコードを所有しているため、これは技術的には可能ですが、簡単ではありません。私たちはこれを簡単にすることを検討しています。詳細については、「セルフホスティング: どこでも ZenOpus Cloud プロジェクトを実行する」(/jp/tips-tricks/self-hosting) を参照してください。
</Accordion>
<Accordion title="どのプロジェクトが最も使用量を生み出しているかを知るにはどうすればよいですか?">
**[設定] → [クラウドと AI のバランス]** に移動すると、プロジェクトごとのクラウドと AI の使用率の内訳を確認できます。

<Note>
現在、リストには、ZenOpus Cloud または AI を使用していないプロジェクトも含め、すべてのプロジェクトが含まれています。使用されていないプロジェクトには常に $0.00 が表示されます。
</Note>
</Accordion>
<Accordion title="プロジェクトを Supabase からクラウドに移行できますか?">
現時点では、Supabase からクラウドへの移行はサポートされていません。プロジェクトがすでに Supabase に接続されている場合は、今後も Supabase をサポートし続けるため、引き続き使用することをお勧めします。
</Accordion>
<Accordion title="クラウド データが読み込まれていません。どうすればいいですか？">
これは、小規模なクラウド プロジェクト インスタンスで実行されているデータが多すぎる場合に発生する可能性があります。これを修正するには、**[クラウド] タブ → [概要] → [詳細設定]** に移動し、より大きなインスタンス サイズを選択してください。より大きなインスタンスに切り替え、数分以内に変更すると、データが読み込まれ、クラウド プロジェクトが再び動作するようになります。インスタンスのサイズが大きくなると、クラウドの使用量が増加し、クラウドの使用コストも増加することに注意してください。
</Accordion>
<Accordion title="プロジェクトでクラウドを有効にした後、リージョンを変更できますか?">
いいえ。プロジェクトでクラウドが有効になると、選択したリージョンはロックされ、変更できなくなります。既存のプロジェクトをリージョン間で移動することはできません。
</Accordion>
<Accordion title="クラウドを有効にする場合、リージョンはどのように選択されますか?">
ZenOpus は、現在位置に最も近い地域を自動的に事前選択しますが、確認する前に変更することもできます。
</Accordion>
<Accordion title="すべての新しいプロジェクトにデフォルトのリージョンを設定できますか?">
はい。 **[設定] → [コネクタ] → [共有コネクタ] → ZenOpus Cloud → [権限の管理]** で、新しいクラウド プロジェクトの優先デフォルト リージョンを設定できます。
</Accordion>
</AccordionGroup>
