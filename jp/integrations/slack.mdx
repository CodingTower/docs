[Slack](https://slack.com/) は、ZenOpus アプリがチームのコミュニケーション、チャネル、Workspace データを操作できるようにするメッセージングおよびコラボレーション プラットフォームです。 Slack コネクタを使用すると、アプリの Slack 統合が可能になり、チームへの通知、会話の要約、構造化された更新の Slack への直接送信が可能になります。

Slack を使用すると、アプリで次のことが可能になります。

- チャンネルとダイレクトメッセージにメッセージを送信する
- パブリックおよびプライベート チャネルからメッセージを読み取る (権限付き)
- Workspace ユーザーとチャネル情報をリストする
- 絵文字を使ってメッセージに反応する
- Slack コンテンツの検索 (個人的なつながり)
- チャネルを管理し、Workspace 管理を実行します (管理者権限あり)

各 Slack 接続は、単一の Slack Workspace に対して承認されます。複数の接続を作成して、複数のワークスペース間で作業することができます。

## Slack 接続の仕組み

ZenOpus は、Slack 用に 1 つの集中管理アプリを使用して、ワークスペース間のすべての Slack 統合を強化します。

ZenOpus Workspace 内:

- 複数の Slack 接続を作成できます。
- 各接続は個別の OAuth 認証です。
- 各接続では次のことが可能です。
    - 異なるスコープを使用する
    - さまざまなアクセス タイプ (ボットまたは個人) を持つ
    - 別の Slack Workspace に接続する
- 単一の Workspace 内の複数のプロジェクトは同じ接続を使用できます。

これにより、環境 (開発と運用など) を分離したり、複数の Slack ワークスペースを管理したりすることが簡単になります。

Slack は、ZenOpus のゲートウェイ アーキテクチャを使用して、安全な OAuth 処理と自動トークン更新を実現します。認証と使用制限の詳細については、[ゲートウェイ ベースのコネクタ](/jp/integrations/introduction#ゲートウェイベースのコネクタ) を参照してください。

## Slackへの接続方法

Workspace **管理者** と **所有者** は Slack に接続できます。

接続が作成されると、Workspace 内のすべてのプロジェクトで Slack が利用できるようになります。

### 前提条件

- アプリをインストールできる (または管理者の承認を要求できる) Slack Workspace
- ZenOpus Workspace 管理者または所有者の役割
- 管理者スコープの場合: Slack Workspace 管理者権限

### Slack 接続をセットアップする

<Steps>
<Step title="Slack コネクタに移動します">
**[設定] → [コネクタ] → [共有コネクタ]** に移動し、**Slack** を選択します。
</Step>
<Step title="新しい接続を追加する">
[**接続の追加**] をクリックします。
</Step>
<Step title="接続に名前を付けます">
**表示名** で、接続に名前を付けます (`Slack Prod` など)。
</Step>
<Step title="アクセスタイプを選択してください">
- **ボットとして接続** (推奨): 操作はボットとして実行されます。 Slack Workspace ごとに 1 つのボット接続で十分です。 ZenOpus プロジェクト内でボットの表示名とアイコンをカスタマイズできます。
- **自分として接続** (個人): 操作は、個人の Slack アカウントと権限を使用して、接続を作成する ZenOpus Workspace 管理者として実行されます。これは、個人データへのアクセスまたは Workspace 管理機能が必要な場合に使用します。
</Step>
<Step title="スコープの構成 (オプション)">
[**詳細設定**] を展開して、スコープ (アプリに必要な Slack API 権限) を表示および構成します。デフォルトのスコープは、一般的な使用例に合わせて事前に選択されています。

- **ボット接続**: デフォルトのスコープ Support は、メッセージの送信、チャネルと DM の読み取り、ユーザーの表示、反応の追加、ボットの ID のカスタマイズです。
- **個人 (あなた) 接続**: デフォルトのスコープ Support は、独自の Slack 権限を使用して Slack コンテンツを読み取り、検索します。 Workspace 管理には、追加の管理スコープと Slack Workspace 管理者権限が必要です。

<Tip>
アプリに必要なスコープのみを選択してください。接続を編集して [**再接続**] をクリックすると、後でスコープを更新できます。完全なスコープのドキュメントについては、[Slack のスコープ リファレンス](https://docs.slack.dev/reference/scopes/) を参照してください。
</Tip>
</Step>
<Step title="Slack に接続し、接続を承認する">
- [**接続**] をクリックします。 Slack 認証ウィンドウが開きます。ブラウザーがポップアップをブロックしていないことを確認してください。
- 接続する Workspace を選択し、要求された権限を確認して、[**許可**] をクリックします。確認メッセージとともに ZenOpus にリダイレクトされます。
</Step>
</Steps>

接続すると、ZenOpus アプリは Slack の使用を開始して、メッセージの送信、チャネルの読み取り、チームの Workspace との統合を行うことができます。

## Slack ボットの動作とチャネル アクセス

Slack を **ボット** として接続すると、メッセージは集中管理されている Slack 用 **ZenOpus アプリ**から送信されます。

各プロジェクトは、独自のボット表示名とアイコンを定義できます。これは、すべてのプロジェクトが同じ Slack 用 ZenOpus アプリを通じてメッセージを送信している場合でも、異なるプロジェクトが Slack 内で異なるボットとして表示される可能性があることを意味します。

### ボットのカスタマイズ

ZenOpus にカスタム ボット名とアイコンを設定するように指示することで、プロジェクトからのメッセージが Slack に表示される方法をカスタマイズできます。

たとえば:

```text
ロケットの絵文字アイコンが付いた「Deploy Bot」ボットとしてメッセージを送信します。
```

一貫したボット名とアイコンを使用すると、チームメイトはどのプロジェクトがメッセージを送信しているかをすぐに認識できます。

### パブリックチャネルへのアクセス

ボットは、Workspace 内のすべてのパブリック チャネルに自動的にアクセスできるようになります。招待状は必要ありません。

### プライベートチャンネルアクセス

`/invite @ZenOpus` を使用してメッセージを読み取ったり投稿したりするには、ボットを招待する必要があります。

アプリがプライベート チャネルに投稿できない場合は、まずボットを招待します。

## 制限事項

Slack コネクタでは次のことはできません。

- 着信 Slack イベントを受信します (Webhook やイベント サブスクリプションはありません)
- スラッシュコマンドを作成する
- インタラクティブなボタンまたはモーダルにリアルタイムで応答します
- Support ユーザーごとの Slack ログイン (各エンド ユーザーが自分の Slack に接続)

## プロジェクトの接続を解除する方法

編集者以上は、接続を完全に削除しなくても、接続から特定のプロジェクトを削除できます。接続は他のプロジェクトでも引き続き利用できます。

プロジェクトのリンクを解除するには:

<Steps>
<Step title="コネクタに移動する">
**[設定] → [コネクタ] → [共有コネクタ]** に移動し、**Slack** を選択します。
</Step>
<Step title="接続を開きます">
管理する接続を開きます。
</Step>
<Step title="プロジェクトの選択">
**リンクされたプロジェクト** で、リンクを解除するプロジェクトにチェックを入れます。
</Step>
<Step title="確認する">
[**プロジェクトのリンクを解除**] をクリックして確認します。
</Step>
</Steps>

リンクを解除すると、これらのプロジェクトはこの接続を通じて **Slack** にアクセスできなくなります。プロジェクトで再び **Slack** が必要な場合は、利用可能な接続にリンクできます。

## 接続を削除する方法

Workspace 管理者と所有者は **Slack** 接続を削除できます。

<Warning>
接続の削除は永続的であり、元に戻すことはできません。リンクされているすべてのプロジェクトから資格情報が削除され、この接続を使用しているアプリは新しい接続が追加されるまで動作を停止します。
</Warning>

削除する前に、**リンクされたプロジェクト** セクションを参照して、現在接続を使用しているプロジェクトを確認してください。

接続を削除するには:

<Steps>
<Step title="コネクタに移動する">
**[設定] → [コネクタ] → [共有コネクタ]** に移動し、**Slack** を選択します。
</Step>
<Step title="接続を開きます">
削除する接続を開きます。
</Step>
<Step title="リンクされたプロジェクトを確認する">
**リンクされたプロジェクト** セクションを確認してください。
</Step>
<Step title="削除">
[**この接続を削除**] で、[**削除**] をクリックして確認します。
</Step>
</Steps>

## AIの開示

ZenOpus は AI を活用したプラットフォームです。 ZenOpus で構築されたアプリは、概要や通知などの AI 支援コンテンツを生成する場合があります。 AI によって生成された出力には不正確な部分が含まれる場合があるため、使用前に確認する必要があります。
