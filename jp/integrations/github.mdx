ZenOpus を使用してビルドすると、コードは ZenOpus プラットフォーム内に存在するため、ほとんどのクリエイターにとって最適です。

独自のコピーが必要な場合、開発者との共同作業を計画している場合、またはプロジェクトを別の場所に移動する場合は、数回クリックするだけですべてを GitHub にエクスポートして同期できます。

## GitHubについて

GitHub は、コードを保存、管理、共有するためのクラウドベースのプラットフォームです。あらゆる変更を追跡し、コラボレーションをサポートし、Vercel や Netlify などの一般的な導入ツールと統合します。これは、開発者向けに構築された **コード用 Google ドライブ** と考えてください。

## ZenOpus を GitHub に接続する理由

ZenOpus プロジェクトを GitHub に接続すると、次のことが可能になります。

- **コードをバックアップします:** プロジェクトを ZenOpus の外に安全に保存します。
- **簡単にコラボレーション:** 開発者を招待し、プル リクエスト、ブランチ、コード レビューを使用します。
- **自動的に同期:** ZenOpus での編集は GitHub に表示され、GitHub での変更はデフォルトのブランチ (`main`) に同期されます。
- **ローカルで作業:** プロジェクトの同期を維持しながら、IDE でクローンを作成、編集、コミットします。
- **どこにでもデプロイ:** セルフホストまたは代替プラットフォームにデプロイします。[セルフホスティング: どこでも ZenOpus Cloud プロジェクトを実行する](/jp/tips-tricks/self-hosting) を参照してください。
- **安全にテストします:** ライブ プロジェクトに変更をマージする前に、**[設定] → [アカウント] → [ラボ]** で実験的な **GitHub ブランチ切り替え**を有効にして機能を開発します。詳細については、[ラボ](/jp/features/labs) を参照してください。

<Note>
ZenOpus を使用するのに GitHub は必要ありません。多くのユーザーは完全に ZenOpus 内でビルドと起動を行います。
</Note>

## 接続する前に

統合がどのように機能するかを必ず理解してください。

- **GitHub アカウントが必要です:** [github.com](http://github.com) で無料で作成します。
- **信頼できる唯一の情報源:** 接続すると、コードは ZenOpus に個別に存在するのではなく、GitHub に存在します。
- **双方向同期:** ZenOpus での編集内容は GitHub に表示され、その逆も同様です。
- **安定したリポジトリ パス:** 接続は、正確な GitHub リポジトリ名、場所、組織、アカウントによって異なります。

<Warning>
接続後に GitHub リポジトリの名前を変更、移動、削除しないでください。そうすると同期が解除され、ZenOpus はプロジェクトを更新できなくなります。
</Warning>

## GitHub 統合をセットアップする

Workspace **管理者および所有者** のみが、アカウントや組織の追加、プロジェクトの接続または切断などの GitHub 統合を管理できます。

GitHub への接続は 3 つの段階で行われます。

1. **GitHub アカウントをリンク** (OAuth 認証)
1. **ZenOpus GitHub アプリ**をアカウントまたは組織にインストールします
1. **個々のプロジェクト**をリポジトリに接続する

### ステップ 1: GitHub アカウントに接続する

この手順では、OAuth 認証を通じて GitHub ID を ZenOpus にリンクします。

1. **[設定] → [コネクタ] → [GitHub]** に移動します。
1. [**GitHub に接続**] をクリックします。
1. GitHub アカウントにサインインするか、**zenopus.dev を承認**して、GitHub ID を確認します。  
次に、プロジェクトを接続するオプションが表示されます。

### ステップ 2: ZenOpus GitHub アプリをインストールする

この手順では、**ZenOpus GitHub アプリ** を GitHub アカウントまたは組織にインストールし、そこでリポジトリを作成および更新する権限を ZenOpus に付与します。

アプリは、個人の GitHub アカウントまたは管理者アクセス権のある任意の組織にインストールできます。

1. **[設定] → [コネクタ] → [GitHub]** に移動します。
1. [**プロジェクトを接続**] をクリックします。
1. [**組織を追加**] をクリックし、**zenopus.dev をインストールして認証する**場所 (個人アカウントまたは組織の 1 つ) を選択します。
1. **すべてのリポジトリ** へのアクセスを許可するか、**選択したリポジトリのみ**へのアクセスを許可するかを選択します。
1. [**インストールと承認**] をクリックして、選択したアカウントまたは組織でリポジトリを作成および更新する権限を ZenOpus に付与します。

### ステップ 3: プロジェクトを GitHub に接続する

この手順では、特定の ZenOpus プロジェクトを GitHub リポジトリにリンクします。各 ZenOpus プロジェクトは **1 つのリンクされたリポジトリ**を持つことができます。

1. **設定 → コネクタ → GitHub** に移動し、**プロジェクトの接続** をクリックするか、プロジェクトの右上隅にある GitHub アイコンをクリックして **GitHub の接続** をクリックします。
1. コードを配置する GitHub 組織またはアカウントを選択します。
1. [**とにかく転送**] をクリックして転送を確認し、双方向同期を設定します。  
プロジェクト用に新しい GitHub リポジトリが作成され、同期が自動的に開始されます。

## プロジェクトを GitHub から切断します

Workspace 管理者と所有者は、プロジェクトの GitHub 設定でいつでもプロジェクトを GitHub から切断できます。

切断する場合:

- **同期が停止します:** ZenOpus と GitHub 間の双方向接続が終了します。
- **リポジトリは GitHub 上に残ります:** すべての履歴とファイルがそのまま残ります。
- **コードは ZenOpus に残ります:** 将来の変更は ZenOpus 内にのみ保存されます。

後で再接続する場合:

- 最新の ZenOpus コードを使用して、**新しい GitHub リポジトリ** が作成されます。
- **元のリポジトリ**は GitHub 上に残りますが、ZenOpus にはリンクされなくなりました。

<Note>
ZenOpus が GitHub リポジトリにアクセスできない場合 (名前変更、移動、削除された場合など)、プロジェクトを切断することはできません。
</Note>

## よくある質問

<AccordionGroup>
<Accordion title="この機能を使用するには GitHub を知る必要がありますか?">
いいえ。接続して認証するだけで、GitHub にコードが安全に保存されます。

ブランチや他の開発者とのコラボレーションなど、より高度な作業を行いたい場合にのみ、GitHub の機能を学ぶ必要があります。
</Accordion>
<Accordion title="「Connect GitHub」をクリックしても何も起こりません。何が問題なのでしょうか?">
おそらくブラウザが GitHub ログイン ポップアップをブロックしたと考えられます。 ZenOpus のポップアップを許可してから、再試行してください。
</Accordion>
<Accordion title="GitHub リポジトリの名前を変更、移動、または削除するとどうなりますか?">
接続が切れてしまいます。 プロジェクトの同期は停止し、ZenOpus では編集できなくなります。

これを修正するには、リポジトリを元の名前で元の場所に復元します。削除した場合は、[GitHub で復元](https://docs.github.com/en/repositories/creating-and-managing-repositories/restoring-a-deleted-repository) できます。
</Accordion>
<Accordion title="GitHub のユーザー名または組織の名前を変更するとどうなりますか?">
接続が切れてしまいます。 プロジェクトの同期は停止し、ZenOpus では編集できなくなります。

これを修正するには、元の GitHub ユーザー名または組織名に戻します。
</Accordion>
<Accordion title="別の GitHub アカウントに切り替えるにはどうすればよいですか?">
リンクされた GitHub アカウントは、一度接続すると変更できません。
</Accordion>
<Accordion title="リポジトリは間違った組織に配置されてしまいました。移動できますか?">
直接ではありません。この回避策を試してください。

1. **プロジェクトをリミックス**して、新しいコピーを作成します。
1. **新しいプロジェクト**を GitHub に接続します。
1. セットアップ中に正しい組織を選択してください。
</Accordion>
<Accordion title="ZenOpus が私のリポジトリにアクセスできません — どうすればよいですか?">
リポジトリの名前を変更、移動、または削除した可能性があります。リポジトリの名前、パス、または所有者に対する変更を復元または元に戻すか、リポジトリが削除されている場合はリポジトリを [復元](https://docs.github.com/en/repositories/creating-and-managing-repositories/restoring-a-deleted-repository) し、プロジェクトが再び動作するようにします。
</Accordion>
<Accordion title="GitHub から切断できますか?">
はい。管理者と所有者は、プロジェクトの GitHub 設定でいつでもプロジェクトを GitHub から切断できます。リポジトリは GitHub 上に残りますが、同期は停止します。後で再接続すると、新しいリポジトリが作成されます。
</Accordion>
<Accordion title="既存の GitHub リポジトリを ZenOpus にインポートできますか?">
いいえ。ZenOpus から GitHub にエクスポートすることのみが可能であり、その逆はできません。
</Accordion>
<Accordion title="私のコミットが ZenOpus に表示されないのはなぜですか?">
ZenOpus は **デフォルト ブランチ** (通常は `main`) のみを同期します。ブランチを `main` または ZenOpus 内の [ブランチの切り替え](/jp/features/labs) にマージします。
</Accordion>
</AccordionGroup>
