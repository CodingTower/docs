---
[Telegram](https://telegram.org/) は、ZenOpus アプリがチャット、グループ、チャネルを通じてユーザーと通信できるようにするメッセージング プラットフォームです。 Telegram コネクタを使用すると、アプリの Telegram 統合が可能になり、Telegram の会話から直接通知の送信、ユーザー メッセージへの応答、ワークフローのトリガーが可能になります。

Telegram ボットはアプリのチャット インターフェイスとして機能し、ユーザーがアプリのインターフェイスを開かずにアップデートを受信したり、アプリの機能を操作したりできるようにします。

Telegram を使用すると、アプリで次のことが可能になります。

- メッセージ、写真、ドキュメント、その他のメディアをユーザーまたはグループに送信します
- ユーザーのメッセージとコマンドを受信して応答します (ポーリング経由)
- インタラクティブなボタンとカスタム キーボードを作成する
- 予約、注文、リクエストの送信などのワークフローを通じてユーザーをガイドします
- チャットを通じてアラート、レポート、リマインダーを配信する
- Telegram グループまたはチャネルに更新をブロードキャストする

Telegram は、アプリがイベントについてユーザーに通知したり、更新をブロードキャストしたり、ユーザーがチャットを通じて直接情報をリクエストできるようにする必要がある場合に最適です。

## 一般的なユースケースとサンプルアプリ

|サンプルアプリ |プロンプトの例 |説明 |
| --- | --- | --- |
| Telegram 通知と注文追跡を備えたオンライン ストア |顧客が注文できるオンライン ストアを構築し、新しい注文が作成されるたびにチームに Telegram メッセージを送信し、顧客が注文番号を Telegram ボットにメッセージして配送状況を確認できるようにします。 | Telegram を通じて店舗活動と顧客の最新情報を管理します。顧客がアプリで注文すると、アプリは注文の詳細を含むメッセージをチームの Telegram チャットに送信します。顧客は Telegram ボットに注文番号をメッセージで送信することもでき、アプリは現在の配送ステータスを返します。 |
| Telegram アラートを備えた API 監視ダッシュボード | API を毎分チェックし、サービスが応答を停止した場合に Telegram アラートを送信する監視ダッシュボードを構築します。 |システムを監視し、問題が発生した場合には Telegram を通じてアラートを送信します。アプリはスケジュールに基づいてサービスの健全性をチェックし、ダウンタイムやエラーが検出された場合には Telegram チャットにメッセージを投稿するため、チームは迅速に対応できます。 |
| Telegram チャットを使用した顧客 Support の受信箱 |顧客が Telegram ボットにメッセージを送信できる Support 受信トレイ アプリを構築すると、その会話が Support ダッシュボードに表示されます。 | Telegram の会話を通じて顧客に Support を提供します。ユーザーが Telegram ボットにメッセージを送信すると、アプリはリクエストを Support システムに記録し、チームはダッシュボードまたは自動応答を通じて応答できます。 |
| Telegram レポートを備えた分析ダッシュボード |毎日の指標を計算し、毎朝チームの Telegram グループに概要を投稿する KPI ダッシュボードを構築します。 | Telegram を通じて定期的なレポートを配信します。このアプリはスケジュールに従ってメトリクスを集計し、書式設定された概要メッセージを Telegram チャットまたはグループに送信するため、チームはダッシュボードを開かずに常に最新情報を入手できます。 |
| Telegram 価格アラートを備えた市場データ トラッカー |暗号通貨または株価を監視し、選択した価格しきい値に達したときに電報メッセージを送信する市場データ トラッカーを構築します。 |追跡された値が変化したときにアラートを送信します。このアプリは API から財務データを監視し、価格が設定されたしきい値を超えたときに Telegram メッセージを送信するため、ユーザーはすぐに反応できます。 |

## Telegram 接続の仕組み

各 Telegram 接続は **ボット トークン** を使用します。これは、[@BotFather](https://t.me/BotFather) を通じて作成された単一の Telegram ボットを表します。

ZenOpus Workspace 内:

- 各接続は **1 つの Telegram ボット**を表します
- 複数のボットを使用する場合は、**複数の接続**を作成できます。
- 複数のプロジェクトで **同じ接続**を使用できます

Telegram は OAuth ではなく **ボット トークン認証モデル**を使用するため、認証フローやトークン更新プロセスはありません。

Telegram は、ZenOpus のゲートウェイ アーキテクチャを使用して、安全な資格情報を処理します。認証と使用制限の詳細については、[ゲートウェイ ベースのコネクタ](/jp/integrations/introduction#ゲートウェイベースのコネクタ) を参照してください。

## 電報の接続方法

Workspace **管理者** と **所有者** は Telegram に接続できます。

接続が作成されると、Workspace 内のすべてのプロジェクトで Telegram が利用できるようになります。

### 前提条件

Telegram を接続する前に、以下があることを確認してください。

- テレグラムアカウント
- [@BotFather](https://t.me/BotFather) を使用して作成された Telegram ボットとボット トークン
- ZenOpus Workspace **管理者または所有者** ロール

### ステップ 1: Telegram ボットを作成し、ボット トークンを取得する

Telegram ボットは、Telegram の公式ボット管理ツールである [@BotFather](https://t.me/BotFather) を通じて作成および管理されます。

ボットを作成してトークンを取得するには:

1. Telegram を開いて **@BotFather** を検索するか、[https://t.me/BotFather](https://t.me/BotFather) にアクセスします。
1. コマンド `/newbot` を送信します。
1. プロンプトに従います。
    - ボットの表示名を選択してください
    - `bot` で終わるユーザー名を選択します (例: `myapp_notify_bot`)
1. BotFather は次のような **ボット トークン**を生成します。
    ```text
    123456789:ABCDefghIJKLMnopQRStuvWXYz1234567
    ```
1. トークンをコピーして安全に保管します。

<Info>
ボット トークンはパスワードのように機能します。安全に保管し、決して公に公開しないでください。次のステップで必要になります。
</Info>

詳細な手順については、Telegram のドキュメントを参照してください: [BotFather](https://core.telegram.org/bots/features#botfather)

### ステップ 2: Telegram を ZenOpus に接続する

1. **[設定] → [コネクタ] → [共有コネクタ]** に移動します。
1. **テレグラム**を選択します。
1. [**接続の追加**] をクリックします。
1. 接続の詳細を入力します。
    - **表示名**: 接続にわかりやすい名前を付けます (`Telegram Prod` など)。
    - **ボット トークン**: @BotFather から受け取ったボット トークンを貼り付けます。
1. [**作成**] をクリックします。

ZenOpus はトークンを検証し、接続を作成します。接続すると、ZenOpus アプリは Telegram メッセージの送受信を開始できるようになります。

## 制限事項

現在、Telegram コネクタは Support を実行しません。

- 受信メッセージを受信するための **Webhook**。代わりに `getUpdates` (ポーリング) を使用して受信メッセージを受信します。
- **ユーザーごとの Telegram ログイン** (各エンド ユーザーが自分のアカウントに接続)

受信メッセージを受信するには、アプリは Telegram Bot API からの更新を定期的に要求する必要があります。

## プロジェクトの接続を解除する方法

編集者以上は、接続を完全に削除しなくても、接続から特定のプロジェクトを削除できます。接続は他のプロジェクトでも引き続き利用できます。

プロジェクトのリンクを解除するには:

<Steps>
<Step title="コネクタに移動する">
**[設定] → [コネクタ] → [共有コネクタ]** に移動し、**Telegram** を選択します。
</Step>
<Step title="接続を開きます">
管理する接続を開きます。
</Step>
<Step title="プロジェクトの選択">
**リンクされたプロジェクト** で、リンクを解除するプロジェクトにチェックを入れます。
</Step>
<Step title="確認する">
[**プロジェクトのリンクを解除**] をクリックして確認します。
</Step>
</Steps>

リンクを解除すると、これらのプロジェクトはこの接続を通じて **Telegram** にアクセスできなくなります。プロジェクトで再び **Telegram** が必要な場合は、利用可能な接続にリンクできます。

## 接続を削除する方法

Workspace 管理者と所有者は **Telegram** 接続を削除できます。

<Warning>
接続の削除は永続的であり、元に戻すことはできません。リンクされているすべてのプロジェクトから資格情報が削除され、この接続を使用しているアプリは新しい接続が追加されるまで動作を停止します。
</Warning>

削除する前に、**リンクされたプロジェクト** セクションを参照して、現在接続を使用しているプロジェクトを確認してください。

接続を削除するには:

<Steps>
<Step title="コネクタに移動する">
**[設定] → [コネクタ] → [共有コネクタ]** に移動し、**Telegram** を選択します。
</Step>
<Step title="接続を開きます">
削除する接続を開きます。
</Step>
<Step title="リンクされたプロジェクトを確認する">
**リンクされたプロジェクト** セクションを確認してください。
</Step>
<Step title="削除">
[**この接続を削除**] で、[**削除**] をクリックして確認します。
</Step>
</Steps>
