---
title: "アプリを Contentful に接続する"
description: "アプリを Contentful に接続して、エントリ、アセット、リッチ テキストなどの公開コンテンツをヘッドレス CMS から取得して表示します。"
---
[Contentful](https://www.contentful.com/) は、コンテンツ管理とコンテンツの表示方法を分離するヘッドレス CMS です。 Contentful コネクタを使用すると、ZenOpus アプリは、Content Delivery API (CDA) を使用して、Contentful スペースから公開されたコンテンツを取得できます。その後、アプリは、Contentful に保存されているコンテンツを利用して、コンテンツ主導のページ、製品カタログ、ブログ、またはドキュメントをレンダリングできます。

Contentful を使用すると、アプリで次のことが可能になります。

- 公開されたエントリ、アセット、コンテンツ タイプをスペースから取得します
- Contentful の CDN をサポートする Content Delivery API を通じてコンテンツを配信します
- CMS コンテンツを活用した動的なページとレイアウトを構築
- コンテンツ タイプ、ロケール、フィールド値によるエントリのフィルタリングとページネーション
- アプリ UI でリッチ テキスト フィールドをレンダリングする

<Note>
Contentful コネクタは **読み取り専用**です。 Support コンテンツの作成、更新、削除は行いません。すべてのコンテンツ管理は、Contentful Web アプリまたは Contentful Management API を通じて行われます。
</Note>

## 一般的なユースケースとサンプルアプリ

|サンプルアプリ |プロンプトの例 |説明 |
| --- | --- | --- |
|マーケティングおよびランディング ページ サイト | Contentful を使用して、ホームページと特集ページが Contentful から読み込まれるマーケティング サイトを構築します。 | CMS コンテンツを活用したマーケティング ページを構築します。このアプリは、Contentful からページ エントリを取得し、コンポーネントとしてレンダリングするため、編集者はコードを変更せずにテキストや画像を更新できます。 |
|製品カタログアプリ | Contentful を使用して、カテゴリ フィルターを使用して Contentful スペースに保存されている製品を表示する製品カタログを作成します。 | Contentfulで管理されている商品リストを表示します。このアプリは、製品エントリを取得し、製品カードをレンダリングし、CMS に保存されているフィールドに基づいてフィルタリングとページネーションをサポートします。 |
|ブログと編集プラットフォーム | Contentful を使用して、Contentful から記事を読み込み、リッチ テキストと画像をレンダリングするブログを構築します。 |コンテンツ豊富なコンテンツに裏打ちされたブログを作成します。アプリは記事エントリを取得し、リンクされたアセットを解決し、リッチ テキスト フィールドを書式設定されたコンテンツとしてレンダリングします。 |
|ドキュメント サイト | Contentful を使用して、各ドキュメント ページが Contentful のエントリとして保存されるドキュメント サイトを構築します。 | CMS コンテンツに基づいてドキュメントを作成します。アプリはドキュメント エントリを取得し、Contentful のコンテンツ構造からナビゲーションとページを生成します。 |
|多言語コンテンツアプリ | Contentful を使用し、Contentful ロケールを使用してユーザーの言語でコンテンツを読み込むアプリを構築します。 | Contentful からローカライズされたコンテンツを提供します。アプリは、要求されたロケールのエントリを取得し、ユーザーの言語に基づいて翻訳されたコンテンツをレンダリングします。 |
|イベントおよびお知らせページ | Contentful を使用して、今後のイベントを Contentful から読み込み、日付順に並べ替えるイベント リスト ページを作成します。 | Contentfulで管理されているイベント一覧を表示します。アプリはイベント エントリを取得し、日付フィールドで並べ替えて、編集者が CMS で管理する今後のイベントをレンダリングします。 |

## 接続方法

Workspace **管理者** と **所有者** は Contentful に接続できます。

異なる API キーまたは異なるスペースを使用して、複数の Contentful 接続を作成できます。これは、開発環境と運用環境を分離したり、複数の Contentful スペースを接続したりする場合に役立ちます。

接続が作成されると、Workspace 内のすべてのプロジェクトで Contentful が使用できるようになります。

### 前提条件

Contentful に接続する前に、以下があることを確認してください。

- 公開されたコンテンツを含むスペースを持つコンテンツフルなアカウント
- そのスペースの **Content Delivery API (CDA) アクセス トークン**
- あなたの**スペースID**
- ZenOpus Workspace 所有者または管理者の役割

<Note>
このコネクタを通じて行われるすべての API リクエストは、Content Delivery API アクセス トークンを使用します。 API の使用量は Contentful プランの制限にカウントされ、請求は ZenOpus ではなく Contentful によって直接処理されます。
</Note>

### ステップ 1: スペース ID を見つけて CDA アクセス トークンを作成する

スペース ID と CDA アクセス トークンはどちらも Contentful Web アプリで利用できます。

**スペース ID を確認するには:**

1. [app.contentful.com](https://app.contentful.com) にログインします。
1. 接続したいスペースを開きます。
1. **[設定] → [一般設定]** に移動します。
1. **スペース ID** をコピーします。

詳細な手順については、充実したドキュメントを参照してください: [スペース ID の検索](https://www.contentful.com/help/spaces/find-space-id)

**コンテンツ配信 API アクセス トークンを作成するには:**

1. 同じスペースで、**設定 → API キー** に移動します。
1. [**API キーの追加**] をクリックするか、既存のキーを開きます。
1. キーに `ZenOpus integration` などの名前を付けます。
1. **コンテンツ デリバリー API アクセス トークン**をコピーします。

詳細な手順については、充実したドキュメントを参照してください: [認証](https://www.contentful.com/developers/docs/references/authentication/)

<Warning>
アクセス トークンはパスワードのように機能します。安全に保管し、決して公開しないでください。次のステップでは、スペース ID とアクセス トークンの両方が必要になります。
</Warning>

### ステップ 2: Contentful を ZenOpus に接続する

1. **[設定] → [コネクタ] → [共有コネクタ]** に移動し、**コンテンツフル** を選択します。
1. [**接続の追加**] をクリックします。
1. **表示名** (例: `Contentful Prod`) を入力します。
1. **スペース ID** を入力してください。
1. **Content Delivery API アクセス トークン**を入力します。
1. [**接続**] をクリックします。

接続すると、ZenOpus アプリは Contentful スペースからコンテンツの取得を開始できます。

## プロジェクトの接続を解除する方法

編集者以上は、接続を完全に削除しなくても、接続から特定のプロジェクトを削除できます。接続は他のプロジェクトでも引き続き利用できます。

プロジェクトのリンクを解除するには:

<Steps>
<Step title="コネクタに移動する">
**[設定] → [コネクタ] → [共有コネクタ]** に移動し、**コンテンツフル** を選択します。
</Step>
<Step title="接続を開きます">
管理する接続を開きます。
</Step>
<Step title="プロジェクトの選択">
**リンクされたプロジェクト** で、リンクを解除するプロジェクトにチェックを入れます。
</Step>
<Step title="確認する">
[**プロジェクトのリンクを解除**] をクリックして確認します。
</Step>
</Steps>

リンクを解除すると、これらのプロジェクトはこの接続を通じて **Contentful** にアクセスできなくなります。プロジェクトに再び**コンテンツ**が必要な場合は、利用可能な接続にプロジェクトをリンクできます。

## 接続を削除する方法

Workspace 管理者と所有者は **コンテンツフル** 接続を削除できます。

<Warning>
接続の削除は永続的であり、元に戻すことはできません。リンクされているすべてのプロジェクトから資格情報が削除され、この接続を使用しているアプリは新しい接続が追加されるまで動作を停止します。
</Warning>

削除する前に、**リンクされたプロジェクト** セクションを参照して、現在接続を使用しているプロジェクトを確認してください。

接続を削除するには:

<Steps>
<Step title="コネクタに移動する">
**[設定] → [コネクタ] → [共有コネクタ]** に移動し、**コンテンツフル** を選択します。
</Step>
<Step title="接続を開きます">
削除する接続を開きます。
</Step>
<Step title="リンクされたプロジェクトを確認する">
**リンクされたプロジェクト** セクションを確認してください。
</Step>
<Step title="削除">
[**この接続を削除**] で、[**削除**] をクリックして確認します。
</Step>
</Steps>
