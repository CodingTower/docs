title: "アプリを Twilio に接続する"
description: "アプリを Twilio に接続して、SMS および MMS メッセージを送信したり、音声通話を行ったり、電話ベースの通信フローを ZenOpus アプリに直接追加したりできます。"
---
[Twilio](https://www.twilio.com/) は、アプリが電話でユーザーに連絡できるようにするクラウド コミュニケーション プラットフォームです。これは、リアルタイムのテキスト更新の送信、電話認証の追加、音声通話のトリガー、または Support メッセージング ベースのワークフローを必要とするアプリに適しています。

Twilio を使用すると、アプリで次のことが可能になります。

- SMS および MMS メッセージの送受信
- 音声通話の発信と管理
- Support WhatsApp メッセージング フロー
- 電話番号を検証して検索する

Twilio は、アプリ内通知や電子メールだけではなく、電話やテキストでアプリがユーザーと通信する必要がある場合に適しています。

## 一般的なユースケースとサンプルアプリ

|サンプルアプリ |プロンプトの例 |説明 |
| --- | --- | --- |
| SMS 通知とアラート |注文の発送時に追跡リンクを含むテキストを顧客に送信する注文追跡アプリを構築します。 |アプリ内のイベントによってトリガーされるリアルタイムの SMS 更新を送信します。アプリはステータスの変化を検出し、メッセージを生成し、顧客に自動的に送信します。 |
|電話認証フロー |アカウントを作成する前に電話番号を確認するために、SMS でユーザーにワンタイム コードを送信するサインアップ フローを構築します。 |電話ベースの認証をオンボーディングまたはログイン フローに追加します。アプリはワンタイム コードを送信し、ユーザーの入力をチェックして、続行する前に番号を確認します。 |
|予約のリマインダー |予約の 24 時間前に顧客にテキスト メッセージを送信し、時間が変更された場合には再度テキスト メッセージを送信する予約アプリを構築します。 |アプリ内のスケジュールされたイベントに基づいて自動リマインダー メッセージを送信します。このアプリは予約データを使用して、手動でフォローアップすることなく、適切なタイミングでテキスト メッセージを送信します。 |
|音声通話の最新情報 |技術者が出張中のときに、録音されたメッセージで顧客に電話をかけるサービス アプリを構築します。 |重要な更新に直ちに対応する必要がある場合、アプリから音声通話を発信します。アプリは電話をかけ、音声メッセージをユーザーに配信します。 |
| WhatsApp Support 受信箱 |ユーザーが WhatsApp でビジネスにメッセージを送信でき、エージェントがアプリから返信できる顧客 Support アプリを構築します。 | WhatsApp を製品の通信チャネルとして追加します。アプリは Twilio を通じてメッセージを送受信するため、ワークフロー内で会話を処理できます。 |
|電話番号の検証 |電話番号をデータベースに保存する前に、電話番号が有効かどうかを確認するリード獲得フォームを作成します。 |電話番号を保存または使用する前に、電話番号を検証してフォーマットします。アプリは Twilio を通じて番号をチェックするため、ワークフローはよりクリーンな連絡先データから開始されます。 |

## Twilio への接続方法

Workspace **管理者** と **所有者** は Twilio に接続できます。

たとえば、開発環境と運用環境を分離したり、異なる Twilio アカウントを管理したりするために、異なる API キーを使用して複数の Twilio 接続を作成できます。

接続が作成されると、Workspace 内のすべてのプロジェクトで Twilio が利用できるようになります。

### 前提条件

Twilio に接続する前に、以下があることを確認してください。

- Twilio アカウント
- ユースケースで SMS、MMS の送信、または通話が必要な場合は、Twilio 電話番号
- **標準** Twilio API キー
- ZenOpus Workspace 所有者または管理者の役割

<Note>
このコネクタを介したすべての SMS、MMS、音声、WhatsApp、およびその他の Twilio の使用量は、Twilio アカウントとプランに基づいて Twilio から直接請求されます。請求は ZenOpus ではなく Twilio によって処理されます。
</Note>

### ステップ 1: Twilio API キーを作成する

Twilio は、単一のトークンの代わりに **API キー SID** と **シークレット** を使用します。 API キーは個別に取り消すことができるため、メインの認証トークンを使用するよりも安全です。

Twilio 標準 API キーを作成するには:

1. [Twilio コンソール](https://console.twilio.com/) に移動し、サインインします。
1. **[アカウント] → [API キーとトークン]** に移動します。
1. [**API キーの作成**] をクリックします。
1. **フレンドリ名** (`ZenOpus integration` など) を入力します。
1. キーの **リージョン** を選択します。
    - **米国 (US1)**: ほとんどのアカウントのデフォルト
    - **アイルランド (IE1)**
    - **オーストラリア (AU1)**
1. **キー タイプ** を **標準** に設定したままにします。
1. [**API キーの作成**] をクリックします。
1. **API キー SID** (`SK` で始まる) と **シークレット** の両方をコピーします。シークレットは一度だけ表示されるため、ページを離れる前に安全に保管してください。

<Info>
API キー シークレットはパスワードのように機能します。安全に保管し、決して公開しないでください。次のステップで必要になります。
</Info>

**アカウント SID** (`AC` で始まる) も必要になります。これは Twilio コンソールで入手できます。 Twilio API 資格情報は地域固有であるため、キーの作成時に選択した地域は、ZenOpus 接続の設定時に使用する地域と一致する必要があります。

詳細な手順については、Twilio のドキュメントを参照してください: [Twilio コンソールで API キーを作成する](https://www.twilio.com/docs/iam/api-keys/keys-in-console)

### ステップ 2: Twilio を ZenOpus に接続する

1. **[設定] → [コネクタ] → [共有コネクタ]** に移動し、**Twilio** を選択します。
1. [**接続の追加**] をクリックします。
1. **表示名** に、接続の名前 (`Twilio Prod` など) を入力します。
1. [**リージョン**] で、API キーに使用されているのと同じ Twilio リージョンを選択します。
    - **米国 (US1)**
    - **アイルランド (IE1)**
    - **オーストラリア (AU1)**
1. **アカウント SID** に、Twilio アカウント SID (`AC` で始まる) を入力します。
1. **標準 API キー SID** に、作成した API キー SID (`SK` で始まる) を入力します。
1. **API キー シークレット** に、キーの作成時に表示されたシークレットを入力します。
1. [**作成**] をクリックします。

接続すると、ZenOpus アプリは Twilio を使用してメッセージを送信し、電話ベースの通信フローを強化できます。

## プロジェクトの接続を解除する方法

編集者以上は、接続を完全に削除しなくても、接続から特定のプロジェクトを削除できます。接続は他のプロジェクトでも引き続き利用できます。

プロジェクトのリンクを解除するには:

<Steps>
<Step title="コネクタに移動する">
**[設定] → [コネクタ] → [共有コネクタ]** に移動し、**Twilio** を選択します。
</Step>
<Step title="接続を開きます">
管理する接続を開きます。
</Step>
<Step title="プロジェクトの選択">
**リンクされたプロジェクト** で、リンクを解除するプロジェクトにチェックを入れます。
</Step>
<Step title="確認する">
[**プロジェクトのリンクを解除**] をクリックして確認します。
</Step>
</Steps>

リンクを解除すると、これらのプロジェクトはこの接続を通じて **Twilio** にアクセスできなくなります。プロジェクトで再び **Twilio** が必要な場合は、利用可能な接続にプロジェクトをリンクできます。

## 接続を削除する方法

Workspace 管理者と所有者は **Twilio** 接続を削除できます。

<Warning>
接続の削除は永続的であり、元に戻すことはできません。リンクされているすべてのプロジェクトから資格情報が削除され、この接続を使用しているアプリは新しい接続が追加されるまで動作を停止します。
</Warning>

削除する前に、**リンクされたプロジェクト** セクションを参照して、現在接続を使用しているプロジェクトを確認してください。

接続を削除するには:

<Steps>
<Step title="コネクタに移動する">
**[設定] → [コネクタ] → [共有コネクタ]** に移動し、**Twilio** を選択します。
</Step>
<Step title="接続を開きます">
削除する接続を開きます。
</Step>
<Step title="リンクされたプロジェクトを確認する">
**リンクされたプロジェクト** セクションを確認してください。
</Step>
<Step title="削除">
[**この接続を削除**] で、[**削除**] をクリックして確認します。
</Step>
</Steps>
