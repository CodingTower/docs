description: "AI を活用した検索エンジンである Perplexity にアプリを接続し、検証可能な引用を含むリアルタイムの情報を取得します。"
---
[Perplexity](https://www.perplexity.ai/) は、ライブ Web データに基づいてソースに裏付けられた簡潔な回答を返す AI 検索プラットフォームです。これは、静的な知識やトレーニングのみの知識ではなく、**新鮮で信頼できる情報**を必要とするアプリ向けに設計されています。

Perplexity を使用すると、アプリで次のことが可能になります。

- Web からの最新情報を使用して質問に回答します
- 明確で検証可能な引用を含む要約を生成する
- 企業、製品、市場をリアルタイムで比較
- 手作業によるリサーチを数秒に短縮

Perplexity は、アプリがニュース、調査、市場情報、事実確認などの**ソースに裏付けされた最新の情報**に依存している場合に適しています。

## 一般的なユースケースとサンプルアプリ

|サンプルアプリ |プロンプトの例 |説明 |
| --- | --- | --- |
|販売準備アシスタント | Perplexity を使用して、各電話の前に会社のコンテキストを収集する販売準備アシスタントを構築します。 |ライブ検索を使用して、会社概要、最近のニュース、リーダーの交代、リスクシグナルを取得します。このアプリは、会議の前に Perplexity クエリを実行し、営業担当者に引用された簡潔な説明を提供します。 |
|競合他社および市場調査ツール | Perplexity を使用して競合他社の調査ツールを構築します。 |資金調達イベント、製品の発売、ポジショニングの変更、市場活動を要約します。アプリは複数の Perplexity 検索を並行して実行し、結果をリサーチ ダッシュボードに集約します。 |
|規制または政策のトラッカー | Perplexity を使用して規制の更新を追跡するアプリを構築します。 |最近の情報源に基づいて、新しい判決、ポリシーの変更、またはコンプライアンスの更新を監視します。このアプリは定期的に Perplexity にクエリを実行し、新しく公開または更新された情報を引用とともに表示します。 |
|ファクトチェック アプリ (「これは本当ですか?」) | Perplexity 検索を使用してファクトチェック アプリを構築します。 |裏付けとなる情報源と矛盾する情報源を検索して、主張を検証します。アプリはステートメントを入力として受け入れ、引用された参考文献を含む合成回答を返します。 |
|研究助手 | Perplexity を使用して、自由回答型の質問に答えるリサーチ アシスタントを構築します。 | 「今週この業界で何が起きましたか?」などの質問に答えます。または「X のトップの競争相手は誰ですか?」このアプリは Perplexity を使用して、構造化された引用された応答を生成します。 |

## Perplexityの接続方法

Workspace **管理者** と **所有者** は Perplexity に接続できます。

異なる API キーを使用して複数の Perplexity 接続を作成できます。これは、環境 (開発と運用など) を分離したり、レート制限を管理したりするのに役立ちます。

接続が作成されると、Workspace 内のすべてのプロジェクトで Perplexity が使用できるようになります。

### 前提条件

Perplexity を接続する前に、以下があることを確認してください。

- 請求と API キー管理のための管理者権限を持つ Perplexity アカウント
- Perplexity API キー
- ZenOpus Workspace 所有者または管理者の役割

<Note>
API にアクセスできるアクティブな Perplexity アカウントが必要です。このコネクタを通じて実行されるすべての検索および回答リクエストは Perplexity API の使用量を消費し、請求は ZenOpus ではなく Perplexity によって直接処理されます。
</Note>

### ステップ 1: Perplexity アカウントと API キーを作成する

API キーを使用すると、Perplexity API に接続し、その機能を使用できます。 Perplexity では複数の API キーを作成できます。

Perplexity API キーを作成するには:

1. [Perplexity](https://www.perplexity.ai/) に移動し、アカウントを設定します。
1. **アカウント (プロフィール アイコン) → API** に移動します。
1. **API グループ** に移動して、組織を設定します。組織の名前、住所、税金の詳細を入力します。
1. **API 請求** に移動し、**支払い方法の追加** をクリックしてクレジット カード情報を入力します。
1. **API キー** に移動し、**キーの作成** をクリックします。キーにわかりやすい名前 (`ZenOpus integration` など) を付けます。
1. 生成された API キーをコピーし、安全に保管します。
    <Info>
    API キーはパスワードのように機能します。安全に保管し、決して公開しないでください。次のステップで必要になります。
    </Info>

詳細な手順については、Perplexity のドキュメントを参照してください: [API グループと請求](https://docs.perplexity.ai/getting-started/api-groups)。

### ステップ 2: Perplexity を ZenOpus に接続する

異なる API キーを使用して複数の接続を作成できます。

Perplexity に接続するには:

1. **[設定] → [コネクタ] → [共有コネクタ]** に移動し、**Perplexity** を選択します。
1. [**接続の追加**] をクリックします。
1. **表示名**: 接続に名前を付けます (例: `Perplexity Prod`)。
1. **認証** で、Perplexity API キー (`pplx-xxxx…`) を入力します。
1. [**作成**] をクリックします。

接続すると、ZenOpus アプリは Perplexity の使用を開始して、リアルタイムの引用情報を取得できます。

## プロジェクトの接続を解除する方法

編集者以上は、接続を完全に削除しなくても、接続から特定のプロジェクトを削除できます。接続は他のプロジェクトでも引き続き利用できます。

プロジェクトのリンクを解除するには:

<Steps>
<Step title="コネクタに移動する">
**[設定] → [コネクタ] → [共有コネクタ]** に移動し、**Perplexity** を選択します。
</Step>
<Step title="接続を開きます">
管理する接続を開きます。
</Step>
<Step title="プロジェクトの選択">
**リンクされたプロジェクト** で、リンクを解除するプロジェクトにチェックを入れます。
</Step>
<Step title="確認する">
[**プロジェクトのリンクを解除**] をクリックして確認します。
</Step>
</Steps>

リンクを解除すると、これらのプロジェクトはこの接続を通じて **Perplexity** にアクセスできなくなります。プロジェクトで再び **Perplexity** が必要な場合は、利用可能な接続にプロジェクトをリンクできます。

## 接続を削除する方法

Workspace 管理者と所有者は **Perplexity** 接続を削除できます。

<Warning>
接続の削除は永続的であり、元に戻すことはできません。リンクされているすべてのプロジェクトから資格情報が削除され、この接続を使用しているアプリは新しい接続が追加されるまで動作を停止します。
</Warning>

削除する前に、**リンクされたプロジェクト** セクションを参照して、現在接続を使用しているプロジェクトを確認してください。

接続を削除するには:

<Steps>
<Step title="コネクタに移動する">
**[設定] → [コネクタ] → [共有コネクタ]** に移動し、**Perplexity** を選択します。
</Step>
<Step title="接続を開きます">
削除する接続を開きます。
</Step>
<Step title="リンクされたプロジェクトを確認する">
**リンクされたプロジェクト** セクションを確認してください。
</Step>
<Step title="削除">
[**この接続を削除**] で、[**削除**] をクリックして確認します。
</Step>
</Steps>
