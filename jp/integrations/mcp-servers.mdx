**MCP (Model Context Protocol)** は、ZenOpus などの AI システムを外部ツール、サービス、データ ソースに接続するオープン スタンダードです。 ZenOpus では、MCP サーバーは **パーソナル コネクタ**として使用されます。

ZenOpus を Notion、Atlassian、Linear などのツールに接続すると、ZenOpus エージェントは実際のチーム コンテキストを使用して、より正確なコード、プロトタイプ、ドキュメントを生成できます。

接続すると、ZenOpus は構造化データ (ドキュメント、問題、図からのテキストなど) を読み取り、サポートされている場合はアイテムの作成や更新などの限定的なアクションを実行できます。

ZenOpus には、すべてのプランにいくつかの事前構築済みパーソナル コネクタが含まれており、すべての有料プランにはカスタム MCP サーバー用の Support が含まれています。

## 個人用コネクタを使用する理由

パーソナル コネクタは、次の 2 つの主要な機能を解放します。

- **既存のコンテキストを ZenOpus に取り込みます。**  
ZenOpus エージェントは、チームのドキュメント、チケット、設計ファイルを読み取り、標準に合わせたプロトタイプとフローを構築できます。
- **ZenOpus アプリをワークフローに接続します。**  
ZenOpus エージェントは、チケット ステータスの更新、プロトタイプ リンクのコメントとしての追加、サポートされている場合の新しいアイテムの作成など、接続されたツールでフォローアップ アクションを実行できます。

ZenOpus は、チームの世界を理解するとさらに強力になり、推測を排除し、イテレーションを高速化して、**アイデアからアプリをこれまでよりも早く実用化する**ことができます。

|アトラシアン | Jira の課題と Confluence ページにアクセスします。 |製品ドキュメントを実用的なプロトタイプに変えます。 ZenOpus は、製品ドキュメントとリンクされた Jira エピック (PRD、仕様、アーキテクチャ ドキュメント) を読み取り、チームの標準に一致するプロトタイプを生成できます。生成されたプロトタイプを Confluence に埋め込んでフィードバックを得ます。 |
|グラノーラ | Granola の会議メモ、決定事項、アクション アイテムにアクセスします。 |会議のコンテキストを仕様書とチケットに変換します。 ZenOpus は、メモを調べなくても、過去の会議を検索し、決定事項や実行項目を引き出し、合意内容を使用して PRD、ユーザー ストーリー、またはフォローアップ タスクを生成できます。 |
|リニア | Linear の問題とプロジェクト データにアクセスします。 |実際の問題とチケットからプロトタイプを作成します。 ZenOpus は、説明や受け入れ基準を含む線形の問題を読み取り、機能的なプロトタイプを生成できます。結果として得られたアプリをレビューのために Linear に戻します。 |

### 事前に構築された個人用コネクタを追加する方法

一般的なサードパーティのアプリやサービスとの事前構築済み統合を使用するには、次の手順を実行します。

1. **[設定] → [コネクタ] → [個人用コネクタ]** に移動します。
1. **パーソナル コネクタを選択します** (例: Notion または Linear)。
1. [**接続**] をクリックします。サインインし、ZenOpus にアカウントへのアクセスを許可します。

### 特定の個人用コネクタの追加セットアップ

[n8n](https://docs.n8n.io/advanced-ai/accessing-n8n-mcp-server)

これらの手順に加えて、n8n で次のことを行う必要があります。

- **[設定] → [MCP アクセス]** に移動し、[**MCP アクセスを有効にする**] を切り替えます (インスタンス所有者または管理者の権限が必要です)
- 公開するワークフローごとに、ワークフロー エディターの **設定** に移動し、**MCP で使用可能** を切り替えます。

[振幅](https://amplitude.com/)

ZenOpus で構築されたアプリを分析する予定がある場合は、MCP コネクタを使用する前に、すでにデータが Amplitude に送信されていることを確認してください。セットアップ手順については、[Amplitude クイックスタート ガイド](https://amplitude.com/docs/get-started/amplitude-quickstart) を参照してください。

## カスタムMCPサーバー（有料プラン）

完全な柔軟性を得るために、**カスタム MCP サーバー**を個人コネクタとして接続し、内部またはサードパーティのシステムを ZenOpus に取り込むことができます。これは、プライベート API、内部 CRM、またはカスタム データ ソースに最適です。

ZenOpus と互換性のある高品質のリモート MCP サーバーの厳選されたリストである [リモート MCP サーバー](https://mcpservers.org/remote-mcp-servers) を調べてください。

### カスタム MCP サーバーに接続する方法

カスタムまたは内部 MCP サーバーに接続するには:

1. **[設定] → [コネクタ] → [個人用コネクタ] に移動します。**
1. [**新しい MCP サーバー**] をクリックします。
1. **サーバー名**: サーバーの明確な名前を入力します (*内部 CRM* または *Analytics API* など)。
1. **サーバー URL**: ZenOpus が MCP サーバーに到達できるアドレスを入力します。
1. **認証:**
    - **OAuth (デフォルト):** ZenOpus はデフォルトで OAuth を使用します。 [**追加と承認**] をクリックすると、承認を求めるメッセージが表示されます。
    - **ベアラー トークンまたは API キー:** Support OAuth を使用しないサーバーの場合は、ベアラー トークンまたは API キーを使用して認証できます。
    - **認証なし**: サーバーが認証を必要としない場合は、資格情報なしで接続します。
1. **サーバーを追加**。カスタム MCP サーバーが個人コネクタのリストに表示され、ビルドのコンテキスト データを提供できるようになりました。

## 実践例

接続された各アプリは、ZenOpus エージェントのコンテキストとして即座に使用できるようになります。自然なプロンプトを使用して、接続されたツールを操作できます。

|シナリオ |プロンプトの例 |結果の例 |
| --- | --- | --- |
|プロダクト マネージャーは Linear または Jira を接続します |現在のスプリントのチケットに基づいてプロトタイプを生成します。 | ZenOpus は、問題の説明と受け入れ基準を読み取り、調整されたテスト可能な UI を構築します。 |
|マーケターが Notion または Confluence をリンクする |第 4 四半期のキャンペーン概要をランディング ページに変えます。 | ZenOpus は、ドキュメントをコンテキストとして使用して、ブランドに基づいた編集可能なページを生成します。 |
|デザイナーがミロをつなぐ |ホワイトボードからサインアップ フローのプロトタイプを作成します。 | ZenOpus は、UI ナビゲーションの足場となるフロー図を読み取り、チームがテストできるインタラクティブなプロトタイプに変換します。 |
|エンジニアは n8n または内部 MCP をリンクします。ユーザーをリストし、削除できるワークフロー UI を構築します。 | ZenOpus は、ライブ データとワークフローを使用するアプリを構築します。 |

## セキュリティと権限

パーソナル コネクタ (MCP サーバー接続) は **ユーザーごとの接続**です。 **[設定] → [コネクタ]**でいつでも確認または取り消しできます。

プロジェクト内にすばやくアクセスするには、プロンプト ボックスで `+` をクリックし、**コネクタ** に移動します。

**Business** および **Enterprise** プランでは、Workspace **管理者および所有者** は、すべてのユーザーの個人コネクタとして使用できるサーバーを管理できます。

- **[設定] → [プライバシーとセキュリティ]** に移動して、Workspace 全体のすべての **MCP サーバー アクセス**を有効または無効にします。
- **[設定] → [コネクタ] → [個人コネクタ] → [管理設定]** に移動して、個々の MCP サーバーを個人コネクタとして有効または無効にします。
