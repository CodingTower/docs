ZenOpus では、**チャット** を通じて Stripe を完全にセットアップできるようになりました。

**チャットによる自動セットアップ (推奨)**  

 **Supabase** に接続し、**API キーの追加** を介して **Stripe Secret Key** を保存した後、必要なことを説明します。

- 「サブスクリプション層を 3 つ追加します…」
- 「電子書籍の 1 回限りのチェックアウトを 29 ドルで作成してください」  
ZenOpus は、チェックアウト / ポータル エッジ機能、RLS を備えたデータベース テーブル、および UI ボタンを生成します。要求しない限り、手動のコーディングや Webhook は必要ありません。

<Note>
- **一回限りの販売**の場合は、カートまたは商品ページがすでに機能していることを確認してください。
- **サブスクリプション**の場合は、ZenOpus が Stripe の顧客を各ユーザーの **ID** にリンクできるように、Supabase Auth が設定されていることを確認してください。
</Note>

### 重要なポイント

- サブスクリプションと 1 回限りの支払いの両方に **チャット主導のフロー** を使用します。
- **Stripe 秘密キーをチャットに貼り付けないでください。** チャット内の **API キーの追加** フォームを使用して構成してください。
- **Webhook はオプトインです。** Webhook を要求しない限り、ZenOpus はエッジ関数ポーリングに依存します。
- **ブラウザ コンソール → ネットワーク/エラー**、**Supabase → Edge Functions → ログ**、**ストライプ ダッシュボード → ログ** でデバッグします。
- 常に **ストライプ テストモード** でテストしてからデプロイしてください。

## 要件

Stripe を統合する前に、次の前提条件が満たされていることを確認してください。

- プロジェクトは**必要**、Supabaseに接続されています。 [Supabase について詳しく見る](/jp/integrations/supabase)
- 製品が適切に設定された **Stripe アカウント**。
- 動作する**フロントエンドとバックエンド**:
    - 個別の製品を販売する場合は、ショッピング カートとチェックアウト ページが機能していることを確認してください。
    - サブスクリプションの場合は、ログイン機能とさまざまな価格帯を設定します。

<Note>
**注意してください** Stripe 統合はプレビューでは機能しません。統合をテストするには、必ずデプロイしてください。また、機能を試すときは、Stripe でテスト モードになっていることを確認してください。支払いをテストする場合、カード番号: 4242 4242 4242 4242、CVC としての任意の 3 桁と将来の日付がカードとして機能します。
</Note>

## Stripe 支払いの設定 (ノーコード チャット フロー)

ZenOpus はすべての Stripe ロジックを生成するようになりました。チャット内の **API キーの追加** フォームで **Stripe Secret Key** を設定し、プロジェクトを **Supabase** に接続したら、チャットで必要なものを ZenOpus に伝えるだけです。手動での支払いリンクは必要ありません。

<Steps>
<Step title="ステップ1">
**プロジェクトを準備します**

- スーパーベースが接続されました
- チャット内の **API キーの追加** フォーム経由で追加された Stripe 秘密キー
- (オプション) 価格または製品 ID が便利です
</Step>
<Step title="ステップ2">
例:

- 29 ドルの「デジタル コース」の 1 回限りのチェックアウトを作成します
- 各ユーザーの ID に関連付けられた年間プレミアム プランを 99 ドルで設定します
</Step>
<Step title="ステップ3">
**確認して適用** ZenOpus は、Edge Functions、データベース テーブル、および UI コンポーネント (すべて Supabase のユーザー ID に関連付けられています) を自動スキャフォールディングします。プレビューを確認し、[**適用**] をクリックして展開します。
</Step>
</Steps>

<Note>
- 安全なロールベースのアクセスを実現するために、サブスクリプションは常に Supabase 内の認証されたユーザーの `id` にリンクされる必要があります。
</Note>

## 高度な統合: Webhook と Supabase

**サブスクリプションやロールベースのアクセス**などの複雑な支払い構造の場合、ZenOpus では **Supabase** を使用して Stripe 統合を安全に処理することをお勧めします。これにより、適切な Webhook 処理、サブスクリプション管理、支払い層に基づくロールベースのアクセス制御が可能になります。

<Note>
ユーザーアカウントへの必要な変更を処理するエッジ機能は、AI によって自動的にセットアップされる必要があります。
</Note>

<Steps>
<Step title="Supabase をプロジェクトに接続する">
始めるのは簡単です。 ZenOpus は、組み込みのネイティブ統合により、Supabase への接続を簡単にします。

1. ZenOpus の右上隅にある **Supabase ボタン** をクリックします。
1. 指示に従ってプロジェクトをリンクします。
1. 接続すると、Supabase により安全な支払い処理、サブスクリプション管理、Webhook 処理、顧客データ ストレージ、およびエラー処理が可能になります。
</Step>
<Step title="安全な支払い処理">
ZenOpus を要求してプロセスを開始します。

<Note>
Stripe をプロジェクトに接続しましょう。まずは安全な支払い処理から始めます。
</Note>

ZenOpus は、支払いを処理するために必要な SQL スキーマを生成します。これには、ユーザー、サブスクリプション、支払いに関するデータベース テーブルが含まれます。変更を適用する前に、特定の製品のニーズに合わせてこれらのテーブルを確認およびカスタマイズできます。
</Step>
<Step title="Webhook のエッジ機能を実装する">
Supabase の **エッジ関数** は、ユーザーの近くで実行される小規模で高性能のサーバーレス関数として機能し、高速な応答を保証します。これらは、データベースを更新する前に、支払い確認などの Webhook イベントを処理するのに役立ちます。
