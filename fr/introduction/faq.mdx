## Commencer

<AccordionGroup>
<Accordion title="Qu'est-ce que ZenOpus ?">
ZenOpus est une plate-forme basée sur l'IA qui permet aux utilisateurs de tout niveau de compétence de créer des sites Web complets en langage naturel. Décrivez simplement ce que vous voulez et ZenOpus le construit pour vous.
</Accordion>
<Accordion title="Que puis-je construire avec ZenOpus ?">
Si vous pouvez l'imaginer, vous pouvez le construire avec ZenOpus. Des simples pages de destination aux outils puissants adaptés à des cas d'utilisation de niche, ZenOpus vous offre la flexibilité nécessaire pour créer tout ce dont vous avez besoin, sans limites.
</Accordion>
<Accordion title="Ai-je besoin d’une expérience en codage pour construire avec ZenOpus ?">
Non. ZenOpus est conçu pour tous les niveaux d’expérience. Que vous soyez développeur ou utilisateur non technique, vous pouvez créer des sites Web en utilisant ZenOpus.
</Accordion>
<Accordion title="Comment créer un projet ?">
Pour créer un projet, tapez simplement un message dans la zone de message du tableau de bord. Décrivez simplement ce que vous voulez construire et ZenOpus le prendra à partir de là et générera un point de départ pour votre projet. N’oubliez pas de commencer petit et de progresser progressivement !
</Accordion>
<Accordion title="Puis-je réutiliser un projet existant ?">
Oui, vous pouvez remixer un projet public existant ou l'un des vôtres. Le remixage vous permet de réutiliser l'état actuel d'un projet comme point de départ et de vous en inspirer. C’est un excellent moyen d’explorer de nouvelles idées, d’effectuer des ajustements ou d’effectuer différentes modifications tout en préservant la version originale.
</Accordion>
</AccordionGroup>

## Construire avec ZenOpus

<AccordionGroup>
<Accordion title="Comment puis-je construire efficacement avec ZenOpus ?">
Construire efficacement avec ZenOpus consiste à utiliser les bonnes techniques. Tout le monde peut le faire avec un peu de pratique. Pour vous aider à démarrer, consultez notre [guide](https://mastering-zenopus.zenopus.app/).
</Accordion>
<Accordion title="Comment dois-je structurer mes invites pour obtenir les meilleurs résultats ?">
- Soyez clair et précis.
- Concentrez-vous sur une fonctionnalité à la fois.
- Construisez progressivement.
- Utilisez des mots à la mode pour réussir le design.
</Accordion>
<Accordion title="Que dois-je faire lorsque je rencontre une erreur ?">
<Steps>
<Step title="Bouton Essayer de réparer">
Utilisez l'option **Essayer de résoudre**. ZenOpus tentera de résoudre le problème automatiquement. L’utilisation du bouton ne déduit aucun crédit.
</Step>
<Step title="Enquêter">
Si **Try-to-fix** ne résout pas le problème, demandez à l'agent ZenOpus d'enquêter sur le problème. Expliquez ce que vous voyez et quelles parties spécifiques ne fonctionnent pas.
</Step>
<Step title="Revenir à une ancienne version">
Si cela ne fonctionne pas, vous pouvez prévisualiser une ancienne version de travail, puis revenir à cette version si nécessaire, ou revenir et modifier un message antérieur pour explorer une approche différente. Rien n'est perdu : toutes les modifications apportées après ce point restent dans le chat et peuvent être réappliquées à tout moment.
</Step>
</Steps>
</Accordion>
<Accordion title="Qu’est-ce que le refactoring et pourquoi est-ce important ?">
La refactorisation améliore la structure et l'efficacité du code sans altérer les fonctionnalités. Cela aide :

- Optimisez les performances en supprimant les redondances.
- Améliorer la maintenabilité pour les futures mises à jour.
- Prévenez les conflits en structurant la logique plus clairement.
</Accordion>
<Accordion title="Puis-je connecter mon projet à un backend pour stocker des données ?">
Si vous devez ajouter des fonctionnalités backend à votre application, vous pouvez :

- Connectez-vous au backend intégré de ZenOpus -  [ZenOpus Cloud](/fr/features/cloud)
- Utiliser l'[intégration Supabase] native de ZenOpus (/fr/integrations/supabase)
</Accordion>
<Accordion title="Puis-je ajouter une connexion à mon site Web ?">
Oui, vous pouvez ajouter une authentification en utilisant [ZenOpus Cloud](/fr/features/cloud) ou en intégrant avec [Supabase](/fr/integrations/supabase). Demandez simplement à l’IA de l’ajouter et suivez les instructions.
</Accordion>
<Accordion title="Puis-je ajouter des API tierces à mon projet ?">
Oui, vous pouvez intégrer des API comme OpenAI, Stripe, etc. en les spécifiant dans votre invite. En savoir plus [ici](/fr/integrations/introduction).
</Accordion>
<Accordion title="Puis-je stocker des clés API sensibles dans ZenOpus ?">
Non ! Ne saisissez jamais vos clés API directement dans ZenOpus. Si vous effectuez une intégration avec une API, nous vous recommandons d'utiliser des secrets dans [ZenOpus Cloud](/fr/features/cloud) ou [Supabase](/fr/integrations/supabase) pour stocker vos clés API, en combinaison avec leurs fonctions Edge.
</Accordion>
<Accordion title="Comment ajouter des paiements sur mon site Web ?">
Vous pouvez ajouter des paiements en ajoutant Stripe à votre projet. Voir [Intégration Stripe](/fr/integrations/stripe) pour le configurer.
</Accordion>
<Accordion title="Comment puis-je améliorer le référencement de mon projet ?">
Garantissez les meilleurs résultats en suivant notre guide [Mise en œuvre des meilleures pratiques SEO et GEO](/fr/tips-tricks/seo-geo).
</Accordion>
<Accordion title="Combien de temps faut-il pour réaliser un projet ?">
Cela dépend de la complexité. En divisant les tâches en étapes plus petites, en planifiant à l'avance et en affinant en fonction des commentaires, vous pouvez avancer rapidement tout en garantissant la qualité.
</Accordion>
</AccordionGroup>

## Caractéristiques

<AccordionGroup>
<Accordion title="Comment trouver les paramètres du projet ?">
Vous pouvez trouver les **Paramètres du projet** en cliquant sur le nom de votre projet en haut à gauche de l'éditeur, puis en sélectionnant **Paramètres**.
</Accordion>
<Accordion title="Puis-je modifier la visibilité de mon projet ?">
Oui, accédez aux **Paramètres du projet** où vous pouvez modifier la visibilité. Voir [Contrôler la visibilité du projet](/fr/features/private-projects) pour plus d'informations.
</Accordion>
<Accordion title="Puis-je demander l’accès à des projets privés ?">
Oui. Lorsque vous accédez à un projet privé, vous pouvez demander un accès de lecteur ou d'éditeur au propriétaire du projet. Cela garantit la sécurité du travail privé tout en facilitant la collaboration.
</Accordion>
<Accordion title="Puis-je masquer le badge ZenOpus ?">
Oui, accédez aux **Paramètres du projet** où vous trouverez l'option permettant de masquer le badge. Seuls les utilisateurs payants peuvent désactiver le badge.
</Accordion>
<Accordion title="Puis-je renommer un projet ?">
Oui, accédez aux **Paramètres du projet** et vous y trouverez la possibilité de renommer le projet.
</Accordion>
<Accordion title="Comment copier (remixer) un projet ?">
Pour faire une copie d'un projet, vous le **remixez**. Accédez aux **Paramètres du projet** et sélectionnez l'option **Remix**.

Le remixage crée une copie exacte du projet dans votre Workspace actuellement sélectionné. Vous pouvez ensuite le modifier indépendamment.

<Note>
Vous ne pouvez remixer les projets publics d'autres utilisateurs que si le projet n'est pas connecté à Supabase.
</Note>
</Accordion>
<Accordion title="Puis-je supprimer un projet ?">
Oui, accédez aux **Paramètres du projet** et vous y trouverez l'option permettant de **Supprimer** le projet.

<Note>
Ceci est définitif et il n’existe aucun moyen de restaurer les projets supprimés.
</Note>
</Accordion>

Plan mode (anciennement mode Chat) est idéal pour planifier les fonctionnalités, déboguer les problèmes et comprendre les changements avant d'écrire ou de modifier du code. Voir [Réflexion avec Plan mode](/fr/features/plan-mode) pour plus d'informations.
</Accordion>
<Accordion title="Puis-je voir le code généré par ZenOpus et le modifier manuellement ?">
Oui. Vous pouvez voir le code et le modifier manuellement à l'aide de [Code mode](/fr/features/code-mode) de ZenOpus.
</Accordion>
<Accordion title="Puis-je modifier manuellement le texte ou les couleurs ?">
Oui, vous pouvez modifier le texte ou les couleurs à l'aide de [Modifications visuelles](/fr/features/visual-edit).
</Accordion>
<Accordion title="Puis-je collaborer sur mes projets ?">
Oui, vous pouvez collaborer sur des projets dans ZenOpus. Voir [Collaboration](/fr/features/collaboration) pour plus d'informations.
</Accordion>
<Accordion title="Comment publier mon projet ?">
Consultez [Publier](/fr/features/publish) et suivez les étapes qui y sont décrites pour publier votre projet.
</Accordion>
<Accordion title="Comment ajouter un domaine personnalisé à mon projet ?">
Consultez [Domaine personnalisé](/fr/features/custom-domain) pour ajouter un domaine personnalisé à votre projet.
</Accordion>
<Accordion title="Puis-je voir les analyses de mes projets publiés ?">
Oui. Voir [Analyses du projet](/fr/features/project-analytics) pour plus d'informations.
</Accordion>
<Accordion title="Puis-je ajouter des images à une invite ZenOpus ?">
Oui. Les images comme source d'inspiration pour la conception, les croquis de la mise en page souhaitée ou les captures d'écran des défauts de l'interface utilisateur sont très utiles lors de la conception de votre site Web.
</Accordion>
<Accordion title="Puis-je ajouter des vidéos à une invite ZenOpus ?">
Les vidéos ne sont pas encore prises en charge, mais les captures d'écran peuvent souvent faire l'affaire.
</Accordion>
<Accordion title="Puis-je exporter tout le code de mon projet ?">
Oui. En transférant votre code vers GitHub, vous pouvez exporter votre code et en faire ce que vous voulez. Voir [Intégration GitHub](/fr/integrations/github) pour plus d'informations.
</Accordion>
<Accordion title="Puis-je démarrer un projet en important du code depuis une source externe telle que GitHub ?">
Non, il n'existe actuellement aucun moyen de démarrer un projet ZenOpus à partir de code déjà existant sur par exemple GitHub.
</Accordion>
<Accordion title="Comment puis-je voir l'historique de mon projet ?">
Vous pouvez afficher l'historique de votre projet en faisant simplement défiler le chat vers le haut ou en appuyant sur l'icône **Afficher l'historique** en haut du chat. Vous pouvez également sélectionner des versions plus anciennes pour les prévisualiser ou restaurer votre projet dans cette version.
</Accordion>
<Accordion title="Puis-je revenir à une version antérieure de mon projet ?">
Oui. Vous pouvez prévisualiser une ancienne version de travail, puis revenir à cette version si nécessaire, ou modifier un ancien message et revenir pour explorer une approche différente. Rien n'est perdu : toutes les modifications apportées après ce point restent dans le chat et peuvent être réappliquées à tout moment.

- Recherchez l'option de retour sous chaque réponse dans le chat ou dans l'onglet Historique lorsque vous sélectionnez la version à laquelle vous souhaitez revenir.
- Recherchez l'option permettant de modifier un message passé et de revenir sous chaque message passé que vous avez envoyé dans le chat.
</Accordion>

## API ZenOpus

<Accordion title="Que puis-je faire avec l'API ZenOpus ?">
L'API **ZenOpus** permet de créer et de partager des applications ZenOpus par programmation, en transformant des idées, des invites ou même des images en applications entièrement générées avec un seul lien.

La première version, **Build with URL**, vous permet de générer des applications directement à partir d'une URL partageable. Avec lui, vous pouvez :

- Intégrez un bouton **Build with ZenOpus** sur votre site Web ou votre application.
- Partagez des invites prédéfinies via des URL avec des amis, des collègues ou sur les réseaux sociaux.
- Automatisez la génération d’applications dans le cadre des flux de travail internes.
- Prototypez rapidement des applications à l'aide d'URL avec des invites et des images de référence.

Découvrez la fonctionnalité et les cas d'utilisation réels dans notre article d'annonce : [Créer des applications à partir d'un lien avec ZenOpus Créer avec une URL](https://zenopus.dev/blog/introducing-zenopus-api-build-with-url).

Lisez le guide technique complet ici : [API ZenOpus : créer avec la documentation URL](/fr/integrations/build-with-url).
</Accordion>

## Gérer votre compte

<AccordionGroup>
<Accordion title="Puis-je modifier l'e-mail que j'utilise pour me connecter ?">
Malheureusement, il n'est pas possible de modifier directement l'adresse e-mail pour le moment. Pour contourner le problème, vous pouvez créer un nouveau compte avec l'adresse e-mail souhaitée et transférer vos projets en suivant ces étapes :

1. Rendez temporairement publics vos projets existants et copiez l’URL.
1. Accédez à l'URL lorsque vous êtes connecté à votre nouveau compte.
1. Remixez les projets dans votre nouveau compte. Veuillez noter que les projets connectés à Supabase doivent être déconnectés de Supabase avant de pouvoir les remixer.
1. Une fois les projets transférés, vous pouvez les rendre à nouveau privés si nécessaire.

Remarque : Si votre cible Workspace est sur un forfait payant, le remixage de projets publics peut être nécessaire pour y accéder. Les projets remixés ne reprendront pas les fonctionnalités au niveau de l'abonnement, à moins que le nouveau Workspace ne les propose également.
</Accordion>
<Accordion title="Puis-je supprimer mon compte ?">
Pour supprimer votre compte, accédez à **Paramètres → Votre compte**, faites défiler vers le bas et appuyez sur **Supprimer le compte**. Cela supprimera définitivement votre compte et toutes les données associées.
</Accordion>
<Accordion title="Puis-je réactiver un compte supprimé ?">
Non, vous ne pouvez pas réactiver un compte supprimé. Cependant, après avoir appuyé sur le bouton pour supprimer le compte, cette action peut être annulée dans les 48 heures pour arrêter la suppression.
</Accordion>
<Accordion title="Puis-je changer le propriétaire Workspace ?">
Oui, un propriétaire peut ajouter plusieurs propriétaires Workspace.
</Accordion>
<Accordion title="Comment puis-je transférer la propriété d’un projet à quelqu’un d’autre ?">
Une solution consiste à remixer le projet, et la personne qui remixe le projet sera désormais propriétaire de cette version.
</Accordion>
<Accordion title="Comment activer l'authentification à deux facteurs (2FA) ?">
**L'authentification à deux facteurs (2FA)** ajoute une couche de sécurité supplémentaire à votre compte ZenOpus en exigeant un code à usage unique lorsque vous vous connectez, en plus de votre méthode de connexion habituelle.

### Configurer 2FA

1. Accédez à **Paramètres → Compte → Votre compte** et activez **Authentification à deux facteurs.**
1. Choisissez et configurez une méthode de vérification en suivant les instructions à l'écran :
    - **Application d'authentification (recommandée)**  
    Utilisez une application telle que Google Authenticator, Microsoft Authenticator, Authy ou 1Password pour générer des codes à usage unique basés sur le temps.
    - **Téléphone (SMS)**  
    Recevez des codes de vérification à 6 chiffres par SMS.
1. Cliquez sur **Vérifier et activer**.

#### Connexion avec 2FA

Lorsque vous vous connectez :

1. Complétez votre méthode de connexion habituelle.
1. Choisissez une méthode de vérification.
1. Saisissez le code à 6 chiffres provenant de votre application d'authentification ou de votre SMS.

#### Gérer la 2FA

- Vous pouvez ajouter ou supprimer des méthodes 2FA des paramètres de votre compte à tout moment.
- Nous vous recommandons de conserver **au moins deux méthodes** configurées pour éviter d'être bloqué.
</Accordion>
<Accordion title="Que sont les niveaux de vibe coding et comment puis-je les ajouter à mon compte LinkedIn ?">
Les niveaux de codage vibratoire reflètent la façon dont vous utilisez activement ZenOpus, en fonction du nombre de messages que vous envoyez lors de la construction. Votre niveau augmente automatiquement à mesure que vous utilisez davantage ZenOpus.

Les niveaux sont :

- **Bronze** : Premiers pas avec ZenOpus
- **Argent** : utilisateur régulier de ZenOpus
- **Or** : constructeur ZenOpus actif
- **Platine** : constructeur ZenOpus très engagé
- **Diamant** : engagement ZenOpus de premier niveau

Vous pouvez afficher votre niveau dans **Paramètres → Votre compte**. En option, vous pouvez connecter votre compte LinkedIn pour afficher votre niveau actuel sous **Licences et certifications**. Ceci est entièrement facultatif et peut être déconnecté à tout moment.

Pour plus d'informations, consultez [Certification LinkedIn Vibe coding](/fr/tips-tricks/linkedin-certification) (bêta).
</Accordion>
</AccordionGroup>

## Comment fonctionne ZenOpus

<AccordionGroup>
<Accordion title="Quelles piles technologiques ZenOpus connaît-il ?">
ZenOpus construit le front-end à l'aide de React, Tailwind et Vite et peut se connecter aux backends OpenAPI. Support pour la persistance et l'authentification des données est disponible à l'aide de ZenOpus Cloud ou Supabase.
</Accordion>
<Accordion title="Est-ce que ZenOpus Support développe une application mobile ?">
Non, ZenOpus se concentre sur les **applications Web**, mais vous pouvez concevoir des applications Web adaptées aux mobiles.
</Accordion>
<Accordion title="Comment ZenOpus se souvient-il du contexte ?">
ZenOpus traite les messages récents pour maintenir le contexte. Pour de meilleurs résultats, incluez des détails pertinents dans chaque invite pour garantir la clarté et l’exactitude.
</Accordion>

En tant que créateur, vous le faites !
