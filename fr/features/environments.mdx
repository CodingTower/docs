Les environnements de test et en direct vous permettent de séparer le **développement** des **données de production** lorsque vous travaillez sur des projets [ZenOpus Cloud](/fr/integrations/cloud).

- **Test** : votre environnement de développement. ZenOpus est toujours intégré à Test lorsque les environnements Test et Live sont activés.
- **En direct** : votre environnement de production. C’est avec cela que les vrais utilisateurs interagissent. Mis à jour uniquement lorsque vous publiez explicitement.

Cela vous donne un moyen sûr de :

- Expérimentez de nouvelles fonctionnalités et des modifications de schéma
- Testez les intégrations, les flux de travail et les fonctions Edge
- Évitez les modifications involontaires des données de production

En isolant le travail de développement dans Test et en publiant intentionnellement, vous maintenez Live stable tout en continuant à itérer rapidement.

## Conditions préalables

- Le projet doit utiliser **ZenOpus Cloud**.
- Vous devez **publier au moins une fois** pour créer une **base de données en direct** avant d'activer le test.

## Comment ça marche

Lorsque vous activez les environnements Test et Live, ZenOpus crée une **base de données de test** à côté de votre **base de données Live (production)**.

Ce qui suit est **copié de Live vers Test une seule fois, lors de la configuration** :

- Schéma de base de données
- Données de la base de données
- Configuration cloud

C'est la seule fois où les données ou la configuration Cloud sont copiées entre les environnements.

Après cela :

- ZenOpus **toujours construit en Test, jamais en Live**
- Lors de la construction dans Test, ZenOpus peut référencer le contexte de Live pour aider à déboguer les problèmes, mais ne modifie jamais les données Live
- Les environnements Test et Live évoluent indépendamment
- Le code de l'application et le schéma de la base de données sont synchronisés via la publication
- Les données de la base de données et la configuration du cloud ne sont jamais partagées, réinitialisées ou écrasées
- La publication synchronise la structure, pas le contenu

**Publication de mises à jour sur le fonctionnement de votre application**. Lorsque vous publiez, ZenOpus synchronise le code de votre application et la structure de votre base de données de Test à Live (jamais l'inverse) afin que votre application de production se comporte de la même manière.

<Note>
Avant chaque publication, ZenOpus crée automatiquement une sauvegarde de votre **base de données Live**. Cela garantit que vous pouvez récupérer les données de production si quelque chose d'inattendu se produit pendant ou après la publication. Contactez Support si vous avez besoin d'aide pour restaurer une sauvegarde.
</Note>

**La publication ne modifie pas les données existantes ni la façon dont votre instance Cloud est configurée**. Chaque environnement possède en permanence ses propres données et paramètres.

| Composant applicatif | Descriptif | Synchronisé lors de la publication et partagé entre les environnements ? |
| --- | --- | --- |
| Code de demande | La logique de votre application (frontend, backend, fonctions Edge), qui définit le fonctionnement de votre application. Test intégré | Oui |
| Schéma de base de données | La structure de votre base de données (tables, colonnes, index, migrations). Test intégré | Oui* |
| Données de base de données | Le contenu de votre base de données (lignes, utilisateurs, enregistrements, téléchargements). Spécifique à l'environnement | Non |
| Configuration cloud | Paramètres d'environnement tels que l'authentification, les secrets, les variables d'environnement, les intégrations et les fonctionnalités. Spécifique à l'environnement | Non |

Les ***migrations de schéma sécurisées** qui ne devraient pas entraîner de perte de données (par exemple, l'ajout de tables ou de colonnes) sont **appliquées automatiquement à Live lors de la publication**.

Les **migrations de données** (déplacement ou transformation de données existantes) ne sont **pas exécutées automatiquement**. Si une modification pourrait **risquer une perte de données**, ZenOpus :

- Explique la situation
- Fournit une requête SQL
- Vous demande de l'exécuter manuellement dans l'éditeur SQL

## Activer les environnements de test et en direct

Pour activer les environnements Test et Live, suivez les étapes ci-dessous :

1. Accédez à **Vue Cloud → Paramètres avancés**.
1. Sélectionnez l'option **Activer les environnements de test et en direct**.
1. Terminez le flux d’intégration.
1. Cliquez sur le bouton **Activer les environnements de test et en direct**.

### Vue Test et Live Cloud

Lorsque vous activez les environnements Test et Live, vous pouvez basculer entre **Test** et **Live** dans la vue Cloud pour inspecter les données et les paramètres. Ce sélecteur contrôle uniquement **ce que vous visualisez**, et non **l'endroit où ZenOpus est construit**.

<Warning>
La modification des données Live ou de la configuration Live Cloud (authentification, secrets, intégrations) affecte **immédiatement** votre application de production.
</Warning>

## Supprimer la base de données de test

Vous pouvez supprimer la base de données Test depuis **Cloud → Paramètres avancés → Zone de danger**.

Cette action :

- Supprime toutes les données de test
- Désactive la fonctionnalité des environnements Test et Live
- N'affecte pas les données en direct

Vous pouvez réactiver les environnements Test et Live ultérieurement si nécessaire. Cela créera une nouvelle base de données de test à partir de l'état Live actuel.
