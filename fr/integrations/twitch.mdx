[Twitch](https://www.twitch.tv/) est une plateforme de diffusion en direct axée sur les jeux, le contenu créatif et l'interaction communautaire en temps réel. Le connecteur Twitch permet à votre application ZenOpus d'accéder à l'API Helix de Twitch afin que vous puissiez alimenter les superpositions de flux, les tableaux de bord et les outils communautaires qui réagissent à ce qui se passe sur une chaîne.

Avec Twitch, votre application peut :

- Afficher le chat en direct depuis une chaîne Twitch
- Afficher le nombre de followers et d'abonnés
- Suivez les objectifs personnalisés tels que les jalons d’abonnés ou d’abonnement
- Recherchez l'état du flux, le nombre de téléspectateurs et les métadonnées de la chaîne
- Récupérer des clips, des programmes et des informations sur les chaînes
- Envoyer des messages de discussion à une chaîne Twitch
- Rechercher des chaînes, des jeux et des catégories

Twitch est un bon choix lorsque votre application vit à côté d'un flux. Les exemples incluent les superpositions, les outils de modération, les tableaux de bord d'événements et les applications d'analyse des créateurs.

## Comment fonctionnent les connexions Twitch

Twitch utilise OAuth 2.0 pour l'authentification. Lorsque vous connectez Twitch, vous autorisez ZenOpus à accéder à Twitch au nom du compte authentifié.

Twitch utilise l'architecture de passerelle de ZenOpus pour la gestion sécurisée d'OAuth et l'actualisation automatique des jetons. Voir [Connecteurs basés sur une passerelle](/fr/integrations/introduction#connecteurs-basés-sur-une-passerelle) pour plus de détails sur l'authentification et les limites d'utilisation.

Chaque connexion Twitch est liée à un compte Twitch. Vous pouvez créer plusieurs connexions si nécessaire. Par exemple, vous pouvez connecter des comptes distincts pour le développement, la production ou différents créateurs.

## Comment connecter Twitch

Workspace **les administrateurs** et les **propriétaires** peuvent connecter Twitch.

Lorsque la connexion est créée, Twitch devient disponible sur tous les projets du Workspace.

### Conditions préalables

Avant de connecter Twitch, assurez-vous d'avoir :

- Un compte Twitch
- Un rôle d'administrateur ou de propriétaire ZenOpus Workspace

### Étape 1 : Choisissez le compte pour vous connecter

Votre connexion utilise directement un compte Twitch. Aucune application de développeur ou clé API n'est requise.

Assurez-vous que le compte connecté dispose des autorisations requises pour votre cas d'utilisation.

Par exemple :

- Pour lire les données des abonnés, le compte doit disposer d'un accès modérateur pour la chaîne.
- Pour envoyer des messages de discussion, le compte doit avoir l'autorisation de publier sur la chaîne.
- Pour accéder aux données d'abonnement, le compte connecté doit être le propriétaire de la chaîne.

Pour la plupart des configurations de superposition, la connexion du propre compte du streamer offre l’accès le plus large.

### Étape 2 : Connectez Twitch à ZenOpus

1. Accédez à **Paramètres → Connecteurs → Connecteurs partagés** et sélectionnez **Twitch**.
1. Cliquez sur **Ajouter une connexion**.
1. Dans **Nom d'affichage**, saisissez un nom pour la connexion. Par exemple `Twitch Production`.
1. **Configurer les étendues (facultatif)**Développez **Paramètres avancés** pour configurer les autorisations de l'API Twitch dont votre application a besoin. Les étendues par défaut incluent : Des étendues supplémentaires sont disponibles pour des fonctionnalités telles que la gestion des clips, les prédictions, les sondages, les analyses et les points de chaîne. Activez uniquement les étendues dont votre application a besoin. Vous pouvez mettre à jour les étendues ultérieurement en modifiant la connexion et en sélectionnant **Reconnecter**. Pour la liste complète des étendues, consultez la [Référence de l'étendue Twitch OAuth](https://dev.twitch.tv/docs/authentication/scopes/).
    - `user:read:email` — Afficher l'e-mail du compte
    - `channel:read:subscriptions` — Afficher les données d'abonnement aux chaînes
    - `moderator:read:followers` — Afficher les données des abonnés pour les chaînes modérées
    - `user:write:chat` — Envoyer des messages au chat Twitch
1. Cliquez sur **Connecter**. Une fenêtre d'autorisation Twitch s'ouvrira. Assurez-vous que les pop-ups sont autorisés.
1. Connectez-vous à Twitch, vérifiez les autorisations demandées et cliquez sur **Autoriser**.

Une fois l'autorisation terminée, la connexion apparaîtra dans ZenOpus et vos applications pourront commencer à utiliser les données Twitch.

## Limites

Le connecteur Twitch ne peut actuellement pas :

- **Lisez le chat Twitch en temps réel via la passerelle**  

 Le chat Twitch utilise IRC sur WebSocket, qui ne peut pas être proxy via la passerelle ZenOpus. Pour lire les messages de chat, votre application doit se connecter directement au point de terminaison de chat de Twitch : `wss://irc-ws.chat.twitch.tv`La lecture du chat public ne nécessite pas d'authentification. L'envoi de messages de discussion via l'API Twitch nécessite une authentification.
- **Recevoir les webhooks EventSub**  

 Les événements Twitch entrants tels que les notifications de démarrage de flux ou les événements de suivi ne sont actuellement pas pris en charge.
- **Support Connexion Twitch par utilisateur**  

 Chaque connexion authentifie un seul compte Twitch. Les utilisateurs finaux de votre application ne peuvent pas connecter leurs propres comptes Twitch via ce connecteur.

## Comment dissocier des projets d'une connexion

Les éditeurs et les éditeurs supérieurs peuvent supprimer des projets spécifiques d'une connexion sans supprimer entièrement la connexion. La connexion restera disponible pour d'autres projets.

Pour dissocier des projets :

<Steps>
<Step title="Accédez aux connecteurs">
Accédez à **Paramètres → Connecteurs → Connecteurs partagés** et sélectionnez **Twitch**.
</Step>
<Step title="Ouvrir la connexion">
Ouvrez la connexion que vous souhaitez gérer.
</Step>
<Step title="Sélectionnez des projets">
Sous **Projets liés**, cochez les projets dont vous souhaitez dissocier.
</Step>
<Step title="Confirmer">
Cliquez sur **Dissocier les projets** et confirmez.
</Step>
</Steps>

Une fois dissociés, ces projets n'auront plus accès à **Twitch** via cette connexion. Si un projet a à nouveau besoin de **Twitch**, vous pouvez le lier à n'importe quelle connexion disponible.

## Comment supprimer une connexion

Les administrateurs et propriétaires Workspace peuvent supprimer les connexions **Twitch**.

<Warning>
La suppression d'une connexion est permanente et ne peut être annulée. Cela supprimera les informations d'identification de tous les projets liés et toutes les applications utilisant cette connexion cesseront de fonctionner jusqu'à ce qu'une nouvelle connexion soit ajoutée.
</Warning>

Avant de supprimer, consultez la section **Projets liés** pour voir quels projets utilisent actuellement la connexion.

Pour supprimer une connexion :

<Steps>
<Step title="Accédez aux connecteurs">
Accédez à **Paramètres → Connecteurs → Connecteurs partagés** et sélectionnez **Twitch**.
</Step>
<Step title="Ouvrir la connexion">
Ouvrez la connexion que vous souhaitez supprimer.
</Step>
<Step title="Examiner les projets liés">
Consultez la section **Projets liés**.
</Step>
<Step title="Supprimer">
Sous **Supprimer cette connexion**, cliquez sur **Supprimer** et confirmez.
</Step>
</Steps>
