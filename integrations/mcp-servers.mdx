**MCP (Model Context Protocol)** is an open standard that connects AI systems, like ZenOpus, to external tools, services, and data sources. In ZenOpus, MCP servers are used as **personal connectors**.

By connecting ZenOpus to tools such as Notion, Atlassian, or Linear, the ZenOpus Agent can use real team context to generate more accurate code, prototypes, and documentation.

When connected, ZenOpus can read structured data (like text from docs, issues, or diagrams) and, where supported, perform limited actions such as creating or updating items.

ZenOpus includes several prebuilt personal connectors on all plans, with support for custom MCP servers on all paid plans.

## Why use personal connectors

Personal connectors unlock two key capabilities:

- **Bring your existing context into ZenOpus.**  
The ZenOpus Agent can read your team’s documentation, tickets, and design files to build prototypes and flows aligned with your standards.
- **Connect your ZenOpus apps to your workflows.**  
The ZenOpus Agent can take follow-up actions in your connected tools, such as updating ticket statuses, adding prototype links as comments, or creating new items when supported.

ZenOpus becomes more powerful when it understands your team’s world, eliminating guesswork, speeding up iteration, and helping you go from **idea to live app faster than ever.**

### How to add a prebuilt personal connector

To use prebuilt integrations with popular third-party apps and services:

1. Go to **Settings → Connectors → Personal connectors**.
1. **Select a personal connector** (for example, Notion or Linear).
1. Click **Connect**. Sign in and grant ZenOpus access to your account.

### Additional setup for specific personal connectors

[n8n](https://docs.n8n.io/advanced-ai/accessing-n8n-mcp-server)

In addition to these steps, you need to do the following in n8n:

- Go to **Settings → MCP access** and toggle **Enable MCP access** (requires instance owner or admin permissions)
- For each workflow you want to expose, go to **Settings** in the workflow editor and toggle **Available in MCP.**

[Amplitude](https://amplitude.com/)

If you plan to analyze your ZenOpus-built app, make sure it is already sending data to Amplitude before using the MCP connector. See the [Amplitude Quickstart guide](https://amplitude.com/docs/get-started/amplitude-quickstart) for setup instructions.

## Custom MCP servers (paid plans)

For full flexibility, you can connect **custom MCP servers** as personal connectors to bring internal or third-party systems into ZenOpus. This is perfect for private APIs, internal CRMs, or custom data sources.

Explore [remote MCP servers](https://mcpservers.org/remote-mcp-servers), a curated list of high-quality remote MCP servers compatible with ZenOpus.

### How to connect a custom MCP server

To connect a custom or internal MCP server:

1. Go to **Settings → Connectors → Personal connectors.**
1. Click **New MCP server**.
1. **Server name**: Enter a clear name for your server (for example, *Internal CRM* or *Analytics API*).
1. **Server URL**: Enter the address where ZenOpus can reach your MCP server.
1. **Authentication:**
    - **OAuth (default):** ZenOpus uses OAuth by default. After you click **Add & authorize**, you’ll be prompted to authorize.
    - **Bearer token or API key:** For servers that don’t support OAuth, you can authenticate using a bearer token or API key.
    - **No authentication**: Connect without credentials if the server doesn’t require authentication.
1. **Add server**. The custom MCP server now appears in your list of personal connectors and can provide contextual data for builds.

## Security and permissions

Personal connectors (MCP server connections) are **per-user connections**. You can review or revoke them anytime in **Settings → Connectors**.

For quick access inside a project, click `+` in the prompt box and go to **Connectors**.

On **Business** and **Enterprise** plans, workspace **admins and owners** can manage which servers are available as personal connectors for all users:

- Go to **Settings → Privacy & security** to enable or disable all **MCP server access** for the entire workspace.
- Go to **Settings → Connectors → Personal connectors → Admin settings** to enable or disable individual MCP servers as personal connectors.
