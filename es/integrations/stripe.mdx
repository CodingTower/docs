ZenOpus ahora te permite configurar Stripe completamente a través del **chat**.

**Configuración automática basada en chat (recomendado)**  

 Después de conectar **Supabase** y guardar tu **Clave secreta de Stripe** mediante **Agregar clave API**, solo describe lo que necesitas:

- "Agregue tres niveles de suscripción..."
- "Crear un pago único para mi libro electrónico a $29"  
ZenOpus genera las funciones de pago/borde del portal, tablas de base de datos con RLS y botones de interfaz de usuario, sin codificación manual ni webhooks a menos que usted los solicite.

<Note>
- Para **ventas únicas**, asegúrese de que su carrito o página de producto ya funcione.
- Para **suscripciones**, confirme que Supabase Auth esté implementada para que ZenOpus pueda vincular a los clientes de Stripe con el **id** de cada usuario.
</Note>

### Conclusiones clave

- Utilice el **flujo basado en chat** tanto para suscripciones como para pagos únicos.
- **Nunca pegues tu clave secreta de Stripe en el chat.** Configúrala a través del formulario **Agregar clave API** en el chat.
- **Los webhooks son de participación voluntaria.** ZenOpus se basa en el sondeo de la función perimetral a menos que usted solicite webhooks.
- Depure en **Consola del navegador → Red/Errores**, **Supabase → Funciones de Edge → Registros** y **Panel de control Stripe → Registros**.
- Pruebe siempre en el **Modo de prueba Stripe** y luego implemente.

## Requisitos

Antes de integrar Stripe, asegúrese de que se cumplan los siguientes requisitos previos:

- El proyecto **debe** estar conectado a Supabase. [Más información sobre Supabase](/es/integrations/supabase)
- Una **cuenta Stripe** con productos configurados correctamente.
- Un **frontend y backend** que funcionen:
    - Para ventas de productos individuales, asegúrese de que el carrito de compras y la página de pago estén funcionales.
    - Para suscripciones, configure funcionalidades de inicio de sesión y diferentes niveles de precios.

<Note>
**Tenga en cuenta** La integración de Stripe no funciona en la vista previa. Para probar la integración, asegúrese de implementarla. También debes asegurarte de estar en modo de prueba en Stripe cuando pruebes la funcionalidad. Al probar el pago, número de tarjeta: 4242 4242 4242 4242, 3 dígitos cualesquiera como CVC y cualquier fecha futura funcionarán como una tarjeta.
</Note>

## Configuración de pago de Stripe (flujo de chat sin código)

ZenOpus ahora genera toda la lógica de Stripe por usted. Una vez que su **Clave secreta de Stripe** esté configurada a través del formulario **Agregar clave API** en el chat y su proyecto esté **conectado a Supabase**, simplemente dígale a ZenOpus lo que necesita en el chat; no se requieren enlaces de pago manuales.

<Steps>
<Step title="Paso 1">
**Prepara tu proyecto**

- Supabase conectada
- Clave secreta de Stripe agregada a través del formulario **Agregar clave API** en el chat
- (Opcional) Precios o ID de productos a mano
</Step>
<Step title="Paso 2">
Ejemplos:

- Cree un pago único para mi “Curso digital” a $29
- Configure un plan Premium anual por $99, vinculado a la identificación de cada usuario
</Step>
<Step title="Paso 3">
**Revisar y aplicar** ZenOpus realiza un scaffolding automático de las funciones perimetrales, las tablas de la base de datos y los componentes de la interfaz de usuario (todos vinculados a la identificación del usuario en Supabase). Verifique la vista previa y luego haga clic en **Aplicar** para implementar.
</Step>
</Steps>

<Note>
- Las suscripciones siempre deben vincularse al `id` del usuario autenticado en Supabase para un acceso seguro basado en roles.
</Note>

## Integración avanzada: Webhooks y Supabase

Para estructuras de pago complejas como **suscripciones y acceso basado en roles**, ZenOpus recomienda usar **Supabase** para manejar de forma segura la integración de Stripe. Esto permite un manejo adecuado de webhooks, gestión de suscripciones y control de acceso basado en roles según niveles de pago.

<Note>
La IA debe configurar automáticamente la función Edge que maneja los cambios necesarios en la cuenta de usuario.
</Note>

<Steps>
<Step title="Conecte Supabase a su proyecto">
Empezar es sencillo. ZenOpus facilita la conexión de Supabase con una integración nativa incorporada:

1. Haga clic en el **botón Supabase** en la esquina superior derecha de ZenOpus.
1. Siga las instrucciones para vincular su proyecto.
1. Una vez conectado, Supabase permite el procesamiento seguro de pagos, la gestión de suscripciones, el manejo de webhooks, el almacenamiento de datos de clientes y el manejo de errores.
</Step>
<Step title="Procesamiento de pagos seguro">
Inicie el proceso solicitando ZenOpus:

<Note>
Conectemos Stripe a mi proyecto. Comenzaremos con el procesamiento de pagos seguro.
</Note>

ZenOpus generará el esquema SQL necesario para manejar los pagos. Esto incluye tablas de bases de datos para usuarios, suscripciones y pagos. Puede revisar y personalizar estas tablas para que se ajusten a las necesidades específicas de su producto antes de aplicar cambios.
</Step>
