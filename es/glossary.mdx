## Conceptos generales

- **IA (Inteligencia Artificial):** La simulación de procesos de inteligencia humana mediante máquinas, especialmente sistemas informáticos, que permite tareas como aprender, razonar y resolver problemas.
- [**Mensaje**](/es/prompting/prompting-one)**:** Un fragmento de texto o entrada que guía a los modelos de IA en la generación de resultados o la realización de determinadas tareas. Está dentro de la funcionalidad principal de ZenOpus y es con indicaciones que usted crea, modifica un componente o elemento específico en su aplicación.

## ZenOpus términos específicos

- [**Plan mode**](/es/features/plan-mode)**:** ZenOpus va más allá de simplemente generar código; se convierte en un asistente interactivo que lo guía a través de cada fase del desarrollo. Ayuda a los fundadores a pensar críticamente, planificar de forma eficaz, depurar de forma inteligente y realizar envíos con confianza.
- **Modo Edición**: La acción de realizar cambios o modificaciones en el contenido o código.
- [**Editar**](/es/features/design)**:** Una herramienta que permite el desarrollo impulsado por IA con controles visuales nativos de Tailwind para un fácil refinamiento.
- **Historial:** El sistema de seguimiento y gestión de cambios en tu aplicación y código a lo largo del tiempo.
- [**Conocimiento**](/es/features/knowledge)**:** Capture los detalles esenciales del proyecto en un documento vivo que evoluciona con su aplicación.
- [**Labs**](/es/features/labs)**:** Funciones experimentales dentro de ZenOpus dedicadas a probar y mostrar funcionalidades nuevas, innovadoras o en desarrollo. Estos podrán ser modificados o eliminados en cualquier momento.
- **Remix:** Puedes remezclar un proyecto público existente o uno propio. La remezcla le permite reutilizar el estado actual de un proyecto como punto de partida y desarrollarlo. Es una excelente manera de explorar nuevas ideas, realizar ajustes o iterar con diferentes cambios conservando la versión original. Tenga en cuenta que no es posible remezclar proyectos cuando Supabase está conectado.
- **Vista previa:** Permite a los usuarios ver o experimentar contenido en vivo o funcionalidades de funciones antes de su finalización o publicación, de forma interactiva.
- **Diff:** Una comparación que muestra las diferencias entre dos versiones de un archivo o código.
- **/index:** Se refiere a la página principal o predeterminada de un sitio web o aplicación, a menudo denominada “index.html” o “index.js”.
- [**ZenOpus API - Compilación con URL**](/es/integrations/build-with-url)**:** La función ZenOpus API Compilación con URL le permite crear aplicaciones mediante programación pasando mensajes e imágenes a través de parámetros de URL. Esto permite una integración perfecta con sitios web, herramientas internas o flujos de trabajo de automatización, lo que permite a los usuarios generar aplicaciones ZenOpus con un solo clic.

## Conceptos de producto y desarrollo

### Gestión y estrategia de producto

- **MVP (Producto Mínimo Viable)**: una versión de un producto con las características suficientes para satisfacer a los primeros usuarios y validar una idea antes del desarrollo a gran escala.
- **Hoja de ruta**: un plan estratégico de alto nivel que describe la visión, la dirección y las características planificadas de un producto a lo largo del tiempo.
- **Solicitud de función**: una sugerencia formal o informal de los usuarios o partes interesadas para una nueva capacidad en un producto.
- **Historia de usuario**: una descripción breve y sencilla de una característica o requisito escrita desde la perspectiva de un usuario final.
- **Viaje del usuario:** La serie de pasos que sigue un usuario para lograr un objetivo específico dentro de un producto o servicio.
- **Persona:** Una representación ficticia de un usuario objetivo, basada en investigaciones, para guiar las decisiones de diseño y desarrollo.

### Conceptos de ingeniería

- **PRD (Documento de requisitos del producto):** Un documento completo que describe los objetivos, características y especificaciones de un producto para guiar el desarrollo.
- [**API**](https://zenopus.dev/blog/2025-01-28-the-power-of-apis)**(Interfaz de programación de aplicaciones):** Una forma para que diferentes servicios se comuniquen. Piense en ello como un protocolo que le indica qué tipo de información puede obtener o enviar entre sistemas. Los tipos más comunes son las API REST, que utilizan muchos sitios, y las API GraphQL, que ofrecen un enfoque más flexible para consultar datos.
- **Refactor:** El proceso de reestructurar el código existente sin cambiar su comportamiento externo para mejorar la legibilidad, el mantenimiento o el rendimiento.
- **GitHub:** Una plataforma basada en web para control de versiones y desarrollo colaborativo de software utilizando Git.

### Datos y análisis

- **Pruebas A/B**: un método para comparar dos versiones de una página web o función para determinar cuál funciona mejor según el comportamiento del usuario.
- **Tasa de conversión**: el porcentaje de usuarios que completan una acción deseada, como registrarse o realizar una compra.
- **Tasa de abandono**: el porcentaje de usuarios que dejan de usar un producto durante un período determinado.
- **Tasa de retención**: el porcentaje de usuarios que continúan usando un producto a lo largo del tiempo.
- **Seguimiento de eventos**: monitorear acciones específicas del usuario (por ejemplo, clics en botones, envíos de formularios) para recopilar información sobre el comportamiento y la participación.

## Guía de desarrollo UI/UX y frontend

### Desarrollo front-end

- **Frontend:** La parte de una aplicación de software con la que los usuarios interactúan directamente, abarcando la interfaz de usuario y los elementos de la experiencia del usuario.
- **React:** Una biblioteca de JavaScript desarrollada por Facebook para crear interfaces de usuario, en particular aplicaciones de una sola página.
- **Gradiente:** Una transición gradual entre dos o más colores o matices en el diseño.
- **Tailwind CSS**: marco CSS de código abierto basado en utilidades que proporciona un conjunto completo de clases predefinidas, lo que permite a los desarrolladores crear diseños personalizados directamente dentro de su HTML aplicando estas clases de utilidad a los elementos.
- **Sistemas de diseño:** Una colección de componentes, pautas y estándares reutilizables que garantizan consistencia y coherencia en el diseño de un producto o marca.

### Conceptos de diseño UI/UX

- **Color de acento:** Un tono distintivo que se utiliza para resaltar elementos clave en un diseño, a menudo alineándose o complementándose con el color primario de la marca para mejorar el interés visual.
- **Tema:** Un conjunto coherente de opciones de diseño, incluidos colores, fuentes y diseños, que definen la estética general y la experiencia del usuario de una aplicación o sitio web.
- **Diseño responsivo:** Un enfoque del diseño web que garantiza que el contenido se adapte perfectamente a distintos tamaños y orientaciones de dispositivos, proporcionando una experiencia de usuario óptima.
- **Above the Fold:** La parte de una página web visible para los usuarios sin necesidad de desplazarse, crucial para captar la atención y transmitir información esencial de inmediato.
- **CTA (llamado a la acción):** Un mensaje, a menudo en forma de botón o enlace, que anima a los usuarios a realizar una acción específica, como "Registrarse" o "Más información".

### Estructura de página y navegación

- **Encabezados (Títulos):** Elementos de texto que presentan y organizan secciones de contenido, generalmente formateados con diferentes niveles (por ejemplo, H1, H2) para establecer una jerarquía de contenido clara.
- **Pie de página:** La sección inferior de una página web, que normalmente contiene información complementaria, como detalles de contacto, enlaces de navegación y exenciones de responsabilidad legales.
- **Ruta de navegación:** Una ayuda de navegación que muestra la ubicación actual del usuario dentro de la jerarquía de un sitio, a menudo representada como una lista horizontal de enlaces.
- **Favicon**: un pequeño icono que se muestra en la pestaña del navegador, en los marcadores y en los resultados de búsqueda, y que normalmente representa una marca o un sitio web.
- **Metatítulo (etiqueta de título)**: el título de una página web que aparece en los resultados del motor de búsqueda y en las pestañas del navegador, lo que influye en el SEO y en las tasas de clics de los usuarios.
- **Meta descripción**: un breve resumen del contenido de una página web que se muestra en los resultados del motor de búsqueda, destinado a mejorar la visibilidad y la participación del usuario.
- **URL canónica**: una etiqueta que se utiliza para especificar la versión preferida de una página web, lo que ayuda a evitar problemas de contenido duplicado en la indexación de los motores de búsqueda.
- **URL Slug**: la parte de una URL que identifica una página particular en un formato legible (por ejemplo, `example.com/product-name`).
- **Mapa del sitio**: un archivo estructurado (XML o HTML) que proporciona una lista de las páginas de un sitio web, lo que ayuda a los motores de búsqueda a indexarlas de manera eficiente.
- **Barra de navegación (Nav Bar)**: un menú horizontal o vertical que proporciona enlaces a secciones esenciales de un sitio web para facilitar la navegación.
- **Omitir enlaces**: enlaces ocultos que permiten a los usuarios omitir contenido repetitivo y navegar directamente al contenido principal, mejorando la accesibilidad.
- **Paginación**: un método para dividir el contenido en varias páginas, comúnmente utilizado en blogs, resultados de búsqueda y listados de productos para mejorar la experiencia del usuario.
- **Enlace de anclaje**: un hipervínculo que dirige a los usuarios a una sección específica dentro de la misma página en lugar de cargar una página nueva.
- **Página 404**: una página de error personalizada que se muestra cuando un usuario intenta acceder a una URL inexistente y lo guía de regreso al contenido relevante.

### Elementos de notificaciones y comentarios

- **Brindis:** Una notificación breve y no intrusiva que aparece temporalmente para informar a los usuarios sobre el resultado de una acción o proporcionar comentarios simples.
- **Barra de refrigerios**: similar a un brindis, pero generalmente ubicada en la parte inferior de la pantalla con acciones opcionales (por ejemplo, "Deshacer" en un elemento eliminado).
- **Información sobre herramientas**: un pequeño cuadro emergente que proporciona información adicional al pasar el cursor sobre un elemento o centrarse en él.
- **Insignia**: un pequeño indicador visual, que suele usarse en íconos o botones, para mostrar notificaciones, recuentos o actualizaciones de estado.
- **Loading Spinner (Loader)**: un indicador visual que representa un proceso en progreso, como cargar una página o enviar un formulario.
- **Barra de progreso**: una barra horizontal que indica el porcentaje de finalización de una tarea o proceso.
- **Skeleton Loader**: una interfaz de usuario de marcador de posición que imita el diseño del contenido final mientras espera que se cargue, lo que mejora el rendimiento percibido.

### Superposiciones y ventanas emergentes

- **Popover:** Una superposición transitoria que presenta información u opciones adicionales relacionadas con un elemento específico de la interfaz de usuario y que normalmente aparece tras la interacción del usuario.
- **Diálogo (modal):** Una ventana que aparece frente al contenido principal para captar la atención del usuario y que a menudo requiere una acción antes de regresar a la interfaz principal.
- **Cajón (panel de barra lateral)**: un panel que se desliza desde el costado de la pantalla para mostrar opciones o configuraciones de navegación sin interrumpir el contenido principal.
- **Lightbox**: una ventana modal que muestra imágenes o medios en una vista ampliada, atenuando el fondo para enfocarlo.
- **Cuadro de alerta**: un cuadro de mensaje generado por el sistema o activado por la aplicación que informa a los usuarios sobre información importante, como errores o advertencias.

### Elementos de navegación y selección

- **Botón:** Un elemento en el que se puede hacer clic que inicia una acción o evento, como enviar un formulario o abrir un cuadro de diálogo.
- **Interruptor (Alternar):** Un control de alternancia que permite a los usuarios cambiar entre dos estados, como encendido o apagado, y que a menudo se asemeja a un interruptor físico.
- **Pestañas**: un componente que organiza el contenido en vistas separadas dentro de una única interfaz, lo que permite a los usuarios cambiar entre secciones sin tener que navegar.
- **Paso a paso (asistente)**: un componente utilizado para procesos de varios pasos, que muestra a los usuarios su paso actual y los guía hasta su finalización.
- **Paginación**: un patrón de interfaz de usuario que divide el contenido en páginas, que a menudo se utiliza para resultados de búsqueda o aplicaciones con mucho contenido.
- **Ruta de navegación**: una ayuda de navegación que muestra la ubicación actual del usuario dentro de la jerarquía de un sitio, a menudo representada como una lista horizontal de enlaces.
- **Acordeón**: una sección plegable que se expande o contrae para revelar u ocultar contenido dinámicamente.
- **Menú desplegable**: una lista de opciones que aparece cuando un usuario hace clic o pasa el cursor sobre un botón o campo, que a menudo se usa para navegación o selección.

### Formularios y elementos de entrada

- **Formulario:** Una colección estructurada de campos de entrada que permiten a los usuarios enviar datos, como información de contacto o consultas de búsqueda.
- **Grupo de radio:** Un conjunto de botones de opción relacionados que permiten a los usuarios seleccionar solo una opción entre múltiples opciones.
- **Casilla de verificación:** Una casilla interactiva que los usuarios pueden marcar o desmarcar para seleccionar o deseleccionar una opción, permitiendo múltiples selecciones dentro de un conjunto.
- **Campo de texto (campo de entrada)**: un elemento básico de la interfaz de usuario donde los usuarios pueden ingresar texto, como en barras de búsqueda, formularios de inicio de sesión o comentarios.
- **Área de texto**: un campo de entrada más grande diseñado para entrada de texto de varias líneas, comúnmente utilizado para mensajes o formularios de comentarios.
- **Seleccionar (cuadro de selección desplegable)**: un elemento de la interfaz de usuario que permite a los usuarios elegir una opción de una lista predefinida, a menudo con el estilo de un menú desplegable.
- **Selector de fecha**: un elemento de la interfaz de usuario que permite a los usuarios seleccionar una fecha de un calendario en lugar de escribirla manualmente.
- **Control deslizante (selector de rango)**: un control que permite a los usuarios establecer un valor dentro de un rango deslizando un controlador a lo largo de una pista.
- **Campo de carga de archivos**: un componente que permite a los usuarios seleccionar y cargar archivos desde su dispositivo.
- **Autocompletar (búsqueda anticipada)**: una entrada de texto que sugiere dinámicamente resultados coincidentes a medida que los usuarios escriben, lo que mejora la usabilidad en las búsquedas.

- **Neobrutalismo:** Una tendencia de diseño web caracterizada por elementos crudos y sin pulir, tipografía atrevida y alto contraste, que se inspira en la arquitectura brutalista de mediados del siglo XX.

- **Nudy:** Un enfoque de diseño minimalista que utiliza paletas de colores nude o neutros para crear una estética suave y discreta.

## Desarrollo backend y bases de datos

### Fundamentos de backend

- **Backend:** La infraestructura del lado del servidor de una aplicación que maneja el procesamiento de datos, el almacenamiento y la lógica empresarial.
- [**Supabase**](/es/integrations/supabase)**:** Una plataforma backend como servicio de código abierto que proporciona una base de datos Postgres, autenticación, API instantáneas y capacidades en tiempo real.
- **Webhooks**: devoluciones de llamadas HTTP basadas en eventos que notifican a las aplicaciones externas cuando ocurren acciones específicas.

### Gestión y consulta de bases de datos

- **CRUD (Crear, Leer, Actualizar, Eliminar):** Las cuatro operaciones básicas realizadas con los datos de una base de datos o aplicación.
- [**SQL (lenguaje de consulta estructurado)**](https://supabase.com/docs/guides/database/overview)**:** Un lenguaje de programación de dominio específico diseñado para administrar y manipular bases de datos relacionales, permitiendo tareas como consultar, actualizar y organizar datos.
- **PostgreSQL**: una potente base de datos relacional de código abierto conocida por su escalabilidad, extensibilidad y cumplimiento de ACID.

### Conceptos específicos de supabase

- [**Función de borde**](https://supabase.com/docs/guides/functions)**:** Funciones de TypeScript del lado del servidor, distribuidas globalmente en el borde, cerca de sus usuarios. Se pueden utilizar para escuchar webhooks o integrar su proyecto Supabase con terceros como Stripe, Anthropic o Resend.
- [**RLS (Seguridad a nivel de fila)**](https://supabase.com/docs/guides/database/postgres/row-level-security)**:** Una función en las bases de datos que permite controlar el acceso a filas específicas en una tabla según los roles o atributos del usuario; básicamente permite a los usuarios establecer reglas de autorización granulares, lo que le permite escribir reglas SQL complejas que se ajusten a sus necesidades comerciales únicas.
- [**Almacenamiento**](https://supabase.com/docs/guides/storage)**:** Un servicio o sistema para almacenar y administrar datos, como imágenes, videos, documentos y cualquier otro tipo de archivo.
- [**Bucket**](https://supabase.com/docs/guides/storage/buckets/fundamentals)**:** En Supabase Storage, un depósito es un contenedor distinto que organiza archivos y carpetas, determina el modelo de acceso (público o privado) y establece restricciones de carga, como el tamaño máximo de archivo y los tipos de contenido permitidos.
- [**Punto final**](https://supabase.com/docs/guides/api)**:** En el contexto de Supabase, un punto final se refiere a una URL específica que permite a los clientes interactuar con la base de datos a través de llamadas API RESTful, lo que permite operaciones como crear, leer, actualizar o eliminar datos.
- [**Proveedor de autenticación:**](https://supabase.com/docs/guides/auth) Una entidad o servicio que verifica las credenciales del usuario, permitiendo el acceso seguro a aplicaciones o sistemas. En el contexto de Supabase, los proveedores de autenticación incluyen varios métodos, como inicios de sesión basados ​​en contraseñas, enlaces mágicos, contraseñas de un solo uso (OTP), inicios de sesión sociales e integraciones de inicio de sesión único (SSO), lo que facilita una autenticación de usuario flexible y segura.

### Seguridad y autenticación

- **OAuth**: un protocolo de autenticación ampliamente utilizado que permite a los usuarios iniciar sesión en aplicaciones utilizando servicios de terceros como Google, Facebook o GitHub.
- **Autenticación de dos factores (2FA)**: una medida de seguridad que requiere que los usuarios proporcionen dos formas de identificación antes de acceder a una cuenta.
- **CORS (intercambio de recursos entre orígenes)**: una política de seguridad que controla qué sitios web o aplicaciones pueden acceder a los recursos de un servidor.
