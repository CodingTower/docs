<AccordionGroup>
<Accordion title="Cree una aplicación web de seguimiento de calorías impulsada por IA">
<iframe src="https://www.youtube.com/embed/c0zhLzcVJRI?si=7vcZf5eoxIobOxdE" title="reproductor de vídeos de youtube" width="100%" height="315" frameBorder="0" allow="accelerometer; autoplay; clipboard-write; encrypted-media; gyroscope; picture-in-picture; web-share" referrerPolicy="strict-origin-when-cross-origin" allowFullScreen />

En este tutorial, lo guiaremos en la creación de una aplicación de seguimiento de calorías impulsada por IA con ZenOpus, completa con autenticación, un hermoso diseño responsivo, integración de OpenAI e implementación en producción.

<Steps>
<Step title="Estableciendo la base">
La base de cualquier proyecto ZenOpus es un buen mensaje. Para esta aplicación, aquí hay un ejemplo:

```text
Cree una aplicación de seguimiento de calorías con las siguientes características:
Página de destino, panel de control y página de comidas.
Campos de entrada para descripciones de comidas o imágenes, con análisis impulsado por GPT-4.
Seguimiento diario del progreso e integración con Supabase para autenticación y gestión de bases de datos.
Interfaz de usuario hermosa y adaptable a dispositivos móviles.
```

Una vez que ZenOpus genera el esqueleto de la interfaz de usuario:

- Revise la configuración de navegación y el diseño de la página de destino.
- Repita el mensaje para refinar el diseño y garantizar la capacidad de respuesta móvil.

<Note>
Sea explícito acerca de comenzar con la interfaz de usuario. ZenOpus funciona mejor cuando las tareas se dividen en pasos claros. Por ejemplo, describa la navegación, las estructuras de las páginas y los temas de diseño al principio del mensaje.
</Note>
</Step>
<Step title="Diseñando la interfaz de usuario">
ZenOpus enfatiza un enfoque paso a paso, similar a la construcción de un edificio:

- Comience con **andamios**: planifique la estructura de su aplicación (por ejemplo, flujo de experiencia del usuario, diseño de página).
- Agregue la **primera capa**: cree páginas vacías con elementos de diseño básicos.
- Refine los **detalles**: agregue colores, animaciones y capacidad de respuesta móvil.

<Note>
Si tiene diseños prefabricados en Figma, puede tomar capturas de pantalla e incluirlas en su mensaje ZenOpus. Proporcionar imágenes de alta resolución garantiza mejores resultados.
</Note>
</Step>
<Step title="Integre Supabase para backend y autenticación">
Una vez que la interfaz de usuario esté lista, es hora de agregar la funcionalidad de backend:

1. Configure una cuenta **Supabase****** y cree un proyecto.
1. Vincúlelo a ZenOpus presionando el botón “Conectar Supabase”.
1. Para el almacenamiento de datos, ZenOpus generará tablas SQL. Por ejemplo:

- Una tabla `profiles` para almacenar información del usuario.
- Una tabla `meals` vinculada a los usuarios para realizar un seguimiento de las entradas diarias.

<Note>
Utilice reglas de seguridad a nivel de fila (RLS) en Supabase para garantizar la privacidad del usuario. ZenOpus puede manejar esto automáticamente.
</Note>
</Step>
<Step title="Integración de OpenAI para el análisis de comidas">
La magia de esta aplicación radica en sus capacidades de inteligencia artificial. Para analizar comidas:

1. Obtenga una clave API de OpenAI desde su panel de desarrollador.
1. Utilice GPT-4 para analizar descripciones o imágenes de comidas.
1. Asegúrese de que las claves API estén almacenadas de forma segura en las funciones de Edge para evitar el acceso no autorizado.

**Características a incluir:**

- **Analizar descripciones de comidas**: Calcule los valores nutricionales (calorías, proteínas, carbohidratos, grasas).
- **Reconocimiento de imágenes**: cargue una foto de la comida y GPT-4 extraerá una descripción y datos nutricionales.

<Note>
*Pro Consejos para la integración de OpenAI:*

- Utilice **llamadas a funciones** para respuestas estructuradas, asegurando que los datos tengan el formato correcto para la inserción en la base de datos.
- Pruebe la funcionalidad pidiéndole a la aplicación que analice entradas de comidas simples como "2 huevos".
</Note>
</Step>
<Step title="Implementación de pagos con Stripe">
Para monetizar la aplicación, agregue pagos basados en suscripción:

1. Cree una cuenta de Stripe y configure un producto (por ejemplo, "Plan estándar" por $12 al mes).
1. Integre Stripe con ZenOpus agregando la clave API y el ID del precio del producto.
1. Redirigir a los usuarios que no pagan a la página de destino. Sólo los usuarios que pagan pueden acceder a la aplicación principal.

<Note>
*Consejos adicionales:*

- Habilite el **portal del cliente** de Stripe para permitir a los usuarios administrar las suscripciones (por ejemplo, cancelar o actualizar los métodos de pago).
- Pruebe los pagos en el modo de prueba de Stripe utilizando los datos de la tarjeta de crédito de prueba proporcionados.
</Note>
</Step>
<Step title="Pruebas y depuración">
Los errores son una parte natural del desarrollo y ZenOpus simplifica la depuración:

- Utilice el botón **“Intentar arreglar”** siempre que se produzca un error de compilación.
- Para problemas de backend, consulte los registros de Supabase para obtener informes de errores detallados.
- Si realiza la implementación en Netlify, revise los registros de implementación para detectar cualquier problema de compilación. Comparta registros con ZenOpus para obtener correcciones personalizadas.

<Note>
Si te quedas atascado, vuelve a una versión anterior usando la función "Revertir" de ZenOpus. Esto ahorra créditos y tiempo.
</Note>
</Step>
<Step title="Implementando su aplicación">
Una vez que su aplicación esté lista, impleméntela con ZenOpus o Netlify:

1. Utilice la función de implementación de ZenOpus para alojar la aplicación en un dominio predeterminado.
1. Para dominios personalizados:

- Conecte su repositorio de GitHub a Netlify.
- Compre un dominio y configure los ajustes de DNS para vincularlo a Netlify.

<Note>
Actualice los metadatos de su aplicación (favicon, metaetiquetas, imagen OG) para una mejor marca y SEO.
</Note>
</Step>
</Steps>

<Step title="Personalización de la interfaz de usuario sin esfuerzo con ediciones visuales">
Una de las mayores mejoras de ZenOpus es la capacidad de modificar elementos de la interfaz de usuario instantáneamente sin consumir créditos de IA. Ahora puedes:

- Edite texto directamente en su proyecto.
- Ajuste los colores y estilos de los botones en tiempo real.

- Pase el cursor sobre los elementos para realizar ajustes rápidos.

<Note>
Los usuarios ahora pueden seleccionar texto, cambiar colores o intercambiar íconos sin esfuerzo sin necesidad de intervención de IA, lo que permite un proceso de iteración de diseño más fluido.
</Note>
</Step>
<Step title="Depuración avanzada y manejo de errores de API">
Anteriormente, los usuarios enfrentaban desafíos al lidiar con errores de integración de API de terceros. ZenOpus ahora:

- Muestra mensajes de error más completos para facilitar la depuración.
- Elimina ventanas emergentes irrelevantes, mejorando el enfoque en temas críticos.
- Habilita la detección de errores basada en IA para correcciones automáticas.

<Note>
En lugar de abrumar a los usuarios con ventanas emergentes genéricas, ZenOpus ahora introduce el contexto de error directamente en su IA, que puede decidir cuándo es necesario abordar un problema y cómo solucionarlo.
</Note>
</Step>
<Step title="Crear fondos y animaciones dinámicas">
ZenOpus ahora admite la integración perfecta de elementos animados e interactivos usando **p5.js** y otras bibliotecas.

#### Proyecto de ejemplo: sitio web de agencia de viajes con banderas interactivas

Una demostración en vivo reciente mostró cómo crear una página de inicio de agencia de viajes que incluye:

1. **Una sección de héroe animada** con banderas que aparecen y se mueven dinámicamente en respuesta a las interacciones del mouse.
1. **Una sección de desplazamiento fijo** que cambia según la posición de desplazamiento del usuario.
1. **Efectos de desplazamiento interactivos** donde las imágenes reaccionan a los movimientos del usuario.

#### Cómo implementar efectos animados

Para crear estos efectos, los usuarios siguieron este sencillo proceso:

- **Defina la animación deseada:**

<Note>
Haga que el fondo sea interactivo con elementos flotantes que se alejan al pasar el cursor sobre él.
</Note>

- **Utilice p5.js para generar efectos:**
    - Implemente un patrón de humo cromático.
    - Cambie imágenes estáticas por animaciones dinámicas de banderas.
- **Mejore la experiencia del usuario con interacciones basadas en la física:**
    - Haga que los elementos se muevan dinámicamente según las acciones del usuario.

<Note>
Si una animación no funciona, use Plan mode de ZenOpus para depurar y refinar el efecto con guía de IA paso a paso.
</Note>
</Step>

1. Copie el mensaje generado por IA.
1. Pégalo en ZenOpus y reemplaza un botón existente.

Este enfoque acelera los flujos de trabajo de diseño y permite interfaces más pulidas y visualmente cohesivas.

<Note>
Este mismo método funciona para importar ajustes preestablecidos de animación desde **motion.dev**, lo que facilita agregar efectos de movimiento de alta calidad sin codificación personalizada.
</Note>
</Step>
