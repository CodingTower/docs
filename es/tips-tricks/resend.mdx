## ¿Qué es el reenvío?

[Reenviar](https://resend.com/emails) es una API de correo electrónico creada para desarrolladores. Se centra en una alta capacidad de entrega, facilidad de integración y una experiencia de desarrollador limpia para enviar correos electrónicos transaccionales y de marketing.

- **ZenOpus**: para generar la lógica de front-end y backend.
- **Supabase**: para autenticación y almacenamiento seguro de datos.

## Reenviar preguntas frecuentes sobre integración

<AccordionGroup>
<Accordion title="¿Qué es el reenvío?">
Resend es una API de correo electrónico creada para desarrolladores. Se centra en una alta capacidad de entrega, facilidad de integración y una experiencia de desarrollador limpia para enviar correos electrónicos transaccionales y de marketing.
</Accordion>
<Accordion title="¿Necesito un plan de reenvío pago para seguir este tutorial?">
No. Puede completar toda la configuración de CRM utilizando el **nivel gratuito de Resend**, que admite el envío de correos electrónicos desde un dominio verificado y hasta 3000 contactos para correos electrónicos de marketing.
</Accordion>
<Accordion title="¿Cómo verifico un dominio en Reenviar?">
Vaya a Reenviar → **Dominios** → Agregue su dominio. Se le proporcionarán registros DNS (TXT, MX) para agregar a través de su proveedor de DNS (por ejemplo, Cloudflare, Namecheap). Una vez agregado, haga clic en **Verificar**.

<Tip>
Utilice un subdominio como `updates.yourdomain.com` para aislar la reputación de su correo electrónico.
</Tip>
</Accordion>
<Accordion title="¿Por qué mis correos electrónicos de reenvío se convierten en spam?">
Hay algunas razones comunes:

- No has verificado tu dominio
- Su correo electrónico carece de una versión de texto sin formato
- Estás enviando desde un remitente genérico como `onboarding@resend.dev`
- No sigues las mejores prácticas (por ejemplo, faltan enlaces para cancelar la suscripción en los correos electrónicos de marketing)

Utilice el panel **Información sobre la capacidad de entrega** de Resend para obtener consejos prácticos.
</Accordion>
<Accordion title="¿Puedo personalizar el diseño de los correos electrónicos que envío?">
Sí. Utilice [React Email](https://react.email/), una biblioteca de código abierto del equipo de Resend, para diseñar correos electrónicos responsivos y con estilo con componentes de React.
</Accordion>
<Accordion title="¿Reenvía Support correos electrónicos de marketing o solo transaccionales?">
Reenviar admite **ambos**:

- **Correos electrónicos transaccionales** (confirmación, reinicio, notificaciones) usando API o SDK.
- **Correos electrónicos de marketing** a través de la función **Transmisión** y **Audiencias**.

Puede crear, obtener una vista previa y realizar un seguimiento de campañas desde el panel.
</Accordion>
<Accordion title="¿Cómo gestiono los suscriptores del boletín en Resend?">
Utilice **Audiencias**:

- Agregue contactos manualmente, mediante CSV o mediante programación utilizando la API.
- Cada audiencia tiene una identificación única que puede usar para asociar suscripciones.
- Los usuarios pueden darse de baja automáticamente a través del enlace de cancelación de suscripción integrado de Resend.
</Accordion>
<Accordion title="¿Puedo configurar la suscripción doble para recibir boletines?">
No de forma predeterminada, pero puedes implementarlo manualmente:

1. Al enviar el formulario, envíe un correo electrónico de confirmación inicial.
1. Pida a los usuarios que hagan clic en un enlace.
1. Al hacer clic, confirme su suscripción y agréguelos a su audiencia de reenvío.
</Accordion>
<Accordion title="¿Puedo programar o automatizar flujos de correo electrónico en Resend?">
Todavía no. Actualmente, el reenvío no incluye campañas de goteo Support ni secuencias automatizadas.

<Tip>
Solución alternativa:

Utilice **Supabase Cron Jobs** para programar tareas que activen el reenvío de correos electrónicos en intervalos establecidos.
</Tip>
</Accordion>
<Accordion title="¿Cómo conecto el reenvío a Supabase Auth?">
Utilice la integración oficial de Supabase mediante Resend:

1. Autentícate con tu proyecto Supabase.
1. Proporcione un dominio y un nombre de remitente.
1. Resend configura automáticamente sus correos electrónicos de autenticación de Supabase para usar su cuenta de Resend.
</Accordion>
<Accordion title="¿Cómo maneja Resend las cancelaciones de suscripción?">
Reenviar procesa automáticamente las bajas de suscripción:

- Debe incluir un enlace para cancelar la suscripción en los correos electrónicos de difusión.
- Cuando un usuario se da de baja, se elimina de la audiencia seleccionada.
- Esto es necesario para cumplir con el correo electrónico (por ejemplo, CAN-SPAM, GDPR).
</Accordion>
</AccordionGroup>
