## Was ist erneut senden?

[Erneut senden](https://resend.com/emails) ist eine E-Mail-API, die für Entwickler entwickelt wurde. Der Schwerpunkt liegt auf hoher Zustellbarkeit, einfacher Integration und einem sauberen Entwicklererlebnis beim Versenden von Transaktions- und Marketing-E-Mails.

## Häufig gestellte Fragen zur Integration erneut senden

<AccordionGroup>
<Accordion title="Was ist erneut senden?">
Resend ist eine E-Mail-API, die für Entwickler entwickelt wurde. Der Schwerpunkt liegt auf hoher Zustellbarkeit, einfacher Integration und einem sauberen Entwicklererlebnis beim Versenden von Transaktions- und Marketing-E-Mails.
</Accordion>
<Accordion title="Benötige ich einen kostenpflichtigen Resend-Plan, um diesem Tutorial zu folgen?">
Nein. Sie können die gesamte CRM-Einrichtung mit **Resends kostenlosem Kontingent** abschließen, das das Senden von E-Mails von einer verifizierten Domain und bis zu 3.000 Kontakten für Marketing-E-Mails unterstützt.
</Accordion>
<Accordion title="Wie verifiziere ich eine Domain in Resend?">
Gehen Sie zu Erneut senden → **Domains** → Fügen Sie Ihre Domain hinzu. Sie erhalten DNS-Einträge (TXT, MX), die Sie über Ihren DNS-Anbieter (z. B. Cloudflare, Namecheap) hinzufügen können. Klicken Sie nach dem Hinzufügen auf **Bestätigen**.

<Tip>
Verwenden Sie eine Subdomain wie `updates.yourdomain.com`, um Ihre E-Mail-Reputation zu isolieren.
</Tip>
</Accordion>
<Accordion title="Warum landen meine E-Mails zum erneuten Senden im Spam?">
Es gibt einige häufige Gründe:

- Sie haben Ihre Domain nicht bestätigt
- Ihrer E-Mail fehlt eine Nur-Text-Version
- Sie senden von einem generischen Absender wie `onboarding@resend.dev`
- Sie halten sich nicht an Best Practices (z. B. fehlende Abmeldelinks in Marketing-E-Mails)

Nutzen Sie das **Deliverability Insights**-Panel von Resend, um umsetzbare Tipps zu erhalten.
</Accordion>
<Accordion title="Kann ich das Design der E-Mails, die ich versende, anpassen?">
Ja. Verwenden Sie [React Email](https://react.email/) – eine Open-Source-Bibliothek des Resend-Teams – um gestaltete, reaktionsfähige E-Mails mit React-Komponenten zu entwerfen.
</Accordion>
<Accordion title="Sendet Support Marketing-E-Mails erneut oder nur transaktionale E-Mails?">
Resend unterstützt **beides**:

- **Transaktions-E-Mails** (Bestätigung, Zurücksetzen, Benachrichtigungen) über die API oder das SDK.
- **Marketing-E-Mails** über die Funktion **Broadcast** und **Zielgruppen**.

Sie können Kampagnen über das Dashboard erstellen, in der Vorschau anzeigen und verfolgen.
</Accordion>
<Accordion title="Wie verwalte ich Newsletter-Abonnenten in Resend?">
Verwenden Sie **Zielgruppen**:

- Fügen Sie Kontakte manuell, per CSV oder programmgesteuert über die API hinzu.
- Jede Zielgruppe verfügt über eine eindeutige ID, mit der Sie Abonnements verknüpfen können.
- Benutzer können sich automatisch über den integrierten Abmeldelink von Resend abmelden.
</Accordion>
<Accordion title="Kann ich Double-Opt-In für Newsletter einrichten?">
Nicht standardmäßig, aber Sie können es manuell implementieren:

1. Senden Sie beim Absenden des Formulars eine erste Bestätigungs-E-Mail.
1. Bitten Sie Benutzer, auf einen Link zu klicken.
1. Bestätigen Sie mit einem Klick ihr Abonnement und fügen Sie sie Ihrer Zielgruppe „Erneut senden“ hinzu.
</Accordion>
<Accordion title="Kann ich E-Mail-Verläufe in Resend planen oder automatisieren?">
Noch nicht. „Resend“ unterstützt derzeit keine Drip-Kampagnen oder automatisierten Sequenzen.

<Tip>
Problemumgehung:

Verwenden Sie **Supabase Cron Jobs**, um Aufgaben zu planen, die das erneute Versenden von E-Mails in festgelegten Intervallen auslösen.
</Tip>
</Accordion>
<Accordion title="Wie verbinde ich Resend mit Supabase Auth?">
Nutzen Sie die offizielle Supabase-Integration von Resend:

1. Authentifizieren Sie sich bei Ihrem Supabase-Projekt.
1. Geben Sie eine Domäne und einen Absendernamen an.
1. Resend konfiguriert Ihre Supabase-Authentifizierungs-E-Mails automatisch für die Verwendung Ihres Resend-Kontos.
</Accordion>
<Accordion title="Wie geht Resend mit Abmeldungen um?">
„Resend“ verarbeitet Abmeldungen automatisch:

- Sie müssen in Broadcast-E-Mails einen Abmeldelink einfügen.
- Wenn sich ein Benutzer abmeldet, wird er aus der ausgewählten Zielgruppe entfernt.
- Dies ist für die E-Mail-Konformität erforderlich (z. B. CAN-SPAM, DSGVO).
</Accordion>
</AccordionGroup>
