## Allgemeine Konzepte

- **KI (Künstliche Intelligenz):** Die Simulation menschlicher Intelligenzprozesse durch Maschinen, insbesondere Computersysteme, die Aufgaben wie Lernen, Denken und Problemlösen ermöglicht.
- [**Eingabeaufforderung**](/de/prompting/prompting-one)**:** Ein Text oder eine Eingabe, die KI-Modelle bei der Generierung von Ausgaben oder der Ausführung bestimmter Aufgaben anleitet. Es gehört zur Kernfunktionalität von ZenOpus und umfasst Eingabeaufforderungen, mit denen Sie eine bestimmte Komponente oder ein bestimmtes Element in Ihrer Anwendung erstellen oder ändern können.

## ZenOpus-spezifische Begriffe

- [**Plan mode**](/de/features/plan-mode)**:** ZenOpus geht über das bloße Generieren von Code hinaus; Es wird zu einem interaktiven Assistenten, der Sie durch jede Phase der Entwicklung führt. Es hilft Gründern, kritisch zu denken, effektiv zu planen, intelligent zu debuggen und selbstbewusst zu handeln.
- **Bearbeitungsmodus**: Die Aktion, bei der Änderungen oder Modifikationen an Inhalten oder Code vorgenommen werden.
- [**Bearbeiten**](/de/features/design)**:** Ein Tool, das eine KI-gesteuerte Entwicklung mit Tailwind-nativen visuellen Steuerelementen für eine einfache Verfeinerung ermöglicht.
- **Verlauf:** Das System zur Verfolgung und Verwaltung von Änderungen an Ihrer App und Ihrem Code im Laufe der Zeit.
- [**Wissen**](/de/features/knowledge)**:** Erfassen Sie wichtige Projektdetails in einem lebendigen Dokument, das sich mit Ihrer App weiterentwickelt.
- [**Labs**](/de/features/labs)**:** Experimentelle Funktionen innerhalb von ZenOpus zum Testen und Präsentieren neuer, innovativer oder in der Entwicklung befindlicher Funktionen. Diese können jederzeit geändert oder entfernt werden.
- **Remix:** Sie können ein bestehendes öffentliches Projekt oder eines Ihrer eigenen remixen. Durch Remixen können Sie den aktuellen Status eines Projekts als Ausgangspunkt wiederverwenden und darauf aufbauen. Dies ist eine großartige Möglichkeit, neue Ideen zu erkunden, Anpassungen vorzunehmen oder mit verschiedenen Änderungen zu iterieren und gleichzeitig die Originalversion beizubehalten. Beachten Sie, dass es nicht möglich ist, Projekte neu zu mischen, wenn Supabase verbunden ist.
- **Vorschau:** Ermöglicht Benutzern, Live-Inhalte oder Funktionsfunktionen auf interaktive Weise anzuzeigen oder zu erleben, bevor sie fertiggestellt oder veröffentlicht werden.
- **Diff:** Ein Vergleich, der die Unterschiede zwischen zwei Versionen einer Datei oder eines Codes zeigt.
- **/index:** Bezieht sich auf die Haupt- oder Standardseite einer Website oder App, oft mit dem Namen „index.html“ oder „index.js“.
- [**ZenOpus API – Mit URL erstellen**](/de/integrations/build-with-url)**:** Mit der Funktion „Mit URL erstellen“ der ZenOpus API können Sie Anwendungen programmgesteuert erstellen, indem Sie Eingabeaufforderungen und Bilder über URL-Parameter übergeben. Dies ermöglicht eine nahtlose Integration mit Websites, internen Tools oder Automatisierungsworkflows, sodass Benutzer ZenOpus-Apps mit einem einzigen Klick generieren können.

## Produkt- und Entwicklungskonzepte

### Produktmanagement und Strategie

- **MVP (Minimum Viable Product)** – Eine Version eines Produkts mit gerade genug Funktionen, um Erstanwender zufrieden zu stellen und eine Idee vor der vollständigen Entwicklung zu validieren.
- **Roadmap** – Ein übergeordneter strategischer Plan, der die Vision, Richtung und geplante Funktionen für ein Produkt im Laufe der Zeit umreißt.
- **Feature Request** – Ein formeller oder informeller Vorschlag von Benutzern oder Stakeholdern für eine neue Funktion in einem Produkt.
- **User Story** – Eine kurze, einfache Beschreibung einer Funktion oder Anforderung aus der Sicht eines Endbenutzers.
- **User Journey:** Die Reihe von Schritten, die ein Benutzer unternimmt, um ein bestimmtes Ziel innerhalb eines Produkts oder einer Dienstleistung zu erreichen.
- **Persona:** Eine fiktive Darstellung eines Zielbenutzers, basierend auf Recherchen, um Design- und Entwicklungsentscheidungen zu leiten.

### Ingenieurkonzepte

- **PRD (Produktanforderungsdokument):** Ein umfassendes Dokument, das die Ziele, Funktionen und Spezifikationen eines Produkts als Leitfaden für die Entwicklung beschreibt.
- [**API**](https://zenopus.dev/blog/2025-01-28-the-power-of-apis)**(Anwendungsprogrammierschnittstelle):** Eine Möglichkeit für verschiedene Dienste, zu kommunizieren. Betrachten Sie es als ein Protokoll, das Ihnen sagt, welche Art von Informationen Sie zwischen Systemen erhalten oder senden können. Die gebräuchlichsten Typen sind REST-APIs, die von vielen Websites verwendet werden, und GraphQL-APIs, die einen flexibleren Ansatz zum Abfragen von Daten bieten.
- **Refactor:** Der Prozess der Umstrukturierung von vorhandenem Code ohne Änderung seines externen Verhaltens, um die Lesbarkeit, Wartbarkeit oder Leistung zu verbessern.
- **GitHub:** Eine webbasierte Plattform für Versionskontrolle und kollaborative Softwareentwicklung mit Git.

### Daten und Analysen

- **A/B-Testing** – Eine Methode zum Vergleichen zweier Versionen einer Webseite oder Funktion, um anhand des Benutzerverhaltens festzustellen, welche die bessere Leistung erbringt.
- **Conversion-Rate** – Der Prozentsatz der Benutzer, die eine gewünschte Aktion abschließen, z. B. eine Anmeldung oder einen Kauf.
- **Abwanderungsrate** – Der Prozentsatz der Benutzer, die ein Produkt über einen bestimmten Zeitraum hinweg nicht mehr verwenden.
- **Retentionsrate** – Der Prozentsatz der Benutzer, die ein Produkt im Laufe der Zeit weiterhin verwenden.
- **Ereignisverfolgung** – Überwachung spezifischer Benutzeraktionen (z. B. Klicken auf Schaltflächen, Formularübermittlungen), um Erkenntnisse über Verhalten und Engagement zu gewinnen.

## Playbook für UI/UX und Frontend-Entwicklung

### Frontend-Entwicklung

- **Frontend:** Der Teil einer Softwareanwendung, mit dem Benutzer direkt interagieren, einschließlich der Benutzeroberfläche und Benutzererlebniselemente.
- **React:** Eine von Facebook entwickelte JavaScript-Bibliothek zum Erstellen von Benutzeroberflächen, insbesondere Single-Page-Anwendungen.
- **Verlauf:** Ein allmählicher Übergang zwischen zwei oder mehr Farben oder Schattierungen im Design.
- **Tailwind CSS**: Open-Source-Utility-First-CSS-Framework, das einen umfassenden Satz vordefinierter Klassen bereitstellt und es Entwicklern ermöglicht, benutzerdefinierte Designs direkt in ihrem HTML zu erstellen, indem sie diese Utility-Klassen auf Elemente anwenden.
- **Designsysteme:** Eine Sammlung wiederverwendbarer Komponenten, Richtlinien und Standards, die Konsistenz und Kohärenz im Design eines Produkts oder einer Marke gewährleisten.

### UI/UX-Designkonzepte

- **Akzentfarbe:** Ein markanter Farbton, der zur Hervorhebung wichtiger Elemente eines Designs verwendet wird und häufig mit der Primärfarbe der Marke übereinstimmt oder diese ergänzt, um das visuelle Interesse zu steigern.
- **Thema:** Eine zusammenhängende Reihe von Designoptionen, einschließlich Farben, Schriftarten und Layouts, die die Gesamtästhetik und das Benutzererlebnis einer Anwendung oder Website definieren.
- **Responsive Design:** Ein Webdesign-Ansatz, der sicherstellt, dass sich Inhalte nahtlos an verschiedene Gerätegrößen und -ausrichtungen anpassen und so ein optimales Benutzererlebnis bieten.
- **Above the Fold:** Der Teil einer Webseite, der für Benutzer ohne Scrollen sichtbar ist und entscheidend ist, um Aufmerksamkeit zu erregen und wichtige Informationen sofort zu übermitteln.
- **CTA (Call to Action):** Eine Aufforderung, häufig in Form einer Schaltfläche oder eines Links, die Benutzer dazu auffordert, eine bestimmte Aktion auszuführen, z. B. „Anmelden“ oder „Mehr erfahren“.

### Seitenstruktur und Navigation

- **Überschriften (Titel):** Textelemente, die Inhaltsabschnitte einleiten und organisieren, typischerweise mit unterschiedlichen Ebenen formatiert (z. B. H1, H2), um eine klare Inhaltshierarchie zu etablieren.
- **Fußzeile:** Der untere Abschnitt einer Webseite, der üblicherweise zusätzliche Informationen wie Kontaktdaten, Navigationslinks und rechtliche Haftungsausschlüsse enthält.
- **Breadcrumb:** Eine Navigationshilfe, die den aktuellen Standort des Benutzers innerhalb der Hierarchie einer Website anzeigt, oft dargestellt als horizontale Linkliste.
- **Favicon** – Ein kleines Symbol, das im Browser-Tab, in den Lesezeichen und in den Suchergebnissen angezeigt wird und normalerweise eine Marke oder Website darstellt.
- **Meta-Titel (Titel-Tag)** – Der Titel einer Webseite, der in Suchmaschinenergebnissen und Browser-Tabs erscheint und Einfluss auf SEO und Benutzer-Klickraten hat.
- **Meta-Beschreibung** – Eine kurze Zusammenfassung des Inhalts einer Webseite, der in Suchmaschinenergebnissen angezeigt wird, mit dem Ziel, die Auffindbarkeit und Benutzerinteraktion zu verbessern.
- **Kanonische URL** – Ein Tag, der zur Angabe der bevorzugten Version einer Webseite verwendet wird und dabei hilft, Probleme mit doppeltem Inhalt bei der Suchmaschinenindizierung zu vermeiden.
- **URL-Slug** – Der Teil einer URL, der eine bestimmte Seite in einem lesbaren Format identifiziert (z. B. `example.com/product-name`).
- **Sitemap** – Eine strukturierte Datei (XML oder HTML), die eine Liste der Seiten einer Website bereitstellt und Suchmaschinen dabei hilft, diese effizient zu indizieren.
- **Navigationsleiste (Navigationsleiste)** – Ein horizontales oder vertikales Menü mit Links zu wichtigen Abschnitten einer Website für eine einfache Navigation.
- **Links überspringen** – Versteckte Links, die es Benutzern ermöglichen, sich wiederholende Inhalte zu umgehen und direkt zum Hauptinhalt zu navigieren, wodurch die Zugänglichkeit verbessert wird.
- **Paginierung** – Eine Methode zur Aufteilung von Inhalten in mehrere Seiten, die häufig in Blogs, Suchergebnissen und Produktlisten verwendet wird, um die Benutzererfahrung zu verbessern.
- **Ankerlink** – Ein Hyperlink, der Benutzer zu einem bestimmten Abschnitt innerhalb derselben Seite weiterleitet, anstatt eine neue Seite zu laden.
- **404-Seite** – Eine benutzerdefinierte Fehlerseite, die angezeigt wird, wenn ein Benutzer versucht, auf eine nicht vorhandene URL zuzugreifen, und ihn zu relevanten Inhalten zurückführt.

### Benachrichtigungen und Feedback-Elemente

- **Toast:** Eine kurze, unaufdringliche Benachrichtigung, die vorübergehend erscheint, um Benutzer über das Ergebnis einer Aktion zu informieren oder einfaches Feedback zu geben.
- **Snackbar** – Ähnlich einem Toast, befindet sich jedoch normalerweise am unteren Bildschirmrand und bietet optionale Aktionen (z. B. „Rückgängig“ bei einem gelöschten Element).
- **Tooltip** – Ein kleines Popup-Fenster, das zusätzliche Informationen bereitstellt, wenn Sie mit der Maus über ein Element fahren oder es darauf fokussieren.
- **Badge** – Ein kleiner visueller Indikator, der häufig auf Symbolen oder Schaltflächen verwendet wird, um Benachrichtigungen, Zählungen oder Statusaktualisierungen anzuzeigen.
- **Loading Spinner (Loader)** – Ein visueller Indikator, der einen laufenden Prozess darstellt, z. B. das Laden einer Seite oder das Absenden eines Formulars.
- **Fortschrittsbalken** – Ein horizontaler Balken, der den Fertigstellungsgrad einer Aufgabe oder eines Prozesses anzeigt.
- **Skeleton Loader** – Eine Platzhalter-Benutzeroberfläche, die das Layout des endgültigen Inhalts nachahmt, während er darauf wartet, dass er geladen wird, und so die wahrgenommene Leistung verbessert.

### Overlays und Pop-ups

- **Popover:** Ein vorübergehendes Overlay, das zusätzliche Informationen oder Optionen im Zusammenhang mit einem bestimmten UI-Element darstellt und normalerweise bei Benutzerinteraktion angezeigt wird.
- **Dialog (modal):** Ein Fenster, das vor dem Hauptinhalt angezeigt wird, um die Aufmerksamkeit des Benutzers zu erregen. Oft ist eine Aktion erforderlich, bevor zur primären Benutzeroberfläche zurückgekehrt wird.
- **Schublade (Seitenleistenbereich)** – Ein Bereich, der von der Seite des Bildschirms eingeschoben wird, um Navigationsoptionen oder -einstellungen anzuzeigen, ohne den Hauptinhalt zu stören.
- **Lightbox** – Ein modales Fenster, das Bilder oder Medien in einer vergrößerten Ansicht anzeigt und den Hintergrund zur Fokussierung abdunkelt.
- **Alert Box** – Ein vom System generiertes oder von der Anwendung ausgelöstes Meldungsfeld, das Benutzer über wichtige Informationen wie Fehler oder Warnungen informiert.

### Navigations- und Auswahlelemente

- **Schaltfläche:** Ein anklickbares Element, das eine Aktion oder ein Ereignis auslöst, z. B. das Absenden eines Formulars oder das Öffnen eines Dialogs.
- **Schalter (Umschalten):** Eine Umschaltsteuerung, mit der Benutzer zwischen zwei Zuständen wechseln können, z. B. Ein oder Aus, was häufig einem physischen Schalter ähnelt.
- **Registerkarten** – Eine Komponente, die Inhalte in separaten Ansichten innerhalb einer einzigen Benutzeroberfläche organisiert, sodass Benutzer zwischen Abschnitten wechseln können, ohne wegnavigieren zu müssen.
- **Stepper (Assistent)** – Eine Komponente, die für mehrstufige Prozesse verwendet wird, den Benutzern ihren aktuellen Schritt anzeigt und sie durch den Abschluss führt.
- **Paginierung** – Ein UI-Muster, das Inhalte in Seiten unterteilt und häufig für Suchergebnisse oder inhaltsintensive Anwendungen verwendet wird.
- **Breadcrumb** – Eine Navigationshilfe, die den aktuellen Standort des Benutzers innerhalb der Hierarchie einer Website anzeigt, oft dargestellt als horizontale Linkliste.
- **Akkordeon** – Ein zusammenklappbarer Abschnitt, der sich erweitert oder verkleinert, um Inhalte dynamisch anzuzeigen oder auszublenden.
- **Dropdown-Menü** – Eine Liste von Optionen, die angezeigt wird, wenn ein Benutzer auf eine Schaltfläche oder ein Feld klickt oder mit der Maus darüber fährt. Sie wird häufig zur Navigation oder Auswahl verwendet.

### Formulare und Eingabeelemente

- **Formular:** Eine strukturierte Sammlung von Eingabefeldern, die es Benutzern ermöglicht, Daten wie Kontaktinformationen oder Suchanfragen einzureichen.
- **Optionsgruppe:** Eine Reihe verwandter Optionsfelder, mit denen Benutzer aus mehreren Optionen nur eine Option auswählen können.
- **Kontrollkästchen:** Ein interaktives Kästchen, das Benutzer aktivieren oder deaktivieren können, um eine Option auszuwählen oder die Auswahl aufzuheben, wodurch mehrere Auswahlen innerhalb eines Satzes möglich sind.
- **Textfeld (Eingabefeld)** – Ein grundlegendes UI-Element, in das Benutzer Text eingeben können, beispielsweise in Suchleisten, Anmeldeformularen oder Kommentaren.
- **Textbereich** – Ein größeres Eingabefeld für mehrzeilige Texteingaben, das häufig für Nachrichten oder Feedbackformulare verwendet wird.
- **Auswählen (Dropdown-Auswahlfeld)** – Ein UI-Element, das es Benutzern ermöglicht, eine Option aus einer vordefinierten Liste auszuwählen, die oft als Dropdown-Liste gestaltet ist.
- **Datumsauswahl** – Ein UI-Element, mit dem Benutzer ein Datum aus einem Kalender auswählen können, anstatt es manuell einzugeben.
- **Schieberegler (Bereichsauswahl)** – Ein Steuerelement, mit dem Benutzer einen Wert innerhalb eines Bereichs festlegen können, indem sie einen Griff entlang einer Spur verschieben.
- **Datei-Upload-Feld** – Eine Komponente, mit der Benutzer Dateien von ihrem Gerät auswählen und hochladen können.
- **Automatische Vervollständigung (Typeahead-Suche)** – Eine Texteingabe, die während der Eingabe durch den Benutzer dynamisch passende Ergebnisse vorschlägt und so die Benutzerfreundlichkeit bei Suchen verbessert.

### Designstile und Trends

- **Neobrutalismus:** Ein Webdesign-Trend, der sich durch rohe, ungeschliffene Elemente, kräftige Typografie und hohen Kontrast auszeichnet und sich von der brutalistischen Architektur der Mitte des 20. Jahrhunderts inspirieren lässt.
- **Retro:** Design, das Stile, Motive und Elemente vergangener Jahrzehnte einbezieht und Nostalgie und Vintage-Ästhetik hervorruft.
- **Hacker:** Eine Ästhetik, die die DIY- und Open-Source-Kultur widerspiegelt und häufig monospaced Schriftarten, dunkle Hintergründe und Terminal-inspirierte Schnittstellen aufweist.
- **Glas (Glasmorphismus):** Ein Designstil verwendet durchscheinende, milchglasartige Elemente mit subtilen Schatten und Rändern, um Tiefe zu erzeugen.
- **Nudy:** Ein minimalistischer Designansatz, der Nude- oder neutrale Farbpaletten verwendet, um eine weiche und dezente Ästhetik zu schaffen.

### Design- und Asset-Ressourcen

- [**21st.dev:**](https://21st.dev/) Eine Open-Source-Community-Registrierung, die minimale, moderne und wiederverwendbare React-UI-Komponenten bietet, die auf Tailwind CSS und Radix UI basieren und Designingenieuren dabei helfen sollen, ausgefeilte Benutzeroberflächen effizienter zu liefern.
- [**Dribble:**](https://dribbble.com/) Eine führende Online-Community, in der Designer und Kreativprofis ihre Arbeiten präsentieren, Design-Inspirationen entdecken und mit potenziellen Kunden oder Kooperationspartnern in Kontakt treten.
- [**Noun Project:**](https://thenounproject.com/) Eine Plattform, die eine umfangreiche Sammlung kostenloser Symbole und Stockfotos bereitstellt und Benutzern den Zugriff auf visuelle Ressourcen für verschiedene Projekte ermöglicht.
- [SVG Repo:](https://www.svgrepo.com/) Eine umfassende Bibliothek mit über 500.000 offen lizenzierten SVG-Vektoren und -Symbolen, die es Benutzern ermöglicht, für verschiedene Projekte geeignete Grafiken zu suchen, zu erkunden, zu bearbeiten und herunterzuladen.
- [Google Fonts:](https://fonts.google.com/) Eine kostenlose Bibliothek weboptimierter Schriftarten, die Designern und Entwicklern eine große Auswahl an Typografieoptionen zur Verbesserung ihrer Projekte bietet.
- [Typewolf:](https://www.typewolf.com/) Eine Ressource, die Designer bei der Auswahl der perfekten Schriftartenkombinationen für ihre Projekte unterstützt, indem sie trendige Schriftarten präsentiert und kuratierte Schriftartenlisten anbietet.

## Backend-Entwicklung und Datenbanken

### Backend-Grundlagen

- **Backend:** Die serverseitige Infrastruktur einer Anwendung, die Datenverarbeitung, Speicherung und Geschäftslogik übernimmt.
- [**Supabase**](/de/integrations/supabase)**:** Eine Open-Source-Backend-as-a-Service-Plattform, die eine Postgres-Datenbank, Authentifizierung, Instant-APIs und Echtzeitfunktionen bereitstellt.
- **Webhooks** – Ereignisgesteuerte HTTP-Rückrufe, die externe Anwendungen benachrichtigen, wenn bestimmte Aktionen stattfinden.

### Datenbankverwaltung und Abfragen

- **CRUD (Erstellen, Lesen, Aktualisieren, Löschen):** Die vier grundlegenden Vorgänge, die an Daten in einer Datenbank oder Anwendung ausgeführt werden.
- [**SQL (Structured Query Language)**](https://supabase.com/docs/guides/database/overview)**:** Eine domänenspezifische Programmiersprache, die für die Verwaltung und Bearbeitung relationaler Datenbanken entwickelt wurde und Aufgaben wie das Abfragen, Aktualisieren und Organisieren von Daten ermöglicht.
- **PostgreSQL** – Eine leistungsstarke relationale Open-Source-Datenbank, die für ihre Skalierbarkeit, Erweiterbarkeit und ACID-Konformität bekannt ist.

### Supabase-spezifische Konzepte

- [**Edge-Funktion**](https://supabase.com/docs/guides/functions)**:** Serverseitige TypeScript-Funktionen, global am Edge verteilt – nah an Ihren Benutzern. Sie können zum Abhören von Webhooks oder zur Integration Ihres Supabase-Projekts mit Drittanbietern wie Stripe, Anthropic oder Resend verwendet werden.
- [**RLS (Row-Level Security)**](https://supabase.com/docs/guides/database/postgres/row-level-security)**:** Eine Funktion in Datenbanken, die die Kontrolle über den Zugriff auf bestimmte Zeilen in einer Tabelle basierend auf Benutzerrollen oder -attributen ermöglicht – im Wesentlichen ermöglicht es Benutzern, detaillierte Autorisierungsregeln festzulegen, sodass Sie komplexe SQL-Regeln schreiben können, die Ihren individuellen Geschäftsanforderungen entsprechen.
- [**Speicher**](https://supabase.com/docs/guides/storage)**:** Ein Dienst oder System zum Speichern und Verwalten von Daten wie Bildern, Videos, Dokumenten und anderen Dateitypen.
- [**Bucket**](https://supabase.com/docs/guides/storage/buckets/fundamentals)**:** In Supabase Storage ist ein Bucket ein eindeutiger Container, der Dateien und Ordner organisiert, das Zugriffsmodell (öffentlich oder privat) bestimmt und Upload-Einschränkungen wie maximale Dateigröße und zulässige Inhaltstypen festlegt.
- [**Endpunkt**](https://supabase.com/docs/guides/api)**:** Im Kontext von Supabase bezieht sich ein Endpunkt auf eine bestimmte URL, die es Clients ermöglicht, über RESTful-API-Aufrufe mit der Datenbank zu interagieren und so Vorgänge wie das Erstellen, Lesen, Aktualisieren oder Löschen von Daten zu ermöglichen.
- [**Authentifizierungsanbieter:**](https://supabase.com/docs/guides/auth) Eine Entität oder ein Dienst, der Benutzeranmeldeinformationen überprüft und so einen sicheren Zugriff auf Anwendungen oder Systeme ermöglicht. Im Kontext von Supabase umfassen Authentifizierungsanbieter verschiedene Methoden wie passwortbasierte Anmeldungen, Magic Links, Einmalpasswörter (OTP), soziale Anmeldungen und Single Sign-On (SSO)-Integrationen, die eine flexible und sichere Benutzerauthentifizierung ermöglichen.

### Sicherheit und Authentifizierung

- **OAuth** – Ein weit verbreitetes Authentifizierungsprotokoll, das es Benutzern ermöglicht, sich über Drittanbieterdienste wie Google, Facebook oder GitHub bei Anwendungen anzumelden.
- **Zwei-Faktor-Authentifizierung (2FA)** – Eine Sicherheitsmaßnahme, bei der Benutzer vor dem Zugriff auf ein Konto zwei Arten der Identifizierung angeben müssen.
- **CORS (Cross-Origin Resource Sharing)** – Eine Sicherheitsrichtlinie, die steuert, welche Websites oder Anwendungen auf Ressourcen auf einem Server zugreifen können.
