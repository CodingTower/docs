Willkommen zu dieser Schritt-für-Schritt-Anleitung zum Erstellen einer Full-Stack-Anwendung mit ZenOpus:

![ZenOpus Dashboard mit Projektkarten, Navigationsseitenleiste und Eingabebereich für Eingabeaufforderungen](/images/zenopus/introduction/getting-started/2025-dec-10-new-dashborad.png)

## Die ZenOpus-Plattformübersicht

<AccordionGroup>
<Accordion title="Erstellen Sie Ihr erstes Projekt">
<Steps>
<Step title="Schritt 1">
Gehen Sie zunächst zu [zenopus.dev](https://zenopus.dev) und erstellen Sie ein Konto. Nach der Registrierung können Sie Ihr erstes Projekt erstellen.
</Step>
<Step title="Schritt 2">
Geben Sie einfach eine erste Eingabeaufforderung ein, um loszulegen, und ZenOpus erledigt den Rest! Dies ist der **Startblock** für jedes Projekt in ZenOpus, bei dem Sie Ihre Ideen sofort zum Leben erwecken können.
</Step>
</Steps>
</Accordion>
<Accordion title="ZenOpus-Dashboard">
Bei ZenOpus ist Einfachheit der Schlüssel:

- [**Hier**](https://zenopus.dev/)**starten Sie Ihr Projekt** mit einer einzigen Eingabeaufforderung.
- **Entdecken Sie alle Projekte**, einschließlich der neuesten und vorgestellten.
- **Starten Sie mit einer Vorlage** aus unserer kuratierten Auswahl.
- **Personalisieren Sie Ihr Erlebnis**, indem Sie Ihr Profil anzeigen und bearbeiten.
- [**Verwalten Sie Ihr Konto**](https://zenopus.dev/settings), um das monatliche Guthaben zu überprüfen und Einstellungen anzupassen.
- [**Upgrade oder Downgrade**](https://zenopus.dev/settings/plans)**Ihren Plan** nahtlos.
</Accordion>
<Accordion title="Projektübersicht">
Bei ZenOpus folgt jedes Projekt einem strukturierten Arbeitsablauf:

- **Einfache Bearbeitung** über eine Chat-basierte Oberfläche.
- **Fügen Sie Bilder hinzu** für präzise Bearbeitungen und Inspiration.
- **Komponenten auswählen und visuell bearbeiten** für ein nahtloses Designerlebnis.
- **Müheloser Wechsel** zwischen Agent mode und Plan mode.
- **Verfolgen Sie Änderungen** mit dem Versionsverlauf und kehren Sie zu einer früheren Version zurück.
- **Navigieren Sie mühelos durch die Seiten** Ihres Projekts.
- **Aktivieren Sie das Remixen** oder benennen Sie Ihr Projekt nach Bedarf um.
- **Verwenden Sie ZenOpus Cloud** (oder integrieren Sie es in Supabase) für Backend-Funktionen.
- **Arbeiten Sie über GitHub zusammen**, um Ihren Code zu bearbeiten und zu verwalten.
- **Veröffentlichen, bereitstellen und teilen** Sie Ihr Projekt mit einem Vorschau-Link (denken Sie daran, es nach Änderungen erneut zu veröffentlichen).
- **Umschalten zwischen Web- und Mobilansicht** für responsives Design.
- **Aktualisieren Sie das Vorschaufenster**, um Aktualisierungen sofort zu sehen.
- **Öffnen Sie Ihr Projekt direkt** über den Vorschau-Link.
</Accordion>
<Accordion title="Projekteinstellungen">
Die Einstellungen jedes Projekts ermöglichen Ihnen Folgendes:

- **Sehen Sie sich wichtige Projektdetails an** wie die Gesamtzahl der Bearbeitungen und das Erstellungsdatum.
- **Richten Sie benutzerdefiniertes Wissen ein**, um den Kontext Ihres Projekts anzupassen.
- **Stellen Sie eine Verbindung zu einem GitHub-Repository her** für eine nahtlose Zusammenarbeit.
- **Passen Sie die Sichtbarkeit des Projekts an** nach Bedarf.
- **Verwalten Sie die Badge-Anzeige** (nur für kostenpflichtige Pläne verfügbar).
- **Benennen Sie Ihr Projekt jederzeit um**.
- **Löschen Sie Ihr Projekt** (⚠️ Gefahrenzone).
</Accordion>
<Accordion title="Projekt-Schnellzugriff">
Sie haben jederzeit schnellen Zugriff auf:

- **Dashboard** für einen Überblick über Ihre Projekte.
- **Kontoeinstellungen** zur Verwaltung Ihres Profils und Ihrer Präferenzen.
- **Dokumentation und Support** für Anleitung und Support.
- **Melden Sie sich sicher** von Ihrem ZenOpus-Konto ab.
</Accordion>
</AccordionGroup>

## Starten Sie Ihr Projekt

Abhängig von Ihren Vorlieben und Ressourcen gibt es mehrere Möglichkeiten, mit ZenOpus zu beginnen:

<AccordionGroup>
<Accordion title="Prompt">
Das auf Eingabeaufforderungen basierende System von ZenOpus vereinfacht die App-Erstellung.

- Beschreiben Sie einfach im Eingabeaufforderungsfeld, was Sie erstellen möchten.
- Je spezifischer Sie sind, desto besser sind die Ergebnisse.
- Beginnen Sie mit klaren und detaillierten Anweisungen.
- Sie können Ihr Projekt im Laufe der Zeit verfeinern und anpassen.

<Info>
Beispiel:

*„Erstellen Sie ein Dashboard mit Benutzeranmeldung, monatlichen Verkäufen in einem Liniendiagramm und Kundendemografien in einem Kreisdiagramm.“*
</Info>
</Accordion>
<Accordion title="Remixen Sie ein bestehendes Projekt">
Sie können ein bestehendes öffentliches Projekt oder eines Ihrer eigenen remixen. Durch Remixen können Sie den aktuellen Status eines Projekts als Ausgangspunkt wiederverwenden und darauf aufbauen. Dies ist eine großartige Möglichkeit, neue Ideen zu erkunden, Anpassungen vorzunehmen oder mit verschiedenen Änderungen zu iterieren und gleichzeitig die Originalversion beizubehalten.

<Note>
Sie können öffentliche Projekte von anderen Benutzern nur dann remixen, wenn das Projekt nicht mit Supabase verbunden ist.
</Note>
</Accordion>
<Accordion title="Mit Figma">
Wenn Sie ein Design in Figma haben, machen Sie einen Screenshot eines beliebigen Teils davon. Sie können den Screenshot direkt in ZenOpus einfügen oder die Bilddatei per Drag-and-Drop verschieben. Sobald Sie die Eingabetaste drücken, wandelt ZenOpus Ihr Design in funktionalen Code um.
</Accordion>
<Accordion title="Anhand einer Skizze">
<Steps>
<Step title="Schritt 1">
Verwenden Sie [**Excalidraw**](https://excalidraw.com/) oder ein ähnliches Tool, um Ihre Benutzeroberfläche zu skizzieren.
</Step>
<Step title="Schritt 2">
Machen Sie einen Screenshot Ihrer Zeichnung und fügen Sie ihn dann ein oder ziehen Sie ihn per Drag-and-Drop in ZenOpus. Die Plattform wandelt Ihre Skizze in funktionierenden Code um.
</Step>
</Steps>

[Weitere Informationen hier.](https://www.youtube.com/watch?v=c0zhLzcVJRI)
</Accordion>
<Accordion title="Klonen einer Website oder Anwendung">
Wenn Sie eine vorhandene Webseite replizieren möchten:

<Steps>
<Step title="Schritt 1">
Machen Sie einen Screenshot mit Tastenkombinationen wie *Cmd+Shift+4* auf dem Mac oder Tools wie der [GoFullPage Chrome Extension](https://chromewebstore.google.com/detail/gofullpage-full-page-scre/fdpohaocaechififmbbbbbknoalclacl).
</Step>
<Step title="Schritt 2">
Fügen Sie den Screenshot ein oder ziehen Sie ihn in ZenOpus. ZenOpus erstellt die Struktur der Webseite in Ihrem Projekt neu.
</Step>
</Steps>

[Weitere Informationen hier.](https://www.youtube.com/watch?v=tYDqBMilHkM)
</Accordion>
</AccordionGroup>

## Bearbeiten Sie Ihr Projekt

<AccordionGroup>
<Accordion title="Fügen Sie Ihrem Projekt individuelles Wissen hinzu">
Das [benutzerdefinierte Wissen] (/de/features/knowledge) in ZenOpus fungiert als **Blaupause** Ihres Projekts und organisiert Funktionalität, Design und Ziele an einem Ort.
</Accordion>
<Accordion title="Verwenden Sie visuelle Bearbeitungen">
Mit Visual Edits können Sie Elemente auf der Seite auswählen und diese entweder sofort bearbeiten, z. B. Text oder Farben aktualisieren, oder Eingabeaufforderungen verwenden, um Funktionalität und Layout anzupassen.

Weitere Informationen finden Sie unter [Visuelle Bearbeitungen](/de/features/visual-edit).
</Accordion>
<Accordion title="Machen Sie eine Änderung rückgängig oder bearbeiten Sie eine Nachricht und verwenden Sie dann wieder einen anderen Ansatz">
Beim Erstellen KI-gesteuerter Apps kommt es auf Geschwindigkeit an, aber auch auf Stabilität und Kontrolle. Aus diesem Grund haben wir die Versionierung aktualisiert, um die Nachverfolgung von Änderungen mühelos zu ermöglichen.

<Steps>
<Step title="Kehren Sie sofort zu früheren Versionen zurück oder wählen Sie einen anderen Ansatz">
Sie können eine Vorschau einer älteren Arbeitsversion anzeigen und dann bei Bedarf zu dieser Version zurückkehren oder eine frühere Nachricht bearbeiten und zurückkehren, um einen anderen Ansatz zu erkunden. Nichts geht verloren – alle nach diesem Zeitpunkt vorgenommenen Änderungen bleiben im Chat und können jederzeit erneut angewendet werden.
</Step>
<Step title="Änderungen an den Lesezeichentasten">
Halten Sie Ihre Arbeit organisiert, indem Sie wichtige Änderungen mit Lesezeichen versehen, sodass Sie sie später leicht finden und erneut aufrufen können.
</Step>
<Step title="Intuitiver Versionsverlauf">
Das Verlaufsfenster funktioniert jetzt wie Google Docs und erleichtert so die Nachverfolgung von Änderungen.
</Step>
</Steps>
</Accordion>

## Fügen Sie Ihrem Projekt Fullstack-Funktionen hinzu

<AccordionGroup>
<Accordion title="Verbinden Sie ein Backend">
Wenn Sie Ihrer Anwendung Backend-Funktionen hinzufügen müssen, können Sie:

- Stellen Sie eine Verbindung zum integrierten Backend von ZenOpus her –  [ZenOpus Cloud](/de/features/cloud)
- Verwenden Sie die native [Supabase-Integration](/de/integrations/supabase) von ZenOpus.
</Accordion>
<Accordion title="Fügen Sie mit Stripe Zahlungsfunktionen zu Ihrer Anwendung hinzu">
Es gibt viele Möglichkeiten, Stripe mit ZenOpus zu verbinden, aber die einfachste Möglichkeit ist die Verwendung der integrierten Zahlungslinks von Stripe. Weitere Informationen finden Sie unter [Stripe-Integration](/de/integrations/stripe).

## Stellen Sie Ihr Projekt bereit und veröffentlichen Sie es

<AccordionGroup>
<Accordion title="Richten Sie SEO ein">
Sorgen Sie für die besten Ergebnisse, indem Sie unserem Leitfaden [Best Practices für SEO und GEO implementieren](/de/tips-tricks/seo-geo) folgen.
</Accordion>
<Accordion title="Fügen Sie eine benutzerdefinierte Domäne hinzu">
Sie können eine benutzerdefinierte Domäne hinzufügen, indem Sie Folgendes verwenden:

- Eintrag (nativ in ZenOpus)
- Netlify
- Vercel
- Namecheap

Weitere Informationen finden Sie unter [Benutzerdefinierte Domäne](/de/features/custom-domain).
</Accordion>
<Accordion title="Mein Projekt veröffentlichen">
Wenn Ihr Projekt fertig ist, veröffentlichen Sie es, um eine gemeinsam nutzbare URL zu generieren. Dies macht es einfach, Ihre Anwendung zu präsentieren oder sie mit Mitarbeitern und Stakeholdern zu teilen.
