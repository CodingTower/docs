**MCP (Model Context Protocol)** è uno standard aperto che collega i sistemi di intelligenza artificiale, come ZenOpus, a strumenti, servizi e origini dati esterni. In ZenOpus, i server MCP vengono utilizzati come **connettori personali**.

Collegando ZenOpus a strumenti come Notion, Atlassian o Linear, l'agente ZenOpus può utilizzare il contesto reale del team per generare codice, prototipi e documentazione più accurati.

Una volta connesso, ZenOpus può leggere dati strutturati (come testo di documenti, problemi o diagrammi) e, ove supportato, eseguire azioni limitate come la creazione o l'aggiornamento di elementi.

ZenOpus include diversi connettori personali predefiniti su tutti i piani, con Support per server MCP personalizzati su tutti i piani a pagamento.

## Perché utilizzare connettori personali

I connettori personali sbloccano due funzionalità chiave:

- **Porta il tuo contesto esistente in ZenOpus.**  
L'agente ZenOpus può leggere la documentazione, i ticket e i file di progettazione del tuo team per creare prototipi e flussi in linea con i tuoi standard.
- **Collega le tue app ZenOpus ai tuoi flussi di lavoro.**  
L'agente ZenOpus può eseguire azioni di follow-up negli strumenti connessi, come l'aggiornamento degli stati dei ticket, l'aggiunta di collegamenti prototipo come commenti o la creazione di nuovi elementi quando supportato.

ZenOpus diventa più potente quando comprende il mondo del tuo team, eliminando congetture, accelerando l'iterazione e aiutandoti a passare dall'**idea all'app live più velocemente che mai.**

### Come aggiungere un connettore personale predefinito

Per utilizzare integrazioni predefinite con app e servizi di terze parti più diffusi:

1. Vai a **Impostazioni → Connettori → Connettori personali**.
1. **Seleziona un connettore personale** (ad esempio, Notion o Linear).
1. Fare clic su **Connetti**. Accedi e concedi a ZenOpus l'accesso al tuo account.

### Configurazione aggiuntiva per connettori personali specifici

[n8n](https://docs.n8n.io/advanced-ai/accessing-n8n-mcp-server)

Oltre a questi passaggi, devi eseguire le seguenti operazioni in n8n:

- Vai su **Impostazioni → Accesso MCP** e attiva **Abilita accesso MCP** (richiede il proprietario dell'istanza o le autorizzazioni di amministratore)
- Per ogni flusso di lavoro che desideri esporre, vai su **Impostazioni** nell'editor del flusso di lavoro e attiva **Disponibile in MCP.**

[Ampiezza](https://amplitude.com/)

Se prevedi di analizzare la tua app realizzata con ZenOpus, assicurati che stia già inviando dati ad Amplitude prima di utilizzare il connettore MCP. Consulta la [Guida rapida di Amplitude](https://amplitude.com/docs/get-started/amplitude-quickstart) per le istruzioni di configurazione.

## Server MCP personalizzati (piani a pagamento)

Per la massima flessibilità, puoi connettere **server MCP personalizzati** come connettori personali per portare sistemi interni o di terze parti in ZenOpus. Questo è perfetto per API private, CRM interni o origini dati personalizzate.

Esplora [server MCP remoti](https://mcpservers.org/remote-mcp-servers), un elenco curato di server MCP remoti di alta qualità compatibili con ZenOpus.

### Come connettere un server MCP personalizzato

Per connettere un server MCP personalizzato o interno:

1. Vai a **Impostazioni → Connettori → Connettori personali.**
1. Fare clic su **Nuovo server MCP**.
1. **Nome server**: inserisci un nome chiaro per il tuo server (ad esempio, *CRM interno* o *API Analytics*).
1. **URL del server**: inserisci l'indirizzo dove ZenOpus può raggiungere il tuo server MCP.
1. **Autenticazione:**
    - **OAuth (impostazione predefinita):** ZenOpus utilizza OAuth per impostazione predefinita. Dopo aver fatto clic su **Aggiungi e autorizza**, ti verrà richiesto di autorizzare.
    - **Token di connessione o chiave API:** per i server che non Support OAuth, puoi eseguire l'autenticazione utilizzando un token di connessione o una chiave API.
    - **Nessuna autenticazione**: connettiti senza credenziali se il server non richiede l'autenticazione.
1. **Aggiungi server**. Il server MCP personalizzato ora viene visualizzato nell'elenco dei connettori personali e può fornire dati contestuali per le build.

## Esempi pratici

Ogni app connessa diventa immediatamente disponibile come contesto per l'agente ZenOpus. Puoi interagire con gli strumenti connessi utilizzando istruzioni naturali.

| Scenario | Esempio di richiesta | Risultato di esempio |
| --- | --- | --- |
| Un product manager collega Linear o Jira | Genera un prototipo basato sui ticket nello sprint corrente. | ZenOpus legge le descrizioni dei problemi e i criteri di accettazione per creare un'interfaccia utente allineata e testabile. |
| Un esperto di marketing collega Notion o Confluence | Trasforma il brief della nostra campagna del quarto trimestre in una landing page. | ZenOpus utilizza il documento come contesto per generare una pagina modificabile in linea con il marchio. |
| Un designer collega Miro | Prototipa il flusso di registrazione dalla nostra lavagna. | ZenOpus legge i diagrammi di flusso per la navigazione dell'interfaccia utente dell'impalcatura e li trasforma in un prototipo interattivo che il tuo team può testare. |
| Un tecnico collega n8n o MCP interno | Crea un'interfaccia utente del flusso di lavoro che elenchi gli utenti e consenta di eliminarli. | ZenOpus crea un'app che utilizza dati e flussi di lavoro in tempo reale. |

## Sicurezza e autorizzazioni

I connettori personali (connessioni al server MCP) sono **connessioni per utente**. Puoi rivederli o revocarli in qualsiasi momento in **Impostazioni → Connettori**.

Per un accesso rapido all'interno di un progetto, fai clic su `+` nella finestra di messaggio e vai a **Connettori**.

Sui piani **Business** e **Enterprise**, Workspace **amministratori e proprietari** possono gestire quali server sono disponibili come connettori personali per tutti gli utenti:

- Vai su **Impostazioni → Privacy e sicurezza** per abilitare o disabilitare tutto l'**accesso al server MCP** per l'intero Workspace.
- Vai a **Impostazioni → Connettori → Connettori personali → Impostazioni amministratore** per abilitare o disabilitare i singoli server MCP come connettori personali.
